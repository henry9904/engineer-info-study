# 1. 서브넷

## 1-1. IP 클래스

### 방법 — 첫 옥텟으로 판별

| 클래스 | 첫 옥텟 범위 | 첫 비트 | 기본 마스크 |
|--------|------------|--------|------------|
| A | 0 ~ 127 | 0 | 255.0.0.0 (/8) |
| B | 128 ~ 191 | 10 | 255.255.0.0 (/16) |
| C | 192 ~ 223 | 110 | 255.255.255.0 (/24) |
| D | 224 ~ 239 | 1110 | (멀티캐스트) |
| E | 240 ~ 255 | 1111 | (연구용) |

> **암기**: A(1~127), B(128~191), C(192~223) — 경계 128, 192, 224.

### 정답표

| 문제 | 정답 |
|------|------|
| 192.1.2.3 | **C 클래스** |
| 129.6.8.4 | **B 클래스** |
| ①165.., ②210.., ③65.. | **① B ② C ③ A** |
| B 클래스 범위 | **128.0.0.0 ~ 191.255.255.255** |

## 1-2. 서브넷 마스크

### 방법 — 빌린 비트 계산

```
서브넷 수 = 2^(빌린 비트),   호스트 수 = 2^(호스트 비트) − 2
(−2 : 네트워크 주소 + 브로드캐스트 주소 제외)
CIDR /n → 앞에서 n비트가 1
```

### 대표 예제
- **/25** = 255.255.255.**128** (128=1000 0000)
- **/26** = 255.255.255.**192**
- **C클래스 26개 서브넷·호스트 4~5개**: 호스트 3비트(2³−2=6≥5), 나머지 5비트 서브넷 → 마스크 255.255.255.**248** (/29)
- **B클래스 6개 서브넷, 최대 호스트**: 서브넷 3비트(2³=8≥6) → /19 → 255.255.**224**.0
- **A클래스 100개 서브넷**: 7비트(2⁷=128≥100) → /15 → 255.**254**.0.0

### 정답표

| 문제 | 정답 |
|------|------|
| A/B/C 기본 마스크 | 255.0.0.0 / 255.255.0.0 / 255.255.255.0 |
| /25, /26 | 255.255.255.128, 255.255.255.192 |
| C·26서브넷·호스트4~5 | 255.255.255.248 |
| B·mask240 서브넷/호스트 | **서브넷 4096, 호스트 14** |
| B·6서브넷 최대호스트 | 255.255.224.0 |
| A·100서브넷 | 255.254.0.0 |
| C·25호스트 최다서브넷(203.253.55.0) | 255.255.255.224 |
| C·호스트4~5 | 255.255.255.248 |
| B(135.45)·호스트1000 | **마스크 255.255.252.0, 서브넷 64개** |
| mask 224 호스트 수 | **30개** (2⁵−2) |
| mask 192 → 3서브넷 호스트 | **62개** (2⁶−2) |
| C mask 255.255.255.0 | **254개** (2⁸−2) |
| C mask 192 서브넷 수 | **4개** (2²) |

> ⚠️ B·mask240: /28이므로 서브넷비트=28−16=12 → 2¹²=**4096**, 호스트비트=4 → 2⁴−2=**14**.

## 1-3. 서브넷 계산 (브로드캐스트 · 사용가능 IP)

### 방법 — 블록 크기로 구간 나누기

```
블록 크기 = 256 − (마스크의 해당 옥텟)
각 서브넷 = 0, 블록, 2블록 ... 로 시작
브로드캐스트 = 다음 블록 시작 − 1
사용 가능 = (네트워크+1) ~ (브로드캐스트−1)
```

### 대표 예제 — 192.168.3.157, mask 255.255.255.192
```
블록 = 256−192 = 64 → 구간: 0~63, 64~127, 128~191, 192~255
157 → 128~191 구간 → 브로드캐스트 = 191 → 192.168.3.191
```

### 정답표

| 문제 | 정답 |
|------|------|
| 192.168.1.0/24 FLSM 4분할, 4번째망 4번째 IP | **③ 192.168.1.196** |
| 200.1.1.0/24 FLSM 10분할, 10번째 브로드캐스트 | **200.1.1.159** |
| net 192.168.100.128, /26 마지막 사용 IP | **192.168.100.190** |
| 192.168.3.157, /26 브로드캐스트 | **192.168.3.191** |
| 192.168.3.65/26 브로드캐스트 | **192.168.3.127** |
| 100.168.3.65/25 브로드캐스트 | **100.168.3.127** |
| 192.168.5.0 /25, 2번째망 최소 IP | **192.168.5.129** |
| 100.100.100.0/24, 6서브넷·20호스트↑ CIDR | **/27** |
| 10.0.0.0 mask 255.240.0.0 유효 서브넷ID | **①, ④** (10.16.0.0, 10.48.0.0) |
| 192.168.0.1 vs .65 다른 서브넷 마스크 | **③ 255.255.255.192** |
| 172.16.0.0/20, 172.16.7.50 첫 호스트 | **172.16.0.1** |

> 172.16.7.50 /20: 3옥텟 블록=16 → 0~15 구간 → 네트워크 172.16.0.0, 첫 호스트 172.16.0.1.

# 2. 주기억장치 관련 계산식

## 2-1. 배치 전략 (First / Best / Worst Fit)

### 방법

```
First Fit: 위에서부터 "처음 들어가는" 공백
Best Fit:  들어갈 수 있는 공백 중 "가장 작은" 것 (내부 단편화 최소)
Worst Fit: 들어갈 수 있는 공백 중 "가장 큰" 것
```

### 대표 예제 — First Fit, 10K 작업
```
공백: B(5K)❌ C(15K)✅ ← 위에서 처음 맞는 15K 공백 = C
```

### 정답표

| 문제 | 정답 |
|------|------|
| First Fit 10K (11,5,15,30,12,25 공백) | **C (15K)** |
| Worst Fit 10K (8,12,10,20,16 중 FREE) | **NO.5 (16K)** — 가장 큰 공백 |

## 2-2. 단편화 (Fragmentation)

### 방법
```
내부 단편화 = 분할 크기 − 작업 크기 (작업이 들어간 경우, 남는 공간)
외부 단편화 = 작업이 못 들어간 분할의 크기 합 (작업 > 분할)
```

### 대표 예제 — External Fragmentation 합
```
A: 20K분할 < ... 작업10K 들어감 → 내부
B: 50K분할 < 작업60K → 못 들어감 → 외부 50K
C: 120K < 160K → 외부 120K
D: 200K > 100K → 들어감(내부)
E: 300K > 150K → 들어감(내부)
외부 단편화 합 = 50 + 120 = 170K
```
→ **170K**

## 2-3. 페이징 / 세그먼테이션

### 방법
```
페이징: 페이지 번호 = 논리주소 ÷ 페이지크기,  변위(offset) = 논리주소 mod 페이지크기
비트 수: 오프셋 비트 = log₂(페이지크기),  페이지번호 비트 = 주소비트 − 오프셋비트
세그먼트: 물리주소 = 시작주소 + 변위 (변위 < 길이 여야 유효)
```

### 대표 예제
- **논리주소 1224, 페이지 512B**: 1224÷512 = 2…200 → 페이지 **2**, 변위 **200**
- **페이지 2048B(=2¹¹), 32비트 주소**: 오프셋 **11비트**, 페이지번호 **21비트**
- **세그먼트(2,176)**: 시작222 + 176 = **398** (176 < 길이198 → 유효)

### 정답표

| 문제 | 정답 |
|------|------|
| First Fit 10K | C |
| Worst Fit 10K | NO.5 |
| External Fragmentation 합 | 170K |
| 논리주소 1224 (페이지512) | 페이지 2, 변위 200 |
| 페이지2048·32비트 | 페이지번호 21비트, 오프셋 11비트 |
| 세그먼트 (2,176) 물리주소 | 398 |

# 3. 페이지 교체 알고리즘

## 방법

```
FIFO: 가장 먼저 들어온 페이지 교체 (큐)
LRU:  가장 오래 참조 안 된 페이지 교체 (최근 사용 기준)
LFU:  참조 횟수가 가장 적은 페이지 교체 (동률이면 보통 오래된 것)
페이지 부재(Page Fault) = 프레임에 없어서 새로 적재한 횟수
페이지 교체 = 부재 중 "프레임이 꽉 차서 밀어낸" 횟수 (= 부재 − 최초 적재)
```

## 대표 예제 — FIFO, 참조 1,2,3,1,2,4,1,2,5 (프레임 3)

| 참조 | 프레임 상태 | 부재? |
|------|-----------|-------|
| 1 | [1] | ✅ |
| 2 | [1,2] | ✅ |
| 3 | [1,2,3] | ✅ |
| 1 | [1,2,3] | hit |
| 2 | [1,2,3] | hit |
| 4 | [2,3,4] (1축출) | ✅ |
| 1 | [3,4,1] (2축출) | ✅ |
| 2 | [4,1,2] (3축출) | ✅ |
| 5 | [1,2,5] (4축출) | ✅ |

→ 페이지 부재 **7회**

## 정답표

| 문제 | 정답 |
|------|------|
| FIFO 참조[1,2,1,0,4,1,3]/3 최종 | **4, 1, 3** |
| FIFO 참조[1,2,3,1,2,4,1,2,5]/3 부재 | **7번** |
| FIFO 참조[0,1,7,2,3,2,7,1,0,3]/4 | **부재 6회, 교체 2회** |
| LRU 참조[1,2,1,0,4,1,3]/3 최종 | **1, 4, 3** (잔여 {1,3,4}) |
| LRU 참조[1,2,3,1,2,4,1,2,5]/4 부재 | **5번** |
| LRU 참조 B,C,B,A,D /3 최종 | **B, D, A** |
| LRU (화살표 시점) 최종 | **7, 3, 1** |
| LFU 참조[2,3,1,2,1,2,4,2,1,3,2]/3 부재 | **5번** |
| LFU 참조[2,3,1,3,1,2,4,5]/4 최종 | **2, 3, 1, 5** |

> ⚠️ LRU 최종 상태는 "잔여 페이지 집합"이 정답 — 슬롯 표기 순서는 교재마다 다를 수 있음.

# 4. 프로세스 스케줄링

## 핵심 공식

```
반환시간(Turnaround) = 완료시간 − 도착시간  (= 대기시간 + 실행시간)
대기시간(Waiting)   = 반환시간 − 실행시간   (= 시작시간 − 도착시간)
평균 = 합 ÷ 프로세스 수
```

| 기법 | 특징 |
|------|------|
| FCFS | 도착 순서대로 (비선점) |
| SJF | 실행시간 짧은 것 먼저 (비선점) |
| SRT | 잔여시간 짧은 것 먼저 (선점) |
| HRN | 우선순위 = (대기+서비스)/서비스 — 높은 값 먼저 |
| RR | 타임퀀텀만큼 돌고 큐 뒤로 (선점) |

## 대표 예제 — SJF 평균 대기 (P=24,9,15,10, 동시 도착)
```
짧은 순: 9 → 10 → 15 → 24
대기: 0, 9, 19, 34 → 평균 (0+9+19+34)/4 = 62/4 = 15.5
```

## 대표 예제 — HRN 우선순위
```
우선순위 = (대기시간 + 서비스시간) / 서비스시간
A(8,2): (8+2)/2 = 5.0   ← 가장 높음
B(10,6): 16/6 ≈ 2.67
C(15,7): 22/7 ≈ 3.14
D(20,8): 28/8 = 3.5
→ A
```

# 5. 디스크 스케줄링

## 방법

```
FCFS: 요청 순서대로
SSTF: 현재 헤드에서 "가장 가까운" 트랙 먼저 (기아 가능)
SCAN(엘리베이터): 한 방향 끝까지 갔다가 반대로 (끝=0 또는 199까지 감)
C-SCAN: 한 방향 끝까지 → 반대 끝으로 "점프" → 다시 같은 방향 (원형)
LOOK: SCAN인데 "요청이 있는 데까지만" (끝까지 안 감)
C-LOOK: C-SCAN인데 요청 있는 데까지만
총 이동거리 = |연속 트랙 위치 차이| 의 합
```

## 대표 예제 — FCFS 총 이동거리 (헤드50, 큐 10,40,55,35)
```
50→10→40→55→35
= |50-10| + |10-40| + |40-55| + |55-35|
= 40 + 30 + 15 + 20 = 105
```

## 대표 예제 — SSTF 총 이동거리 (헤드50, 큐 80,20,100,30,70,130,40)
```
가까운 것부터: 50→40→30→20→70→80→100→130
= 10+10+10+50+10+20+30 = 140
```

## 정답표

| 유형 | 문제 | 정답 |
|------|------|------|
| FCFS | 이동거리(헤드50, 10/40/55/35) | 105 |
| SSTF | 먼저 처리(헤드50→0방향, 큐…) | 51 |
| SSTF | 이동거리(헤드50, 80/20/100/30/70/130/40) | 140 |
| SCAN | 이동거리(헤드30→0, 7/46/15/38/3) | 76 |
| SCAN | 마지막 트랙(헤드50→40, 7/55/15/38/3) | 55 |
| C-SCAN | 순서(헤드35→47, …) | 47→86→139→158→175→199→0→13→22 |
| C-SCAN | 이동거리(헤드50→0, …) | 380 |
| C-SCAN | 방식 판별(서비스순서 …0,12,35) | C-SCAN |
| LOOK | 먼저 처리(헤드60→50, 70/80/100/90) | 70 |
| LOOK | 이동거리(헤드50→0, 10/40/55/35) | 85 |
| C-LOOK | 이동거리(헤드50→199방향, …) | 318 |

> SCAN 예제(헤드30, 0방향): 30→15→7→3→0→38→46 = 30 + 46 = 76 (0까지 갔다가 반대).

# 6. 기타 계산식

## 6-1. 크론 표현식 (crontab)

### 방법 — 5개 필드
```
분(0-59) 시(0-23) 일(1-31) 월(1-12) 요일(0-7, 0·7=일)
*/n = n마다,   a-b = 범위,   a,b = 나열
```

### 정답표

| 문제 | 정답 |
|------|------|
| 주기 실행 명령·데몬 | crontab, crond |
| `*/30 * * * * backup.sh` 의미 | backup.sh를 30분마다 실행 |
| 매주 월 10시 check.sh | `0 10 * * 1 /etc/check.sh` |
| 일~화 16:30 batch.sh | `30 16 * * 0-2 /etc/batch.sh` |
| 일~목 02:30·14:30 | `30 02,14 * * 0-4 /etc/batch.sh` |
| 1~12월 2개월마다 1일 01:10 | `10 1 1 1-12/2 * /etc/batch.sh` |
| 매주 금 18:50 | `50 18 * * 5 /etc/batch.sh` |

## 6-2. 퍼미션 (chmod · umask)

### 방법
```
r=4, w=2, x=1 → 소유자/그룹/기타 각각 합산 (8진수 3자리)
umask: 파일 기본 666 − umask,  디렉토리 기본 777 − umask
umask 값 = 기본권한 − 실제권한
```

### 정답표

| 문제 | 정답 |
|------|------|
| -rwxrw-r-x | chmod **765** test.txt |
| u:rwx, g/o:r-x | chmod **755** test.txt |
| -rwxr-xr-- | chmod **754** test.txt |
| umask 022 파일 생성 권한 | 666−022 = **rw-r--r--** (644) |
| umask 022 디렉토리 권한 | 777−022 = **rwxr-xr-x** (755) |
| 파일 644일 때 umask | 666−644 = **022** |
| a 유저를 test.sh 소유자로 | **chown a test.sh** |

## 6-3. LOC 기법 (비용 산정)

### 방법
```
예측 LOC = (낙관치 + 4×기대치 + 비관치) / 6   ← 3점 추정
개발 기간(월) = 총 LOC ÷ (인원 × 1인당 월 생산성)
노력(인월, M/M) = 총 LOC ÷ 1인당 월 생산성
```

### 대표 예제
```
총 36,000 LOC, 6명, 월 300 LOC/인
기간 = 36000 / (6 × 300) = 36000 / 1800 = 20개월
```

### 정답표

| 문제 | 정답 |
|------|------|
| LOC 예측 3항목 | 낙관치, 기대치(중간치), 비관치 |
| 36000/(6×300) | 20개월 |
| 80000/(8×500) | 20개월 |
| 50000/(10×200) | 25개월 |
| 50000/(5×200) | 50개월 |

## 6-4. 임계 경로 (CPM)

### 방법
```
임계 경로 = 시작→종료 경로 중 "소요 기간이 가장 긴" 경로 (= 최소 완료 시간)
여유시간(Slack) = 가장 늦은 착수일 − 가장 빠른 착수일 (임계경로 위 작업은 0)
```

### 대표 예제 — 작업 C (선행 A15, B10 / C10, D25, E15)
```
C의 가장 빠른 착수 = max(A끝15, B끝10) = 15
E는 C 다음(C끝25), D는 B 다음(B끝35)
전체 최소완료 = max(경로) → C의 늦은 착수도 15 → Slack 0
→ C: 빠른 착수 15, 늦은 착수 15, 여유 0
```

### 정답표

| 문제 | 정답 |
|------|------|
| CPM 임계경로 소요기일(01) | 14일 |
| 작업 C 빠른/늦은 착수·여유 | 15일, 15일, 0일 |
| CPM 최소 완료 시간(03) | 50일 |

## 📌 시험 직전 핵심 암기

- **서브넷**: 블록크기 = 256−마스크옥텟, 호스트 = 2^호스트비트 − 2, 브로드캐스트 = 다음블록−1
- **배치**: First(처음), Best(가장 작은 공백), Worst(가장 큰 공백)
- **페이지 부재 vs 교체**: 교체 = 부재 − 프레임수(최초 적재분)
- **스케줄링 공식**: 반환 = 완료−도착, 대기 = 반환−실행
- **HRN 우선순위** = (대기+서비스)/서비스 (클수록 우선)
- **디스크**: SCAN=끝까지, LOOK=요청까지, C-*=한 방향 원형
- **umask**: 파일 666−umask, 디렉토리 777−umask
- **LOC 기간** = 총LOC ÷ (인원 × 월생산성)
- **CPM 임계경로** = 가장 긴 경로 = 최소 완료시간, 여유 = 늦은착수−빠른착수
