## Q01. 세션 하이재킹 (Session Hijacking)

### 지문 키워드
> **"클라이언트와 서버 간에 인증된 통신이 이루어지고 있을 때"**
> **"공격자가 중간에서 인증에 필요한 특정 정보를 가로채어 사용자의 권한을 탈취"**
> "네트워크 패킷을 스니핑하여 인증 정보를 탈취하거나, 웹 브라우저에 저장된 인증용 쿠키를 탈취"

### 왜 이게 답인가?

| 지문 키워드 | 연결 포인트 | 정답 |
|------------|------------|------|
| **인증된 통신 중** 공격 | 이미 성립된 세션을 탈취 | 세션 하이재킹 |
| **쿠키/패킷 스니핑**으로 권한 획득 | 세션 토큰을 훔쳐 정상 사용자 행세 | 세션 하이재킹 |

### 메커니즘

```
[정상 통신]
클라이언트 ─── 인증 완료, 세션 생성 ───▶ 서버
              (세션 ID = ABC123)

[세션 하이재킹]
클라이언트       공격자           서버
```

**유사 공격 비교표**

## Q02. 개체 무결성 / 참조 무결성 / 도메인 무결성

### 지문 키워드
> **① "기본키(Primary Key)를 구성하는 모든 속성은 절대로 NULL 값이나 중복된 값을 가질 수 없다"**
> **② "외래키(Foreign Key)의 값은 반드시 참조하고 있는 테이블의 기본키 값으로 존재하거나 NULL이어야 한다"**
> **③ "특정 속성(컬럼)에 대해 미리 정의된 형식과 범위 내에서만 데이터가 입력되도록 제한"**

### 왜 이게 답인가?

| 지문 핵심 키워드 | 연결 포인트 | 정답 |
|----------------|------------|------|
| **기본키, NULL 불가, 중복 불가** | PK의 속성 규칙 → 개체(엔티티) 식별 보장 | **① 개체 무결성** |
| **외래키, 참조 테이블의 기본키로 존재** | FK → PK 참조 관계 보장 | **② 참조 무결성** |
| **미리 정의된 형식과 범위** | 속성값의 타입/범위 제한 | **③ 도메인 무결성** |

### 메커니즘

```
[개체 무결성]          [참조 무결성]           [도메인 무결성]
학생 테이블            수강 테이블              학생 테이블
┌────────┐           ┌──────────┐            ┌──────────────┐
│ 학번(PK)│           │ 학번(FK) │            │ 나이: 1~150  │
│≠NULL   │←──참조──│≠NULL 또는│            │ 성별: M/F만  │
│≠중복   │           │ 학생.학번 │            │ 타입: INT    │
└────────┘           └──────────┘            └──────────────┘
```

**무결성 제약 조건 전체 비교**

| 종류 | 대상 | 규칙 |
|------|------|------|
| 개체 무결성 | PK | NULL 불가, 중복 불가 |
| 참조 무결성 | FK | 참조 테이블 PK에 존재하거나 NULL |
| 도메인 무결성 | 속성값 | 미리 정의된 타입/범위/형식 |
| 고유(유일) 무결성 | 특정 컬럼 | 중복 불가 (PK 아닌 컬럼) |
| NULL 무결성 | 컬럼 | NOT NULL 제약 |

> **암기법**: `개참도` — 개체→참조→도메인 순으로 기억

### ✅ 정답
① 개체 무결성  
② 참조 무결성  
③ 도메인 무결성

## Q03. CRC (Cyclic Redundancy Check)

### 지문 키워드
> **"송신 측은 전송하려는 데이터에 특정 다항식을 적용하여 얻은 나머지 값을 데이터와 함께 전송"**
> **"수신 측에서는 동일한 다항식으로 나누어 나머지를 비교하여 데이터의 손상 여부를 판단"**
> "이더넷과 같은 네트워크 통신 프로토콜에서 널리 사용"

### 왜 이게 답인가?

| 지문 키워드 | 연결 포인트 | 정답 |
|------------|------------|------|
| **다항식 적용 → 나머지 값** 함께 전송 | 다항식(Polynomial) 나눗셈 기반 오류 검출 | CRC |
| **수신 측이 나머지 비교** | 체크섬이 아닌 나머지 비교 방식 | CRC |

### 메커니즘

```
[송신 측]
데이터(M) ──÷ 생성다항식(G)──▶ 나머지(R) 계산
전송 데이터 = M + R

[수신 측]
수신 데이터 ──÷ 생성다항식(G)──▶ 나머지 = 0이면 정상
                                         ≠ 0이면 오류 발생
```

**오류 검출/수정 방법 비교표**

| 방법 | 원리 | 특징 |
|------|------|------|
| **CRC** | 다항식 나눗셈 나머지 비교 | 오류 검출만, 이더넷 사용 |
| 패리티 비트 | 비트 홀수/짝수 개수 확인 | 단순, 1비트 오류만 검출 |
| 해밍 코드 | 패리티 + 위치 정보 | 오류 검출 + 수정 가능 |
| 체크섬 | 데이터 합산 값 비교 | 간단하지만 정확도 낮음 |

### ✅ 정답
CRC (Cyclic Redundancy Check, 순환 중복 검사)

## Q05. Java — try-catch-finally 예외처리

### 문제
```java
class ExceptionTest {
    public static void main(String[] args) {
        int a = 5, b = 0;
        try {
            System.out.println(a / b);
        } catch (ArithmeticException e) {
            System.out.printf("출력1");
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.printf("출력2");
        } catch (NumberFormatException e) {
            System.out.printf("출력3");
        } catch (Exception e) {
            System.out.printf("출력4");
        } finally {
            System.out.printf("출력5");
        }
    }
}
```

### 핵심 개념 — 예외 처리 흐름

**Java 예외 처리 규칙:**
1. try 블록에서 예외 발생 → 해당 예외 타입의 catch로 점프
2. 일치하는 catch 실행 후 나머지 catch는 건너뜀
3. **finally는 예외 발생 여부와 무관하게 항상 실행**

> ⚠️ **핵심 함정**: `5 / 0` → `ArithmeticException` 발생 (정수 나눗셈만 해당, 실수 나눗셈은 Infinity)

**Java 주요 예외 타입**

| 예외 | 발생 상황 |
|------|----------|
| **ArithmeticException** | 정수를 0으로 나눌 때 |
| ArrayIndexOutOfBoundsException | 배열 인덱스 초과 |
| NumberFormatException | 숫자로 변환 불가한 문자열 |
| NullPointerException | null 참조 접근 |

### 단계별 트레이싱

| 단계 | 코드 | 결과 |
|------|------|------|
| 1 | `a=5, b=0` | 정상 |
| 2 | `System.out.println(a / b)` | **5/0 → ArithmeticException 발생** |
| 3 | `catch (ArithmeticException e)` | ✅ 매칭 → `printf("출력1")` 실행 |
| 4 | 나머지 catch 블록 | 건너뜀 |
| 5 | `finally { printf("출력5") }` | **항상 실행** |

### ✅ 정답
```
출력1출력5
```
(println이 아닌 printf라 줄바꿈 없이 붙어서 출력)

## Q06. ARP / RARP

### 지문 키워드
> **① "IP 주소를 가지고 있는 호스트가 수신 호스트의 IP 주소는 알고 있지만, 해당 IP 주소에 대응하는 물리적인 MAC 주소를 모르는 경우 이를 얻기 위해 사용하는 프로토콜"**
> **② "네트워크에 연결된 호스트가 자신의 MAC 주소는 알고 있지만, IP 주소를 모르는 경우 서버에 질의하여 자신의 IP 주소를 얻기 위해 사용하는 프로토콜"**

### 왜 이게 답인가?

| 지문 핵심 키워드 | 연결 포인트 | 정답 |
|----------------|------------|------|
| **IP 알고 있음 → MAC 모름** → MAC 획득 | Address Resolution (주소 해석) | **① ARP** |
| **MAC 알고 있음 → IP 모름** → IP 획득 | Reverse(역방향) Address Resolution | **② RARP** |

### 메커니즘

```
[ARP: IP → MAC]
호스트A                       네트워크                  호스트B
"192.168.1.2의               브로드캐스트               IP=192.168.1.2
 MAC 주소가 뭐야?"  ────────────────────────────────▶  "내 MAC은 AA:BB:CC"
                  ◀────────────────────────────────
                              유니캐스트 응답

[RARP: MAC → IP]
디스크 없는 클라이언트         RARP 서버
"내 MAC은 DD:EE:FF           "DD:EE:FF의 IP는
 IP 주소가 뭐야?"  ─────────▶  192.168.1.5야"
                  ◀─────────
```

**주소 관련 프로토콜 비교**

| 프로토콜 | 방향 | 역할 |
|----------|------|------|
| **ARP** | IP → MAC | IP로 MAC 알아내기 |
| **RARP** | MAC → IP | MAC으로 IP 알아내기 |
| DNS | 도메인 → IP | 도메인으로 IP 알아내기 |
| DHCP | 자동 | MAC → IP 동적 할당 |

### ✅ 정답
① ARP  
② RARP

## Q07. SQL — CROSS JOIN + WHERE 필터 실행 결과

### 지문 키워드
```sql
SELECT emp.name, sal.incentives
FROM emp, sal
WHERE emp.id = sal.id AND sal.incentives >= 500;
```

**테이블 데이터**

| emp.id | emp.name | | sal.id | sal.incentives |
|--------|----------|-|--------|----------------|
| 1001 | 김철수 | | 1002 | 300 |
| 1002 | 홍길동 | | 1004 | 300 |
| 1004 | 강감찬 | | 1008 | 1000 |
| 1008 | 이순신 | | 1009 | 500 |

### 단계별 트레이싱

**SQL 실행 순서: FROM → WHERE → SELECT**

```
1단계: FROM emp, sal
  → emp×sal 카르테시안 곱 (4×4 = 16행)

2단계: WHERE emp.id = sal.id (EQUI JOIN 조건)
  → 매칭 결과:
     emp.id=1002, sal.id=1002 → 홍길동, 300
     emp.id=1004, sal.id=1004 → 강감찬, 300
     emp.id=1008, sal.id=1008 → 이순신, 1000
     (1001은 sal에 없음, 1009는 emp에 없음)

3단계: AND sal.incentives >= 500
  → 300 < 500 → 홍길동 제외
  → 300 < 500 → 강감찬 제외
  → 1000 >= 500 → 이순신 ✅

4단계: SELECT emp.name, sal.incentives
```

> ⚠️ **핵심 함정**: `sal.id=1009(500)`은 emp에 매칭 id가 없으므로 JOIN 결과에서 제외됨

### ✅ 정답
| name | incentives |
|------|-----------|
| 이순신 | 1000 |

## Q08. 관계형 DB 용어 매칭

### 지문 내용
1. 하나의 테이블에 존재하는 속성(Attribute)의 개수  
2. 하나의 테이블에 존재하는 튜플(Tuple)의 개수  
3. 다른 테이블의 기본 키(Primary Key)를 참조하는 키  
4. 하나의 속성이 가질 수 있는 값의 범위  

**보기**: ㄱ.degree, ㄴ.cardinality, ㄷ.primary, ㄹ.foreign key, ㅁ.attribute, ㅂ.domain, ㅅ.candidate key, ㅇ.tuple

### 왜 이게 답인가?

| 설명 | 연결 포인트 | 정답 |
|------|------------|------|
| **속성의 개수** | 차수(degree) = 열(컬럼) 수 | **ㄱ. degree** |
| **튜플의 개수** | 카디널리티 = 행(레코드) 수 | **ㄴ. cardinality** |
| **다른 테이블 PK 참조** | Foreign(외래) Key | **ㄹ. foreign key** |
| **속성값의 범위** | 도메인 = 허용 가능한 값의 집합 | **ㅂ. domain** |

### 메커니즘

```
학생 테이블
┌─────────┬──────────┬──────┬────────┐
│ 학번(PK) │   이름   │ 나이 │ 학과코드(FK) │
```

## Q11. C언어 — malloc + 2D배열 재배치

### 문제
```c

void func(int **res, int cols, int rows){
    for (int i = 0; i < (cols*rows); i++){
        int r = (i + 1) % (rows * cols);
        res[r / cols][r % cols] = A[i];
    }
}

int main(){
    int **res;
    int cols = 3, rows = 3, sum = 0;
    res = (int **)malloc(sizeof(int*) * rows);
    for (int i = 0; i < rows; i++)
        res[i] = (int *)malloc(sizeof(int) * cols);
    func(res, cols, rows);
    for(int i = 0; i < rows * cols; i++)

}
```

```
  index 2 → [0][2]
  index 3 → [1][0]
```

### 단계별 트레이싱

**func 실행 — A배열을 2D res에 재배치**

| i | A[i] | r=(i+1)%9 | res[r/3][r%3] | 결과 |

| 0 | 8 | 1 | res[0][1] = 8 | |

## Q13. Java — static + 동적 바인딩

### 문제
```java
class Parent {
    static int total = 0;
    int v = 1;
    public Parent() {
        total += (++v);
        showValue();
    }
    public void showValue() {
        total += total;
    }
}
class Child extends Parent {
    int v = 10;
    public Child() {
        v += 2;
        total += (v++);
        showValue();
    }
    @Override
    public void showValue() {
        total += total * 2;
    }
}
class Main {
    public static void main(String[] args) {
        new Child();
        System.out.println(Parent.total);
    }
}
```

### 핵심 개념 — static 변수 + 동적 바인딩

> ⚠️ **핵심 함정 1**: `static int total` → 클래스 공유, Parent/Child 모두 **같은 total**을 씀

> ⚠️ **핵심 함정 2**: **동적 바인딩** — Parent() 생성자 안에서 `showValue()`를 호출해도, 실제 생성 중인 객체가 Child이므로 **Child의 showValue()가 호출됨**

> ⚠️ **핵심 함정 3**: `int v` — Parent.v와 Child.v는 **별개의 필드**. `++v`는 Parent 생성자에서 실행되므로 **Parent.v**를 증가시킴

### 단계별 트레이싱

**new Child() 호출 시 순서: Parent() 먼저 → 그 다음 Child()**

**[1] Parent() 실행**

| 단계 | 코드 | Parent.v | Child.v | total |
|------|------|----------|---------|-------|
| 시작 | - | 1 | 10 | 0 |
| `total += (++v)` | Parent.v: 1→2, total+=2 | **2** | 10 | **2** |
| `showValue()` | **동적바인딩 → Child.showValue()** | 2 | 10 | - |
| Child.showValue() | `total += total*2` → 2+4=6 | 2 | 10 | **6** |

**[2] Child() 실행**

| 단계 | 코드 | Parent.v | Child.v | total |
|------|------|----------|---------|-------|
| `v += 2` | Child.v: 10→12 | 2 | **12** | 6 |
| `total += (v++)` | total+=12=18, Child.v: 12→13 | 2 | **13** | **18** |
| `showValue()` | Child.showValue() 호출 | 2 | 13 | - |
| Child.showValue() | `total += total*2` → 18+36=54 | 2 | 13 | **54** |

**[3] System.out.println(Parent.total) → 54**

### ✅ 정답
```
54
```

## Q15. 구문(문장) 커버리지

### 지문 내용
```
[소스코드]                      [제어 흐름도]
int fnc(int a){
    int total = 0;              ① int total = 0
    while( a >= 0 ){            ② a >= 0 (조건)
        if( a % 2 == 0 )        ③ a % 2 == 0 (조건)
            total = total + a;  ④ total = total + a
        a = a - 2;              ⑤ a = a - 2
    }
    return total;               ⑥ return total
}
```

### 핵심 개념 — 구문 커버리지 (Statement Coverage)

**구문 커버리지**: 소스코드의 **모든 실행 문장**이 최소 1번 이상 실행되도록 테스트 케이스 설계

**다른 커버리지와 비교**

| 커버리지 | 조건 | 강도 |
|----------|------|------|
| 구문(Statement) | 모든 문장 1회 실행 | 낮음 |
| 분기(Branch/Decision) | 모든 분기(T/F) 1회 실행 | 중간 |
| 조건(Condition) | 각 조건식 T/F 1회 실행 | 높음 |
| 경로(Path) | 모든 경로 실행 | 가장 높음 |

### 흐름도 빈칸 정답

| 번호 | 소스코드 |
|------|----------|
| ① | `int total = 0;` |
| ② | `a >= 0` |
| ③ | `a % 2 == 0` |
| ④ | `total = total + a;` |
| ⑤ | `a = a - 2;` |
| ⑥ | `return total;` |

### 구문 커버리지 경로 (모든 문장 실행)

**모든 구문(①~⑥)을 최소 1회 실행하려면 ④도 반드시 거쳐야 함**

```
① → ② → ③ → ④ → ⑤ → ② → ⑥
         ↑ (a%2==0 True 경로로 ④ 실행 필요)
```

> **설명**: while 루프에서 짝수 a가 있으면 ③→④→⑤ 경로, 다음 반복에서 a<0으로 ②→⑥

### ✅ 정답
① `total = 0;`  
② `a >= 0`  
③ `a % 2 == 0`  
④ `total = total + a;`  
⑤ `a = a - 2;`  
⑥ `return total;`  
구문 커버리지 경로: **① → ② → ③ → ④ → ⑤ → ② → ⑥**

### 핵심 개념 — 분할정복 패턴

**동작 원리**: 배열을 반으로 나누고, 중간값 + max(왼쪽 재귀, 오른쪽 재귀)

> ⚠️ **종료 조건**: `st >= end` → return 0 (범위 1개 이하일 때)

### 단계별 트레이싱 (재귀 트리)

```
func(a, 0, 4)
├── mid=2, a[2]=8

│   ├── mid=1, a[1]=5
│   ├── func(a, 0, 1)
│   │   ├── mid=0, a[0]=3
│   │   ├── func(a, 0, 0) = 0  [st>=end]

│   └── = 12 + max(0, 0) = 12
└── = 8 + max(8, 12) = 8 + 12 = 20
```

## Q17. Python — 트리 노드 홀수 레벨 합산

### 문제
```python
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []

def tree(li):
    nodes = [Node(i) for i in li]
    for i in range(1, len(li)):
        nodes[(i - 1) // 2].children.append(nodes[i])
    return nodes[0]

def s(node, level=0):
    if not node:
        return 0
    total = node.value if level % 2 else 0
    for child in node.children:
        total += s(child, level + 1)
    return total

li = [3, 5, 8, 12, 15, 18, 20]
root = tree(li)
print(s(root))
```

### 핵심 개념 — 이진 트리 + 레벨별 합산

**부모 인덱스 공식**: `nodes[(i-1)//2]` → i번 노드의 부모

> ⚠️ **핵심**: `level % 2` → 홀수 레벨(1,3,5...)만 합산, 짝수 레벨(0,2,4...)은 0

### 단계별 트레이싱

**트리 구조 생성**
```
i=1: parent=(0)//2=0, nodes[0].children += nodes[1]
i=2: parent=(1)//2=0, nodes[0].children += nodes[2]
i=3: parent=(2)//2=1, nodes[1].children += nodes[3]
i=4: parent=(3)//2=1, nodes[1].children += nodes[4]
i=5: parent=(4)//2=2, nodes[2].children += nodes[5]
i=6: parent=(5)//2=2, nodes[2].children += nodes[6]

트리:
         3  (level 0, 값=3)
        / \
       5   8  (level 1, 값=5,8)
      / \ / \
    12 15 18 20  (level 2, 값들)
```

**s() 실행 — 레벨별 합산**

| 노드 | level | level%2 | total 기여 |
|------|-------|---------|-----------|
| 3 | 0 | 0 (짝수) | **0** |
| 5 | 1 | 1 (홀수) | **5** |
| 8 | 1 | 1 (홀수) | **8** |
| 12 | 2 | 0 (짝수) | 0 |
| 15 | 2 | 0 (짝수) | 0 |
| 18 | 2 | 0 (짝수) | 0 |
| 20 | 2 | 0 (짝수) | 0 |

```
s(root) = 0 + 5 + 8 = 13
```


## Q18. C언어 — 연결리스트 reconnect

### 문제
```c
typedef struct Data {
    int value;
    struct Data *next;
} Data;

Data* insert(Data* head, int value) { /* prepend */ }

Data* reconnect(Data* head, int value) {
    if (head == NULL || head->value == value) return head;
    Data *prev = NULL, *curr = head;
    while (curr != NULL && curr->value != value) {
        prev = curr;
        curr = curr->next;
    }
    if (curr != NULL && prev != NULL) {
        prev->next = curr->next;  // 현재 노드를 체인에서 제거
        curr->next = head;        // 현재 노드가 기존 head를 가리킴
        head = curr;              // head를 현재 노드로 변경
    }
    return head;
}

int main() {
    Data *head = NULL;
    for (int i = 1; i <= 5; i++)
        head = insert(head, i);
    head = reconnect(head, 3);
    for (curr = head; curr != NULL; curr = curr->next)
        printf("%d", curr->value);
}
```

### 핵심 개념 — 연결리스트 구조 변경

**insert는 앞에 삽입 (prepend)** → i=1~5 삽입 결과: `5→4→3→2→1→NULL`

> ⚠️ **핵심**: reconnect는 value=3인 노드를 찾아서 **맨 앞으로 이동**시키는 함수

### 단계별 트레이싱

**초기 연결리스트**: `5 → 4 → 3 → 2 → 1 → NULL`

**reconnect(head, 3) 실행**

```
탐색:
prev=NULL, curr=5 (5≠3 → 계속)
prev=5,    curr=4 (4≠3 → 계속)
prev=4,    curr=3 (3=3 → 탈출)

curr(3)≠NULL && prev(4)≠NULL → 구조 변경:

1. prev->next = curr->next
   4->next = 2    (4가 2를 가리킴)
   
2. curr->next = head
   3->next = 5    (3이 기존 head인 5를 가리킴)
   
3. head = curr
   head = 3
```

**변경 후 연결리스트**:
```
3 → 5 → 4 → 2 → 1 → NULL
```

**출력**: `35421`

### ✅ 정답
```
35421
```

## Q19. C언어 — 비트 AND 연산 (16진수)

### 문제
```c
typedef struct {
    char name[10];
    unsigned char score[3];
} Node;

int f(unsigned char score) {
    return score & 0xA5;
}

int main() {
    Node *nodes = malloc(2 * sizeof(Node));
    nodes[0] = (Node){"Kim", {0xF0, 0xF5, 0xDB}};
    nodes[1] = (Node){"Lee", {0xED, 0xD3, 0xF2}};
    int result = 0;
    for (int i = 0; i < 2; i++)
        for (int j = 0; j < 3; j++)
            result += f(nodes[i].score[j]);
    printf("%d\n", result);
}
```

### 핵심 개념 — 비트 AND 연산

**0xA5 = 1010 0101**

> ⚠️ **핵심**: AND 연산은 마스크처럼 동작. **0xA5의 비트가 0인 자리는 결과가 항상 0**

### 단계별 트레이싱

**각 score에 대해 score & 0xA5 계산**

| 이름 | score (16진) | score (2진) | &0xA5 (2진) | 결과 (10진) |
|------|-------------|-------------|-------------|------------|
| Kim | 0xF0 | 1111 0000 | 1010 0000 | **0xA0 = 160** |
| Kim | 0xF5 | 1111 0101 | 1010 0101 | **0xA5 = 165** |
| Kim | 0xDB | 1101 1011 | 1000 0001 | **0x81 = 129** |
| Lee | 0xED | 1110 1101 | 1010 0101 | **0xA5 = 165** |
| Lee | 0xD3 | 1101 0011 | 1000 0001 | **0x81 = 129** |
| Lee | 0xF2 | 1111 0010 | 1010 0000 | **0xA0 = 160** |

**AND 계산 상세 예시 (0xF0 & 0xA5)**:
```
  1111 0000  (0xF0)
& 1010 0101  (0xA5)
-----------
  1010 0000  (0xA0 = 160)
```

**합산**:
```
160 + 165 + 129 + 165 + 129 + 160 = 908
```


## Q20. Java — 메서드 오버로딩 + 재귀

### 문제
```java
class Main {
    static int add(String s) {
        int t = Integer.valueOf(s);
        if (t <= 1) return t;
        return add(t - 1) + add(t - 3);  // ← add(int i) 호출
    }
    static int add(int i) {
        if (i <= 1) return i;
        return add(i - 1) + add(i - 2);
    }
    public static void main(String[] args) {
        System.out.println(add("5"));
    }
}
```

### 핵심 개념 — 메서드 오버로딩 + 피보나치 변형 재귀

> ⚠️ **핵심 함정 1**: `add("5")` → `add(String s)` 호출  
> `t=5`, `return add(4) + add(2)` → 이때 **add(int i)** 호출됨!

> ⚠️ **핵심 함정 2**: `add(int i)` 는 피보나치 변형 — `add(i-1) + add(i-2)` (파라미터 타입이 다른 별개의 함수)

### 단계별 트레이싱

**add(int i) 먼저 계산 (피보나치 변형)**

| 호출 | 조건 | 계산 | 결과 |
|------|------|------|------|
| add(0) | 0<=1 | return 0 | **0** |
| add(1) | 1<=1 | return 1 | **1** |
| add(2) | - | add(1)+add(0) | **1+0=1** |
| add(3) | - | add(2)+add(1) | **1+1=2** |
| add(4) | - | add(3)+add(2) | **2+1=3** |

**add(String "5") 계산**
```
t = 5
return add(4) + add(2)  ← add(int i) 호출
     = 3     + 1
     = 4
```

**재귀 트리**
```
add("5")
├── add(4) [int] = 3
│   ├── add(3) = 2
│   │   ├── add(2) = 1
│   │   └── add(1) = 1
│   └── add(2) = 1
└── add(2) [int] = 1
```

### ✅ 정답
```
4
```
