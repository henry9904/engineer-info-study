## Q01. 응집도 순서

### 지문 키워드
> **응집도가 높은 순에서 낮은 순 / 보기: ① 기능 ② 교환 ③ 우연 ④ 시간**

### 메커니즘 — 응집도 7단계 (높음 → 낮음)

| 순위 | 종류 | 핵심 |
|------|------|------|
| 1 | **기능적** | 단 하나의 기능만 수행 |
| 2 | 순차적 | 앞 출력이 뒤 입력 |
| 3 | **통신적(교환)** | 같은 데이터 참조/변경 |
| 4 | 절차적 | 순서대로 실행 |
| 5 | **시간적** | 같은 시간대에 실행 |
| 6 | 논리적 | 유사 기능 묶음 |
| 7 | **우연적** | 아무 관계 없음 |

> **암기법**: `기순통절시논우` — 기능이 제일 좋고 우연이 제일 나쁨

### ✅ 정답
```
①②④③ (기능 → 교환 → 시간 → 우연)
```

## Q02. OSPF

### 지문 키워드
> **"링크 상태 라우팅 프로토콜"** / **"최단 경로 우선"** / **"대규모 네트워크에서 효율적"**

### OSPF vs RIP

| | OSPF | RIP |
|--|------|-----|
| 방식 | **링크 상태** | 거리 벡터 |
| 홉 제한 | **없음** | 최대 15홉 |
| 거리 기준 | **대역폭** | 홉 카운트 |
| 업데이트 | **변화 시만** | 30초 주기 |

### ✅ 정답
```
OSPF (Open Shortest Path First)
```

## Q09. LRU / LFU 페이지 교체

### 참조 순서: 1 2 3 1 2 4 1 2 5 7 / 프레임 3개

### LRU 트레이싱 (가장 오래 전에 사용된 것 교체)

| 참조 | 프레임 | Miss? |
|------|--------|-------|
| 1 | [1] | ✅ |
| 2 | [1,2] | ✅ |
| 3 | [1,2,3] | ✅ |
| 1 | [2,3,1] | Hit |
| 2 | [3,1,2] | Hit |
| 4 | [1,2,4] ← 3 교체 | ✅ |
| 1 | [2,4,1] | Hit |
| 2 | [4,1,2] | Hit |
| 5 | [1,2,5] ← 4 교체 | ✅ |
| 7 | [2,5,7] ← 1 교체 | ✅ |

→ **LRU: 6번**

### LFU 트레이싱 (가장 적게 사용된 것 교체)

| 참조 | 프레임(페이지:빈도) | Miss? |
|------|---------------------|-------|
| 1 | [1:1] | ✅ |
| 2 | [1:1,2:1] | ✅ |
| 3 | [1:1,2:1,3:1] | ✅ |
| 1 | [1:2,2:1,3:1] | Hit |
| 2 | [1:2,2:2,3:1] | Hit |
| 4 | [1:2,2:2,4:1] ← 3 교체 | ✅ |
| 1 | [1:3,2:2,4:1] | Hit |
| 2 | [1:3,2:3,4:1] | Hit |
| 5 | [1:3,2:3,5:1] ← 4 교체 | ✅ |
| 7 | [1:3,2:3,7:1] ← 5 교체 | ✅ |

→ **LFU: 6번**

### ✅ 정답
```
LRU: 6
LFU: 6
```

## Q10. 제3정규형 (3NF)

### 개념 — 정규화 단계

| | 조건 |
|--|------|
| 1NF | 원자값 |
| 2NF | 부분 함수 종속 제거 |
| **3NF** | **이행적 함수 종속 제거** |

### 이행적 종속
```
(학생아이디, 강좌명) → 담당강사
강좌명 → 담당강사  ← 이행적 종속 존재!
→ 제3정규형 위반 구조
```

### ✅ 정답
```
제3정규형 (3NF)
```

## Q12. SQL — AND/OR 우선순위

```sql
WHERE EMPNO > 100 AND SAL >= 3000 OR EMP = 200
= (EMPNO > 100 AND SAL >= 3000) OR EMP = 200
→ 200번 행만 해당 (EMP=200) → COUNT(*) = 1
```

> **AND가 OR보다 우선순위 높음** — 시험 최빈출 함정!

### ✅ 정답
```
COUNT(*) = 1
```

## Q13. C언어 — 비트 시프트

```c
v1=0, v2=35, v3=29
if(0>35 ? 35 : 0) → if(0) → else
v3 = 29 << 2 = 29 × 4 = 116
v2+v3 = 35+116 = 151
```

> **`n << k` = n × 2^k**

### ✅ 정답
```
151
```

## 📌 이 회차 핵심 암기

- **응집도** `기순통절시논우` — 기능(최고) → 우연(최악)
- **OSPF** = 링크상태, 홉제한없음, 대규모 / **RIP** = 거리벡터, 15홉, 소규모
- **MC/DC** = 분기+조건 동시 만족 + 각 조건 독립적 영향
- **APT** = 장기간 지속적 표적 / **Rootkit** = 관리자 권한 탈취 + 은폐
- **AND > OR 우선순위** — SQL 조건절 함정
- **싱글톤**: 모든 인스턴스가 동일 객체 → count 공유
- **비트시프트**: `n << k` = n × 2^k
