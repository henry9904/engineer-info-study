## Q01. RARP

### 지문 키워드
> **"물리 주소(MAC)를 IP 주소로 변환하는 프로토콜"**

### 왜 이게 답인가?

| 지문 키워드 | 연결 포인트 | 정답 |
|------------|------------|------|
| **MAC → IP 변환** | Reverse ARP | RARP |

> ⚠️ **함정**: ARP는 IP→MAC, **RARP는 MAC→IP**(반대). 방향 주의!

### ✅ 정답
```
RARP (Reverse Address Resolution Protocol)
```

## Q16. SQL — AND / OR 우선순위

### 문제
```sql
SELECT COUNT(*) FROM TABLE01
WHERE EMPNO > 100 AND SAL >= 3000 OR EMPNO = 200
```
```
EMPNO  SAL
100    1000
200    3000
300    1500
```

### 핵심 개념 — AND가 OR보다 우선

> ⚠️ **핵심 함정**: `AND`가 `OR`보다 우선순위 높음 → `(EMPNO>100 AND SAL>=3000) OR EMPNO=200`.

### 단계별 트레이싱

| EMPNO | (>100 AND SAL>=3000) | OR EMPNO=200 | 포함? |
|-------|----------------------|--------------|-------|
| 100 | (F AND ...) = F | F | ✗ |
| 200 | (T AND T) = T | T | ✅ |
| 300 | (T AND F) = F | F(300≠200) | ✗ |

→ COUNT = **1**

### ✅ 정답
```
1
```

## Q20. C — 구조체 배열 포인터

### 문제
```c
struct insa { char name[10]; int age; }
    a[] = { "Kim",28, "Lee",38, "Park",41, "Choi",30 };
struct insa *p;
p = a;       // p = &a[0] (Kim)
p++;         // p = &a[1] (Lee)
printf("%s ", p->name);
printf("%d", p->age);
```

### 핵심 개념 — 구조체 포인터 `p++`는 다음 구조체로

> `p++`는 `sizeof(struct insa)`만큼 이동 → a[0](Kim) → a[1](Lee).

### 단계별 트레이싱

```
a[0] = {Kim, 28}
a[1] = {Lee, 38}
p = a → p++ → &a[1]
p->name = "Lee", p->age = 38
→ Lee 38
```

## 📌 이 회차 핵심 암기

- **ARP(IP→MAC) vs RARP(MAC→IP)** — 방향 반대
- **DB 설계**: 개념(E-R)→논리(정규화)→물리(저장)
- **카디널리티=행, 차수=열**
- **EAI 유형**: P2P·Hub&Spoke·Message Bus·Hybrid
- **데이터 모델 3요소**: 구조·연산·제약조건
- **결합도 약→강**: 자·스·제·외·공·내 (내용=최악)
- **세션 하이재킹**: TCP 시퀀스 위조로 인증 세션 탈취
- **웹서비스**: SOAP(통신)·WSDL(명세)·UDDI(등록)
- **반정규화**: 성능 위해 의도적으로 중복 허용
