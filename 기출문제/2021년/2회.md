## Q04. 트랜잭션 원자성 (Atomicity)

### 지문 키워드
> 트랜잭션 ACID 중 **원자성** 설명

### 왜 이게 답인가?

> 원자성 = 트랜잭션의 연산은 **모두 반영(commit)되거나, 하나라도 실패하면 모두 취소(rollback)** 되어야 한다. "all or nothing".

### 메커니즘 — ACID

```
원자성(Atomicity): 모두 실행 or 모두 취소
일관성(Consistency): 실행 전후 DB 일관 상태 유지
격리성(Isolation): 동시 트랜잭션 간 간섭 없음
지속성(Durability): 완료된 결과는 영구 보존
```

### ✅ 정답
```
트랜잭션의 연산은 모두 반영되거나 전혀 반영되지 않아야 한다 (All or Nothing)
```

## Q17. Java — 상속 + super

### 문제
```java
class Class01 {
    int sum(int x, int y) { return x + y; }
}
class Class02 extends Class01 {
    int sum(int x, int y) { return x - y + super.sum(x, y); }
}
// main
Class01 c1 = new Class01();
Class02 c2 = new Class02();
System.out.println( c1.sum(3,2) + c2.sum(3,2) );
```

### 핵심 개념 — 오버라이딩 + super 호출

### 단계별 트레이싱

```
c1.sum(3,2) = 3 + 2 = 5
c2.sum(3,2) = (3-2) + super.sum(3,2) = 1 + (3+2) = 1 + 5 = 6
합계 = 5 + 6 = 11
```

### ✅ 정답
```
11
```

## Q18. C — 배열/포인터 표기

### 문제
```c
int array[3];
int s = 0;
*(array + 0) = 1;              // array[0] = 1
array[1] = *(array + 0) + 2;   // array[0]+2 = 3
array[2] = *array + 3;         // array[0]+3 = 4
for (i=0;i<3;i++) s += array[i];
printf("%d", s);
```

### 핵심 개념 — `*(array+n)` == `array[n]`, `*array` == `array[0]`

### 단계별 트레이싱

```
array[0] = 1
array[1] = 1 + 2 = 3
array[2] = 1 + 3 = 4
s = 1 + 3 + 4 = 8
```

### ✅ 정답
```
8
```

## Q19. Python — 시프트 연산 (>>)

### 문제
```python
a = 100
result = 0
for i in range(1, 3):       # i = 1, 2
    result = a >> i         # 매번 새로 대입 (누적 아님!)
    result = result + 1
print(result)
```

### 핵심 개념 — `>>` (오른쪽 시프트 = ÷2^n), result 재대입

> ⚠️ **함정**: `result = a >> i`가 매 반복마다 result를 **새로 덮어씀** → 마지막 i=2의 결과만 남음.

### 단계별 트레이싱

```
i=1: result = 100 >> 1 = 50 → +1 = 51
i=2: result = 100 >> 2 = 25 → +1 = 26   (이전 51 무시)
print(result) = 26
```

### ✅ 정답
```
26
```

## Q20. C — 거듭제곱 반복

### 문제
```c
int fp(int base, int exp) {
    int result = 1;
    for (int i=0; i<exp; i++) result *= base;
    return result;
}
// fp(2, 10)
```

### 핵심 개념 — 2를 10번 곱함 = 2^10

```
fp(2,10) = 2^10 = 1024
```

### ✅ 정답
```
1024
```

## 📌 이 회차 핵심 암기

- **Ad-hoc**: 인프라 없이 노드 자율 구성 / **UI**(매개체) vs **UX**(경험)
- **디자인 패턴 3분류**: 생성·구조·행위 (Observer/Command=행위)
- **ACID**: 원자성(all or nothing)·일관성·격리성·지속성
- **정규화**: 1NF(원자값)→2NF(부분종속 제거)→3NF(이행종속 제거)→BCNF
- **응집도 약→강**: 우·논·시·절·교·순·기 (높을수록 좋음)
- **커버리지 위계**: 구문<분기(결정)<조건<조건/결정<MC/DC
- **럼바우 OMT**: 객체·동적·기능 (객체 모델링이 선행)
- **static**: 객체 없이 호출 (Q16)
- **오버라이딩 + super.sum()** (Q17)
- **`>>` 시프트 + 재대입 함정** (Q19)
