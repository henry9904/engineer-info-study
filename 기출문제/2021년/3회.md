## Q15. SQL — CROSS JOIN + LIKE

### 문제
```sql
SELECT COUNT(*) AS CNT FROM T1 CROSS JOIN T2
WHERE T1.NAME LIKE T2.RULE
```
```
T1.NAME = {ALLEN, SCOTT, SMITH}
T2.RULE = {S%, %T%}
```

### 핵심 개념 — CROSS JOIN(모든 조합) + LIKE 패턴

```
S%   : S로 시작
%T%  : 중간에 T 포함
CROSS JOIN → T1×T2 모든 조합(3×2=6쌍)을 만들고 WHERE로 필터
```

### 단계별 트레이싱

| NAME | LIKE 'S%' | LIKE '%T%' | 매칭 수 |
|------|-----------|------------|---------|
| ALLEN | ✗ | ✗ (T없음) | 0 |
| SCOTT | ✅ (S시작) | ✅ (T포함) | 2 |
| SMITH | ✅ (S시작) | ✅ (T포함) | 2 |

→ COUNT = 0 + 2 + 2 = **4**

### ✅ 정답
```
4
```

## Q17. C — 구조체 포인터 산술

### 문제
```c
struct jsu { char name[12]; int os, db, hab, hhab; };
struct jsu st[3] = { {"데이터1",95,88}, {"데이터2",84,91}, {"데이터3",86,75} };
struct jsu* p = &st[0];
(p+1)->hab  = (p+1)->os + (p+2)->db;
(p+1)->hhab = (p+1)->hab + p->os + p->db;
printf("%d", (p+1)->hab + (p+1)->hhab);
```

### 핵심 개념 — `(p+n)->필드`는 st[n]의 필드

> `p=&st[0]`, `p+1`=&st[1], `p+2`=&st[2]. hab/hhab는 초기화 안 돼 계산으로 채움.

### 단계별 트레이싱

```
st[0]: os=95, db=88
st[1]: os=84, db=91
st[2]: os=86, db=75

(p+1)->hab = st[1].os + st[2].db = 84 + 75 = 159
(p+1)->hhab = 159(hab) + st[0].os + st[0].db = 159 + 95 + 88 = 342
출력 = hab + hhab = 159 + 342 = 501
```

### ✅ 정답
```
501
```

## Q18. C — 포인터 배열 + 이중 역참조

### 문제
```c
int *arr[3];
int a = 12, b = 24, c = 36;
arr[0] = &a; arr[1] = &b; arr[2] = &c;
printf("%d", *arr[1] + **arr + 1);
```

### 핵심 개념 — `*arr[1]` vs `**arr`

```
arr는 "int* 3개"의 배열
*arr[1] = *(arr[1]) = *(&b) = b = 24
**arr   = *(*arr) = *(arr[0]) = *(&a) = a = 12
```

### 단계별 트레이싱

```
*arr[1] + **arr + 1
= 24 + 12 + 1
= 37
```

### ✅ 정답
```
37
```

## Q19. Java — 비트/논리 연산자 (^, |, &)

### 문제
```java

if((a==2 | a==c) & !(c>d) & (1==b ^ c!=d)) {

    else System.out.println(b);
} else { ... }
```

### 핵심 개념 — `|` `&` `^` (비트 논리 연산자)

> ⚠️ `|`, `&`는 **단락 없이** 양쪽 다 평가. `^`(XOR)는 **다르면 true**.

### 단계별 트레이싱

**바깥 if 조건:**
```
(a==2 | a==c): (false | 3==3=true) = true
!(c>d): !(3>5) = !false = true
(1==b ^ c!=d): (1==4=false) ^ (3!=5=true) = false^true = true
→ true & true & true = true → if 진입
```
**a = b + c = 4 + 3 = 7**
```
안쪽 if(7==b ^ c!=a): (7==4=false) ^ (3!=7=true) = false^true = true
→ println(a) → 7
```

### ✅ 정답
```
7
```

## Q20. Java — 싱글톤 (static 공유)

### 문제
```java
class Connection {
    private static Connection _in = null;
    private int count = 0;
    static public Connection get() {
        if(_in == null) { _in = new Connection(); return _in; }
        return _in;
    }
    public void count() { count++; }
    public int getCount() { return count; }
}
// main
Connection conn1 = Connection.get();  conn1.count();
Connection conn2 = Connection.get();  conn2.count();
Connection conn3 = Connection.get();  conn3.count();
System.out.print(conn1.getCount());
```

### 핵심 개념 — 싱글톤은 인스턴스가 하나

> ⚠️ **핵심**: `get()`은 static `_in`이 null일 때만 new. 이후엔 **같은 객체** 반환. conn1=conn2=conn3 **모두 동일 객체** → count가 공유되어 3번 증가.

### 단계별 트레이싱

```
conn1 = get() → _in 생성 (count=0), conn1.count() → count=1
conn2 = get() → 기존 _in 반환, conn2.count() → count=2 (같은 객체!)
conn3 = get() → 기존 _in 반환, conn3.count() → count=3
conn1.getCount() → 3
```

### ✅ 정답
```
3
```
