## Q05. SQL — GROUP BY + COUNT + AS

### 지문 키워드
> 학과별 튜플 수 출력, **WHERE 금지, GROUP BY·집계함수·AS 필수**

### 개념
```
GROUP BY 학과 → 학과별 그룹
COUNT(*) → 그룹별 행 수
AS 학과별튜플수 → 별칭
```

### 단계별 트레이싱
```
전기: 1명, 컴퓨터: 2명, 건축: 2명 → 학과별 COUNT
```

### ✅ 정답
```sql
SELECT 학과, COUNT(*) AS 학과별튜플수
FROM 학생
GROUP BY 학과;
```

## Q10. 프로세스 상태 전이

### 지문 키워드
> 프로세스 상태 전이도의 빈칸 ①~③

### 왜 이게 답인가?

### 메커니즘 — 프로세스 상태

```
      제출 → 준비(Ready) ⇄ 실행(Run) → 종료
                 ↑            ↓
                 └── 대기(Block) ←┘  (I/O 요청 시 대기, 완료 시 준비로)

① 준비(Ready): CPU 할당 대기
② 실행(Run): CPU 점유 중
③ 대기(Block/Wait): I/O 등으로 대기
```

- 준비→실행: 디스패치, 실행→준비: 타임아웃(선점)
- 실행→대기: I/O 요청, 대기→준비: I/O 완료

### ✅ 정답
```
① 준비   ② 실행   ③ 대기
```

## Q16. C — 문자 포인터 `*p+2`

### 문제
```c
char *p = "KOREA";
printf("%s\n", p);        // ①
printf("%s\n", p+3);      // ②
printf("%c\n", *p);       // ③
printf("%c\n", *(p+3));   // ④
printf("%c\n", *p+2);     // ⑤ ← 함정
```

### 핵심 개념 — `*(p+3)` vs `*p+2`

> ⚠️ **핵심 함정**: `*p+2` = `(*p)+2` = 'K'(75)+2 = 77 = **'M'**. `*(p+3)`(E)과 다름!

```
K(0) O(1) R(2) E(3) A(4)
```

### 단계별 트레이싱

| 줄 | 식 | 값 | 출력 |
|----|-----|-----|------|
| ① | %s, p | 전체 | KOREA |
| ② | %s, p+3 | 인덱스3부터 | EA |
| ③ | %c, *p | p[0] | K |
| ④ | %c, *(p+3) | p[3] | E |
| ⑤ | %c, *p+2 | 'K'(75)+2=77 | M |

### ✅ 정답
```
KOREA
EA
K
E
M
```

## Q18. Java — 10진 → 2진 변환

### 문제
```java
int a[] = new int[8];
int i = 0, n = 10;
while ( ① ) {           // i < a.length
    a[i++] = ② ;        // n % 2 (나머지 = 2진 자리)
    n /= 2;
}
for (i=7; i>=0; i--) System.out.printf("%d", a[i]);
// 출력: 00001010
```

### 핵심 개념 — 나머지 연산으로 2진 변환

> `n % 2`로 이진 자리(0/1)를 얻고 `n /= 2`로 다음 자리. 역순 출력.

### 단계별 트레이싱

```
n=10: 10%2=0 → a[0]=0, n=5
n=5 : 5%2=1  → a[1]=1, n=2
n=2 : 2%2=0  → a[2]=0, n=1
n=1 : 1%2=1  → a[3]=1, n=0
... 나머지 a[4~7]=0
역순 출력: a[7]..a[0] = 0000 1010
```

### ✅ 정답
```
① i < a.length   ② n % 2
```

## Q19. Java — 상속 재귀 (동적 바인딩)

### 문제
```java
class Parent {
    int com(int num) {
        if (num <= 1) return num;
        return com(num-1) + com(num-2);
    }
}
class Child extends Parent {
    int com(int num) {
        if (num <= 1) return num;
        return com(num-1) + com(num-3);   // -2가 아니라 -3
    }
}
Parent obj = new Child();
System.out.print(obj.com(4));   // (지문의 compute는 com의 오타)
```

### 핵심 개념 — 실제 객체(Child)의 com() 호출

> ⚠️ `obj`는 Parent 타입이나 실제 객체는 Child → 모든 `com()`은 **Child.com()** (n-1, n-3).

### 단계별 트레이싱

```
com(1)=1, com(0)=0, com(-1)=-1
com(2)=com(1)+com(-1)=1+(-1)=0
com(3)=com(2)+com(0)=0+0=0
com(4)=com(3)+com(1)=0+1=1
```

### ✅ 정답
```
1
```
