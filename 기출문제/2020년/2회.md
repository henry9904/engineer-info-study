## Q18. Python — set 연산

### 문제
```python
asia = {'한국', '중국', '일본'}
asia.add('베트남')      # 추가
asia.add('중국')        # 중복 → 무시
asia.remove('일본')     # 제거
asia.update(['홍콩', '한국', '태국'])  # 여러 개 추가(한국 중복 무시)
print(asia)
```

### 핵심 개념 — set은 중복 불허 · 순서 없음

### 단계별 트레이싱

```
{한국, 중국, 일본}
+ 베트남 → {한국, 중국, 일본, 베트남}
+ 중국(중복) → 변화 없음
- 일본 → {한국, 중국, 베트남}
+ [홍콩, 한국(중복), 태국] → {한국, 중국, 베트남, 홍콩, 태국}
```

### ✅ 정답
```
{'한국', '중국', '베트남', '홍콩', '태국'}
```
> ⚠️ 원소 5개. **출력 순서는 실행마다 다름**(set은 무순서).

## Q20. Java — super() 생성자

### 문제
```java
class A {
    int a;
    public A(int n) { a = n; }
    public void print() { System.out.println("a=" + a); }
}
class B extends A {
    public B(int n) {
        super(n);       // 부모 생성자 호출 (a = n)
        super.print();  // 부모 print()
    }
}
// main: B obj = new B(10);
```

### 핵심 개념 — super(n)으로 부모 생성자 호출

### 단계별 트레이싱

```
new B(10) → B(10):
   super(10) → A(10): a = 10
   super.print() → A.print(): "a=10"
```

> main에서 obj 생성만 해도 생성자 안의 print()가 실행됨.

### ✅ 정답
```
a=10
```

## 📌 이 회차 핵심 암기

- **애자일**(반복·유연) vs 폭포수(순차·경직)
- **DB 설계**: 요·개·논·물·구
- **웹서비스**: SOAP(통신)·WSDL(명세)·UDDI(등록)
- **형상관리**: 변경 추적·통제 (Git/SVN/CVS)
- **정적 분석**(실행 X) vs 동적 분석(실행 O)
- **IPSec**: 3계층 암호화 (AH+ESP)
- **RTO(복구 시간)** vs RPO(데이터 손실 시점)
- **옵저버 패턴**: 상태 변화 → 구독자 자동 통지 (일대다)
- **chmod 751**: rwx/r-x/--x
- **super(n)**: 부모 생성자 호출 (Q20)
- **set**: 중복 불허·무순서 (Q18)
