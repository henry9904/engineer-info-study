## Q01. RAID 0

### 지문 키워드
> **"두 개 이상 HDD를 병렬 연결, 하나처럼 이용"**
> **"스트라이프(Stripe) 방식"**
> **"하나에 문제 생기면 다른 디스크까지 사용 불가"**

### 왜 이게 답인가?

| 지문 키워드 | 연결 포인트 | 정답 |
|------------|------------|------|
| **스트라이프·병렬·성능** | 데이터 분산 저장 | RAID 0 |
| **하나 고장 → 전체 사용 불가** | 이중화(패리티) 없음 | RAID 0 |

### 메커니즘 — RAID 레벨 비교

```
RAID 0: 스트라이핑, 속도↑ 안정성 없음 (1개 고장 = 전체 손실)
RAID 1: 미러링, 똑같이 복제 (안정성↑, 용량 절반)
RAID 5: 스트라이핑 + 분산 패리티 (1개 고장 복구 가능)
```

### ✅ 정답
```
RAID 0
```

## Q02. REDO / UNDO

### 지문 키워드
> **1. "로그로 오류 트랜잭션을 재실행하여 복구"**
> **2. "로그로 오류 관련 내용을 취소하여 복구"**

### 왜 이게 답인가?

| 지문 핵심 키워드 | 연결 포인트 | 정답 |
|----------------|------------|------|
| **재실행**하여 복구 | 다시 실행 | **REDO** |
| **취소**하여 복구 | 되돌림 | **UNDO** |

### 메커니즘

```
REDO: 커밋됐지만 반영 안 된 트랜잭션 → 다시 실행 (앞으로)
UNDO: 커밋 안 된 트랜잭션의 변경 → 취소 (뒤로)
```

### ✅ 정답
```
① REDO   ② UNDO
```

## Q03. 삭제 이상 (Deletion Anomaly)

### 지문 키워드
> 데이터베이스 이상 현상 중 **삭제 이상** 서술

### 왜 이게 답인가?

> 한 튜플을 삭제할 때, **함께 지우고 싶지 않은 정보까지 연쇄적으로 삭제**되어 정보 손실이 발생하는 현상.

### 메커니즘 — 이상 현상 3종

```
삽입 이상: 원치 않는 데이터를 함께 넣어야만 삽입 가능
삭제 이상: 한 튜플 삭제 시 필요한 정보까지 함께 손실  ← 이 문제
갱신 이상: 중복 데이터 일부만 갱신 → 불일치 발생
```

### ✅ 정답
```
한 튜플을 삭제할 때, 함께 저장된 다른 정보까지 연쇄적으로 삭제되어
원치 않는 정보 손실이 발생하는 현상
```

## Q10. 슈퍼키 / 후보키 (유일성·최소성)

### 지문 키워드
> **1. "슈퍼키는 (①)의 속성"**
> **2. "후보키는 (①)과 (②)의 속성"**

### 왜 이게 답인가?

| 키 | 성질 |
|----|------|
| 슈퍼키 | **유일성**만 (튜플 구별) |
| 후보키 | **유일성 + 최소성** (불필요 속성 없음) |

### 메커니즘

```
유일성(Uniqueness): 각 튜플을 유일하게 식별
최소성(Minimality): 키를 구성하는 속성이 모두 필요 (하나 빼면 식별 불가)

슈퍼키 = 유일성 O, 최소성 X (속성 더 붙어도 됨)
후보키 = 유일성 O, 최소성 O
```

### ✅ 정답
```
① 유일성
② 최소성
```

## Q12. V모델 테스트 단계

### 지문 키워드
> V모델에서 개발 단계 ↔ 테스트 단계 대응

### 왜 이게 답인가? — V모델 대응 관계

```
요구사항 분석 ───────────────── 인수 테스트 (4)
   기능명세 분석 ─────────── 시스템 테스트 (3)
      설계 ──────────── 통합 테스트 (2)

```

| 단계 | 대응 테스트 |
|------|------------|
| (1) 개발 ↔ | **단위 테스트** |
| (2) 설계 ↔ | **통합 테스트** |
| (3) 기능명세 ↔ | **시스템 테스트** |
| (4) 요구사항 ↔ | **인수 테스트** |

> **암기법**: 아래(개발)→위(요구) = 단위 → 통합 → 시스템 → 인수.

### ✅ 정답
```
① 단위 테스트
② 통합 테스트
```

## Q17. Java — 참조 전달 + 필드 변경

### 문제
```java
class A { int a; int b; }
static void func1(A a) { a.a *= 10; }
static void func2(A a) { a.a += a.b; }
// main
A a = new A();
a.a = 100;
func1(a);       // a.a = 1000
a.b = a.a;      // a.b = 1000
func2(a);       // a.a = 1000 + 1000
System.out.println(a.a);
```

### 핵심 개념 — 객체는 참조 전달 (변경 유지)

> ⚠️ **핵심**: 객체 `a`를 함수에 넘기면 **같은 객체의 주소**가 전달되어, 함수 안 변경이 원본에 반영된다.

### 단계별 트레이싱

| 단계 | 실행 | a.a | a.b |
|------|------|-----|-----|
| 초기 | a.a=100 | 100 | 0 |
| func1(a) | a.a *= 10 | 1000 | 0 |
| a.b=a.a | | 1000 | 1000 |
| func2(a) | a.a += a.b | 2000 | 1000 |

### ✅ 정답
```
2000
```

## Q18. C — 재귀 (i-2 감소)

### 문제
```c
int func(int i) {
    if (i <= 1) return 1;
    return i * func(i-2);      // 2씩 감소!
}
// func(5)
```

### 핵심 개념 — 2씩 줄이는 곱 (이중 계승)

> `i * func(i-2)` → 5, 3, 1 순으로 곱함 (홀수만).

### 단계별 트레이싱

```
func(5) = 5 * func(3)
        = 5 * (3 * func(1))
        = 5 * (3 * 1)
        = 15
```

### ✅ 정답
```
15
```

## Q19. C — 숫자 뒤집기 빈칸

### 문제
```c
int num = 1234, div = 10, res = 0;
while (num ① 0) {          // num > 0
    res = res * div;
    res = res + num ② div;  // num % div (끝자리)
    num = num ③ div;        // num / div
}
printf("%d", res);          // 4321
```

### 핵심 개념 — 끝자리 추출(%) + 자리 밀기(*10)

### 단계별 트레이싱

| num | res*10 | +num%10 | num/10 | res |
|-----|--------|---------|--------|-----|
| 1234 | 0 | +4 | 123 | 4 |
| 123 | 40 | +3 | 12 | 43 |
| 12 | 430 | +2 | 1 | 432 |
| 1 | 4320 | +1 | 0 | 4321 |

→ 4321 (숫자 뒤집기)

### ✅ 정답
```
① >   ② %   ③ /
```

## Q20. C — 최대 소인수

### 문제
```c
int isPrime(int n) { for(i=2;i<n;i++) if(n%i==0) return 0; return 1; }
int main() {
    int number = 13195, max_div = 0;
    for (i=2; i<number; i++)
        if (isPrime(i)==1 && number%i==0)   // 소수이면서 약수
            max_div = i;
    printf("%d", max_div);                   // 가장 큰 소인수
}
```

### 핵심 개념 — 13195의 가장 큰 소인수

```
13195 = 5 × 7 × 13 × 29
소인수: 5, 7, 13, 29
가장 큰 소인수 = 29
```

### ✅ 정답
```
29
```

## 📌 이 회차 핵심 암기

- **RAID 0**(스트라이핑·안정성X) / RAID 1(미러링) / RAID 5(분산 패리티)
- **REDO**(재실행) vs **UNDO**(취소) — 로그 회복
- **이상 현상**: 삽입·삭제·갱신 이상
- **TKIP**=WEP 대체 무선 보안 / **NUI**=자연스러운 동작 인터페이스 / **ISMS**=정보보호 인증
- **블랙박스**(동등분할·경계값·원인효과) vs **화이트박스**(경로·루프·커버리지)
- **슈퍼키(유일성)** vs **후보키(유일성+최소성)**
- **워터링 홀**=표적이 자주 가는 사이트 매복(APT)
- **V모델**: 단위→통합→시스템→인수 (개발↔요구 대응)
- **객체는 참조 전달** → 함수 내 변경이 원본 반영 (Q17)
- **13195 = 5·7·13·29** → 최대 소인수 29 (Q20)
