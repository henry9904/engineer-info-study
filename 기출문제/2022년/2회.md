## Q01. 관계해석 (Relational Calculus)

### 지문 키워드
> **"코드(E. F. Codd)가 술어 해석(Predicate Calculus)에 기반해 제안"**
> **"원하는 정보를 정의할 때 계산 수식 사용"**
> **"무엇(what)만 정의하는 비절차적 특성"**

### 왜 이게 답인가?

| 지문 키워드 | 연결 포인트 | 정답 |
|------------|------------|------|
| **술어 해석 기반** | Calculus(해석) | 관계해석 |
| **"무엇"만 정의(비절차적)** | 결과만 기술 | 관계해석 |

### 메커니즘 — 관계대수 vs 관계해석

```
관계대수(Algebra):  "어떻게" 구할지 절차 기술 (σ, π, ⋈ ...)  → 절차적
관계해석(Calculus): "무엇"을 원하는지만 기술 (술어 논리)     → 비절차적
```

> ⚠️ **함정**: "절차적·연산자"면 관계대수, "비절차적·술어논리"면 관계해석.

### ✅ 정답
```
관계해석 (Relational Calculus)
```

## Q04. IGP / EGP / OSPF / BGP

### 지문 키워드
> **① "동일 그룹(AS) 내 라우팅 정보 교환 프로토콜"**
> **② "다른 그룹 간 라우팅 정보 교환 프로토콜"**
> **③ "①의 대표: RIP와 (③)"**
> **④ "②의 대표"**

### 왜 이게 답인가?

| 지문 핵심 키워드 | 연결 포인트 | 정답 |
|----------------|------------|------|
| **AS 내부 라우팅** | Interior Gateway Protocol | **IGP** |
| **AS 간 라우팅** | Exterior Gateway Protocol | **EGP** |
| **IGP 대표(RIP 외)** | 링크 상태 | **OSPF** |
| **EGP 대표** | 인터넷 백본 | **BGP** |

### 메커니즘

```
[AS 내부] ─ IGP (RIP, OSPF) ─ 그룹 안 라우팅
      │
[AS 간]  ─ EGP (BGP)       ─ 그룹 사이 라우팅
```

### ✅ 정답
```
① IGP   ② EGP   ③ OSPF   ④ BGP
```

## Q12. SQL — > ALL 서브쿼리

### 지문 키워드
> **"B 제조사 전체 제품의 단가보다 더 큰 단가"** 제품 출력
> B 제조사 단가: 1500, 2000

### 개념 — ALL vs ANY

| 연산 | 의미 |
|------|------|
| `> ALL (서브쿼리)` | 서브쿼리 **모든 값보다 큰** (=최댓값보다 큰) |
| `> ANY (서브쿼리)` | 서브쿼리 **어느 하나보다 큰** (=최솟값보다 큰) |

> "B의 **전체** 단가보다 크다" = B의 최댓값(2000)보다 커야 함 → `> ALL`.

### 단계별 트레이싱

```
B 제조사 단가 = {1500, 2000}
> ALL {1500, 2000} → 2000 초과
단가 3000(C) 만족 → 출력
```

### ✅ 정답
```
ALL
```

## Q17. C — 포인터 + 배열 차분

### 문제
```c
int a[4] = {0, 2, 4, 8};
int b[3];
int sum = 0;
for (int i=1; i<4; i++) {
    p = a+i;
    b[i-1] = *p - a[i-1];     // a[i] - a[i-1] (차분)
    sum += b[i-1] + a[i];
}
printf("%d", sum);
```

### 핵심 개념 — `*p = *(a+i) = a[i]`, 인접 차분 누적

### 단계별 트레이싱

| i | b[i-1]=a[i]-a[i-1] | a[i] | sum += b+a[i] | sum |
|---|--------------------|------|---------------|-----|
| 1 | 2-0=2 | 2 | 2+2=4 | 4 |
| 2 | 4-2=2 | 4 | 2+4=6 | 10 |
| 3 | 8-4=4 | 8 | 4+8=12 | 22 |

### ✅ 정답
```
22
```

## Q18. Java — switch fall-through

### 문제
```java
int i = 3;
int sum = 0;
switch (i) {
    case 0: sum++;
    case 1: sum+=3;
    case 2: sum-=10;
    case 3: sum=0;
    case 4: sum+=3;
    case 5: sum-=10;
    default: sum--;
}
```

### 핵심 개념 — break 없음 → case 3부터 끝까지 실행

> ⚠️ **핵심 함정**: break가 하나도 없다. i=3 → case 3부터 **아래 전부** 실행.

### 단계별 트레이싱

```
case 3: sum = 0       → 0
case 4: sum += 3      → 3
case 5: sum -= 10     → -7
default: sum--        → -8
```

### ✅ 정답
```
-8
```

## Q19. Java — 객체 필드 변경 + 반복

### 문제
```java
class Cal {
    int a;
    Cal(int a) { this.a = a; }
    int func() {
        int b = 1;
        for (int i=1; i<a; i++) b = a*i + b;
        return a + b;
    }
}
// main
Cal obj = new Cal(3);
obj.a = 5;                 // a를 5로 덮어씀!
int b = obj.func();
System.out.println(obj.a + b);
```

### 핵심 개념 — 생성자 3이지만 obj.a=5로 변경 후 func 실행

> ⚠️ **함정**: 생성자로 3을 넣었지만 `obj.a = 5`로 바꿨으므로 func에서 a=5.

### 단계별 트레이싱

```
func() with a=5:  b=1
i=1: b = 5*1 + 1 = 6
i=2: b = 5*2 + 6 = 16
i=3: b = 5*3 + 16 = 31
i=4: b = 5*4 + 31 = 51
return a+b = 5 + 51 = 56  → b(main) = 56

println(obj.a + b) = 5 + 56 = 61
```

### ✅ 정답
```
61
```

### 문제
```python
str1 = "REMEMBER NOVEMBER"
str2 = str1[:3] + str1[12:16]
str3 = "R AND %s" % "STR"
print(str2 + str3)
```

### 핵심 개념 — 인덱스 세기 + % 포맷

```
인덱스: R0 E1 M2 E3 M4 B5 E6 R7  8 N9 O10 V11 E12 M13 B14 E15 R16
str1[:3]    = "REM"
str1[12:16] = "EMBE"   (12~15)
str2 = "REM" + "EMBE" = "REMEMBE"
str3 = "R AND STR"
str2 + str3 = "REMEMBER AND STR"
```

### ✅ 정답
```
REMEMBER AND STR
```

## 📌 이 회차 핵심 암기

- **관계대수(절차적)** vs **관계해석(비절차적·술어논리)**
- **라우팅**: IGP(AS 내부: RIP·OSPF) vs EGP(AS 간: BGP)
- **알파(내부·개발환경) vs 베타(사용자·실환경)** 테스트, **회귀**=수정 후 부작용 검사
- **Fan-In(들어옴) vs Fan-Out(나감)**
- **SOLID**: SRP·OCP·LSP·ISP·DIP — "안 쓰는 인터페이스 분리"=ISP
- **/26 서브넷**: 블록 64, 132→네트워크 .128, 호스트 62개
- **> ALL**(최댓값 초과) vs **> ANY**(최솟값 초과) (Q12)
- **COUNT(COL2)**는 NULL 제외 (Q13)
- **함수 종속**: 완전/부분/이행 — 2NF(부분제거)·3NF(이행제거)
- **switch fall-through**: break 없으면 아래 case 전부 (Q18)
- **객체 필드 변경**: obj.a=5로 덮으면 생성자 값 무시 (Q19)
