## Q01. C언어 — 포인터 배열 이동 함수

### 문제

```c
int* next(int *p) { return p + 1; }       // int 포인터 +1
int** shift(int **pp) { return pp + 1; }   // int** 포인터 +1
int x[]={10,20,30}, y[]={40,50,60};
int *arr[] = { x, y, x+1, y+2 };   // {x[0]주소, y[0]주소, x[1]주소, y[2]주소}
int **pp = arr;
printf("%d ", **pp);              // ①
printf("%d ", *next(*(pp+1)));    // ②
pp = shift(pp);                   // pp → arr[1]
printf("%d ", **pp);              // ③
*pp = next(*pp);                  // arr[1]을 한 칸 앞으로
printf("%d ", **pp);              // ④
```

### 🤔 포인터 배열 메모리 그림

```
x = [10,20,30]      y = [40,50,60]

int *arr[] = 주소를 담는 칸 4개(포인터 배열):
   arr[0] ─► &x[0](10)
   arr[1] ─► &y[0](40)
   arr[2] ─► &x[1](20)
   arr[3] ─► &y[2](60)

int **pp = arr;   // pp는 "arr의 칸"을 가리키는 이중 포인터
   *pp   = arr[?] (칸 안의 주소)      → 한 번 풀면 int* (어떤 원소 주소)
   **pp  = 그 주소가 가리키는 int      → 두 번 풀면 값
```
> 두 헬퍼: `next(int*)` = 값 주소를 +1(옆 원소), `shift(int**)` = 칸을 +1(옆 포인터). 무엇을 +1 하는지 구분이 핵심.

### 한 줄씩 따라가기

| 줄 | 식 풀이 | 값 | 출력 |
| -- | -- | -- | -- |
| pp=arr | pp → arr[0] | | |
| ① | **pp = *arr[0] = x[0] | 10 | **10** |
| ② | *(pp+1)=arr[1]=&y[0], next→&y[1], * | y[1]=50 | **50** |
| pp=shift(pp) | pp → arr[1] | | |
| ③ | **pp = *arr[1] = y[0] | 40 | **40** |
| *pp=next(*pp) | arr[1]: &y[0]→&y[1] (칸 내용 교체) | | |
| ④ | **pp = *arr[1] = y[1] | 50 | **50** |

### ✅ 정답

```
10 50 40 50 
```

## Q05. Python — 분할정복 홀짝 부호

### 문제

```python
def diff_sum(arr):
    if len(arr) == 1:
        val = arr[0]
        return val if val % 2 == 0 else -val    # 짝수 +, 홀수 -
    mid = len(arr) // 2
    left = diff_sum(arr[:mid]); right = diff_sum(arr[mid:])
    if (arr[mid-1] + arr[mid]) % 2 == 0:
        return left + right + mid
    else:
        return left + right - mid
nums = [3, 8, 5, 2, 7, 4]
```

### 한 단계씩 따라가기

```
diff_sum([3,8,5,2,7,4]): mid=3, arr[2]+arr[3]=5+2=7 홀 → left+right-3
  left = diff_sum([3,8,5]): mid=1, arr[0]+arr[1]=3+8=11 홀 → left+right-1
     left=diff_sum([3]): 3홀 → -3
     right=diff_sum([8,5]): mid=1, arr[0]+arr[1]=8+5=13 홀 → left+right-1
        diff_sum([8])=+8, diff_sum([5])=-5 → 8+(-5)-1 = 2
     → -3 + 2 - 1 = -2
  right = diff_sum([2,7,4]): mid=1, arr[0]+arr[1]=2+7=9 홀 → left+right-1
     diff_sum([2])=+2, right=diff_sum([7,4]): 7+4=11 홀 → (-7)+(+4)-1 = -4
     → 2 + (-4) - 1 = -3
  최종: left(-2) + right(-3) - 3 = -8
```

### ✅ 정답

```
-8
```

## Q06. Java — 싱글턴 패턴

### 문제 (요지)

```java
class Single {
    private static Single instance = new Single();
    private int num = 0;
    private Single() { num++; }   // 생성 시 num=1
    public static Single get() { return instance; }  // 항상 같은 인스턴스
    public void add() { num++; }
}
a=get(); b=get(); a.add(); b.add(); c=get(); c.add(); d=get(); d.show();
```

### 한 줄씩 따라가기

```
클래스 로딩 시 instance 1회 생성 → num=1 (생성자 num++)
a=b=c=d = 모두 같은 instance
a.add() → num=2
b.add() → num=3
c.add() → num=4
d.show() → num=4
a==b, b==c, c==d → 모두 같은 객체 → true, true, true
```

### ✅ 정답

```
4
true
true
true
```
> ⚠️ 싱글턴: 생성자 private + 정적 인스턴스 1개. num은 공유(4).

## Q07. SQL — COUNT DISTINCT + HAVING/정렬

### 문제

```sql
SELECT DEPT, COUNT(DISTINCT EMP) AS CNT, SUM(AMOUNT) AS TOTAL
FROM SALES WHERE MONTH <= 3
GROUP BY DEPT HAVING SUM(AMOUNT) >= 4000 ORDER BY TOTAL DESC;
```

### 단계별 트레이싱

```
WHERE MONTH<=3 필터 (4월 Choi Eraser 제외):
   A: Kim Pen 2000(1월), Kim Pencil 1000(2월) → EMP={Kim}=1, SUM=3000
   B: Park Pen 2500(2월), Park Pen 3000(2월) → EMP={Park}=1, SUM=5500
   C: Han Pen 3500(3월), Yun Pencil 1500(1월) → EMP={Han,Yun}=2, SUM=5000
HAVING SUM>=4000: A(3000) 제외 → B, C
ORDER BY TOTAL DESC: B(5500), C(5000)
```

### ✅ 정답

| DEPT | CNT | TOTAL |
| ---- | --- | ----- |
| B | 1 | 5500 |
| C | 2 | 5000 |

> ⚠️ A는 4월 제외 후 SUM 3000 < 4000이라 탈락. B는 Park만이라 CNT=1.

## Q08. C언어 — 연결 리스트 노드 앞으로 이동

### 문제

```c
// insert 앞쪽 삽입: 1,2,3,4,5 → 5→4→3→2→1
// move(head, 3): 값 3 노드를 맨 앞으로
```

### 1단계 — 리스트

```
insert 1~5 → 5 → 4 → 3 → 2 → 1
```

### 2단계 — move(head, 3) 링크 재배치

```
시작:  head → 5 → 4 → 3 → 2 → 1 → NULL

① 3을 떼기: 3의 앞 노드(4)의 next를 3의 다음(2)으로 이어붙임
      4.next = 2   →  5 → 4 → 2 → 1   (3만 빠짐)

② 3을 맨 앞에 끼우기:
      3.next = head(5),  head = 3
      →  3 → 5 → 4 → 2 → 1

핵심: "앞 노드 찾기"가 필요 → prev를 따라오며 탐색해야 4.next를 고칠 수 있다.
```

### ✅ 정답

```
3 5 4 2 1 
```
> ⚠️ 앞쪽 삽입이라 5→4→3→2→1. 3을 떼서(4→2 연결) 맨 앞으로 → 3 5 4 2 1.

## Q12. Python — 가변 기본 인자 + copy/deepcopy

### 문제

```python
def append_item(item, my_list=[]):   # 기본 리스트 공유
    my_list.append(item); return my_list
list1 = append_item(1)      # 기본 [1]
list2 = append_item(2)      # 기본 [1,2] (공유)
list3 = append_item(3, [])  # 새 [3]
list4 = copy.copy(list2)    # 얕은 복사(리스트 복제)
list5 = copy.deepcopy(list2)
list1.append(9); list4.append(10); list5.append(11)
```

### 한 줄씩 따라가기

```
list1 = 기본리스트 = [1]
list2 = 기본리스트 = [1,2]  (list1과 같은 객체!)
list3 = [3]  (새 리스트)
list4 = copy(list2) = [1,2] (독립 복사본)
list5 = deepcopy(list2) = [1,2] (독립)
list1.append(9): 기본리스트 → [1,2,9] → list1, list2 모두
list4.append(10): list4 → [1,2,10]
list5.append(11): list5 → [1,2,11]

print list1: [1,2,9]
print list2: [1,2,9]  (list1과 동일 객체)
print list3: [3]
print list4: [1,2,10]
print list5: [1,2,11]
```

### ✅ 정답

```
[1, 2, 9]
[1, 2, 9]
[3]
[1, 2, 10]
[1, 2, 11]
```
> ⚠️ list1·list2는 같은 기본 리스트(공유) → 둘 다 [1,2,9]. list4·5는 복사본이라 독립.

## Q14. Java — 생성자 가상 호출 + 필드 은닉

### 문제 (요지)

```java
class A { A(){ println("A"); show(); } void show(){ println("A"); } }
class B extends A { String msg="B"; B(){ println("B"); show(); } }
class C extends B { String msg="C"; C(){ println("C"); show(); } void show(){ println(msg); } }
B a = new C();  println(a.msg);
```

### 🤔 여기서 부딪히는 3가지 함정

```
① 가상 호출(show): 인스턴스 메서드는 "실제 객체 타입"으로 → 객체는 C → 항상 C.show()
② 생성 순서: super()가 완전히 끝난 뒤에야 그 클래스의 [필드 초기화] → [생성자 본문]
   → 부모 생성자가 도는 동안 자식(C) 필드는 아직 기본값(String=null)!
③ 필드 은닉(field hiding): B.msg 와 C.msg 는 "서로 다른 변수"(같은 이름이지만 숨김)
   → a.msg 는 "선언 타입 B" 기준 → B.msg="B" (메서드와 달리 필드는 선언타입!)
```

### 생성 타임라인 — C.msg가 언제 채워지나

```
new C() 시작
 └ C() 호출 → 먼저 super B()
     └ B() 호출 → 먼저 super A()
         └ A() 본문: "A" 출력 → show()
                     └ 객체=C → C.show() → C.msg 아직 null → "null"
         A() 끝 → (B의 필드 msg="B" 초기화) → B() 본문: "B" 출력 → show()
                     └ C.show() → C.msg 여전히 null(C 필드 아직) → "null"
         B() 끝 → (C의 필드 msg="C" 초기화 ✅) → C() 본문: "C" 출력 → show()
                     └ C.show() → C.msg="C" → "C"
 마지막: a.msg → a는 선언타입 B → B.msg = "B"
```

| # | 실행 | 값 | 출력 |
| -- | -- | -- | -- |
| 1 | A() 본문 | | **A** |
| 2 | A() 속 show()→C.show() | C.msg=null | **null** |
| 3 | B() 본문 | | **B** |
| 4 | B() 속 show()→C.show() | C.msg=null(아직) | **null** |
| 5 | C() 본문 | | **C** |
| 6 | C() 속 show()→C.show() | C.msg="C" | **C** |
| 7 | a.msg (선언타입 B) | B.msg="B" | **B** |

### ✅ 정답


## Q15. C언어 — 이중 연결 리스트 이동

### 문제

```c
// add 앞쪽 삽입: 1,2,3,4 → 4↔3↔2↔1 (양방향)
// second = h->next (3), last = h->next->next->next (1)
// moveAfter(second, last): last(1)를 second(3) 뒤로 이동
```

### 1단계 — 리스트

```
add 1,2,3,4 → 4 ↔ 3 ↔ 2 ↔ 1
first=4, second=3, last=1
```

### 🤔 이중 연결 리스트 이동 = 링크 6개 갈아끼우기

```
양방향이라 각 노드는 prev·next 두 개를 가짐.
"떼기(1을 원래 자리에서)" + "끼우기(3 뒤에)" → 총 6개 포인터 재설정
```

### 2단계 — moveAfter(3, 1) 링크 재배치

```
시작:  4 ⇄ 3 ⇄ 2 ⇄ 1        (⇄ 는 prev/next 양방향)

① 1 떼기: 1의 앞(2)과... 2의 next를 NULL로(1이 맨 뒤였음)
      → 4 ⇄ 3 ⇄ 2

② 1을 3 뒤에 끼우기:
      3.next = 1,  1.prev = 3
      1.next = 2,  2.prev = 1     (원래 3.next였던 2를 1 뒤로)

결과:  4 ⇄ 3 ⇄ 1 ⇄ 2
```
> 단방향과 달리 prev 링크까지 맞춰야 뒤로도(2→1→3) 순회가 깨지지 않는다. 정방향 출력만 물으면 next 체인만 따라가면 됨: 4→3→1→2.

### ✅ 정답

```
4 3 1 2 
```
> ⚠️ 앞쪽 삽입으로 4↔3↔2↔1. moveAfter(3,1)로 1을 3 다음에 → 4 3 1 2.

## Q18. C언어 — 재귀 순서(중위 출력)

### 문제

```c
int func(int n) {
    if (n <= 1) return 1;
    int a = func(n-1);
    printf("%d ", n);       // 중간에 출력
    int b = func(n-2);
    return a + b;
}
// func(4)
```

### 🤔 핵심 — 출력이 func(n-1)과 func(n-2) 사이

### 한 호출씩 따라가기

```
func(4): a=func(3)
   func(3): a=func(2)
      func(2): a=func(1)=1, print "2", b=func(0)=1, return 2
      print "3", b=func(1)=1, return func(2)+1 = 2+1 = 3
   print "4", b=func(2)
      func(2): a=func(1)=1, print "2", b=func(0)=1, return 2
   return func(3)+func(2) = 3+2 = 5
출력: 2 3 4 2  (호출 순서대로)
result = 5
```

### ✅ 정답

```
2 3 4 2 
5
```
> ⚠️ 출력은 func(n-1) 완료 후, func(n-2) 전. func(4)=피보나치 유사 → 결과 5.
