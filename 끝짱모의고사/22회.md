## Q01. C언어 — 포인터 배열 이동 함수

### 문제

```c
int* next(int *p) { return p + 1; }       // int 포인터 +1
int** shift(int **pp) { return pp + 1; }   // int** 포인터 +1
int x[]={10,20,30}, y[]={40,50,60};
int *arr[] = { x, y, x+1, y+2 };   // {x[0]주소, y[0]주소, x[1]주소, y[2]주소}
int **pp = arr;
printf("%d ", **pp);              // ①
printf("%d ", *next(*(pp+1)));    // ②
pp = shift(pp);                   // pp → arr[1]
printf("%d ", **pp);              // ③
*pp = next(*pp);                  // arr[1]을 한 칸 앞으로
printf("%d ", **pp);              // ④
```

### 한 줄씩 따라가기

```
arr[0]=&x[0], arr[1]=&y[0], arr[2]=&x[1], arr[3]=&y[2]
pp = arr (→ arr[0])

① **pp = *(*pp) = *arr[0] = x[0] = 10 → "10"
② *next(*(pp+1)):
   *(pp+1) = arr[1] = &y[0]
   next(&y[0]) = &y[1]
   *(&y[1]) = 50 → "50"
pp = shift(pp) → pp = arr+1 (→ arr[1])
③ **pp = *arr[1] = y[0] = 40 → "40"
*pp = next(*pp): *pp=arr[1]=&y[0], next → &y[1], arr[1]=&y[1]
④ **pp = *arr[1] = y[1] = 50 → "50"
```

### ✅ 정답

```
10 50 40 50 
```

## Q05. Python — 분할정복 홀짝 부호

### 문제

```python
def diff_sum(arr):
    if len(arr) == 1:
        val = arr[0]
        return val if val % 2 == 0 else -val    # 짝수 +, 홀수 -
    mid = len(arr) // 2
    left = diff_sum(arr[:mid]); right = diff_sum(arr[mid:])
    if (arr[mid-1] + arr[mid]) % 2 == 0:
        return left + right + mid
    else:
        return left + right - mid
nums = [3, 8, 5, 2, 7, 4]
```

### 한 단계씩 따라가기

```
diff_sum([3,8,5,2,7,4]): mid=3, arr[2]+arr[3]=5+2=7 홀 → left+right-3
  left = diff_sum([3,8,5]): mid=1, arr[0]+arr[1]=3+8=11 홀 → left+right-1
     left=diff_sum([3]): 3홀 → -3
     right=diff_sum([8,5]): mid=1, arr[0]+arr[1]=8+5=13 홀 → left+right-1
        diff_sum([8])=+8, diff_sum([5])=-5 → 8+(-5)-1 = 2
     → -3 + 2 - 1 = -2
  right = diff_sum([2,7,4]): mid=1, arr[0]+arr[1]=2+7=9 홀 → left+right-1
     diff_sum([2])=+2, right=diff_sum([7,4]): 7+4=11 홀 → (-7)+(+4)-1 = -4
     → 2 + (-4) - 1 = -3
  최종: left(-2) + right(-3) - 3 = -8
```

### ✅ 정답

```
-8
```

## Q06. Java — 싱글턴 패턴

### 문제 (요지)

```java
class Single {
    private static Single instance = new Single();
    private int num = 0;
    private Single() { num++; }   // 생성 시 num=1
    public static Single get() { return instance; }  // 항상 같은 인스턴스
    public void add() { num++; }
}
a=get(); b=get(); a.add(); b.add(); c=get(); c.add(); d=get(); d.show();
```

### 한 줄씩 따라가기

```
클래스 로딩 시 instance 1회 생성 → num=1 (생성자 num++)
a=b=c=d = 모두 같은 instance
a.add() → num=2
b.add() → num=3
c.add() → num=4
d.show() → num=4
a==b, b==c, c==d → 모두 같은 객체 → true, true, true
```

### ✅ 정답

```
4
true
true
true
```
> ⚠️ 싱글턴: 생성자 private + 정적 인스턴스 1개. num은 공유(4).

## Q07. SQL — COUNT DISTINCT + HAVING/정렬

### 문제

```sql
SELECT DEPT, COUNT(DISTINCT EMP) AS CNT, SUM(AMOUNT) AS TOTAL
FROM SALES WHERE MONTH <= 3
GROUP BY DEPT HAVING SUM(AMOUNT) >= 4000 ORDER BY TOTAL DESC;
```

### 단계별 트레이싱

```
WHERE MONTH<=3 필터 (4월 Choi Eraser 제외):
   A: Kim Pen 2000(1월), Kim Pencil 1000(2월) → EMP={Kim}=1, SUM=3000
   B: Park Pen 2500(2월), Park Pen 3000(2월) → EMP={Park}=1, SUM=5500
   C: Han Pen 3500(3월), Yun Pencil 1500(1월) → EMP={Han,Yun}=2, SUM=5000
HAVING SUM>=4000: A(3000) 제외 → B, C
ORDER BY TOTAL DESC: B(5500), C(5000)
```

### ✅ 정답

| DEPT | CNT | TOTAL |
| ---- | --- | ----- |
| B | 1 | 5500 |
| C | 2 | 5000 |

> ⚠️ A는 4월 제외 후 SUM 3000 < 4000이라 탈락. B는 Park만이라 CNT=1.

## Q12. Python — 가변 기본 인자 + copy/deepcopy

### 문제

```python
def append_item(item, my_list=[]):   # 기본 리스트 공유
    my_list.append(item); return my_list
list1 = append_item(1)      # 기본 [1]
list2 = append_item(2)      # 기본 [1,2] (공유)
list3 = append_item(3, [])  # 새 [3]
list4 = copy.copy(list2)    # 얕은 복사(리스트 복제)
list5 = copy.deepcopy(list2)
list1.append(9); list4.append(10); list5.append(11)
```

### 한 줄씩 따라가기

```
list1 = 기본리스트 = [1]
list2 = 기본리스트 = [1,2]  (list1과 같은 객체!)
list3 = [3]  (새 리스트)
list4 = copy(list2) = [1,2] (독립 복사본)
list5 = deepcopy(list2) = [1,2] (독립)
list1.append(9): 기본리스트 → [1,2,9] → list1, list2 모두
list4.append(10): list4 → [1,2,10]
list5.append(11): list5 → [1,2,11]

print list1: [1,2,9]
print list2: [1,2,9]  (list1과 동일 객체)
print list3: [3]
print list4: [1,2,10]
print list5: [1,2,11]
```

### ✅ 정답

```
[1, 2, 9]
[1, 2, 9]
[3]
[1, 2, 10]
[1, 2, 11]
```
> ⚠️ list1·list2는 같은 기본 리스트(공유) → 둘 다 [1,2,9]. list4·5는 복사본이라 독립.

## Q14. Java — 생성자 가상 호출 + 필드 은닉

### 문제 (요지)

```java
class A { A(){ println("A"); show(); } void show(){ println("A"); } }
class B extends A { String msg="B"; B(){ println("B"); show(); } }
class C extends B { String msg="C"; C(){ println("C"); show(); } void show(){ println(msg); } }

```

### 한 줄씩 따라가기

```
B a = new C():
   C() → super B() → super A():
      A() println("A"), show() → 실제타입 C → C.show() → println(C.msg)
         C.msg 미초기화 → null → "null"
   B() println("B"), show() → C.show() → C.msg 아직 미초기화 null → "null"
   C() println("C"), show() → C.show() → C.msg="C" 초기화됨 → "C"
println(a.msg): a 선언타입 B → B.msg="B" → "B"
```
출력: A, null, B, null, C, C, B

### ✅ 정답

```
A
null
B
null
C
C
B
```
> ⚠️ 생성 중 C.show()는 C.msg 미초기화라 null 두 번. a.msg는 선언타입 B → "B".

```

```
2 3 4 2

### 자주 나오는 용어 (이 회차 등장)

- **접근제어 모델**: 벨-라파둘라(기밀)/비바(무결).

- **테스트 오라클**: 참/휴리스틱/일관성/샘플링.
- **트랜잭션**: 원자성/격리성 위배.
- **형상관리**: 형상 통제(변경 승인).
- **SOLID**: SRP/DIP.
- **DB 스키마**: 개념 스키마, 논리/물리 독립성.
- **메모리**: 압축/통합.
- **보안**: 레인보우 테이블/솔팅.
- **이메일**: SMTP(발송)/IMAP(동기화).
