## Q02. C언어 — 이중 포인터 산술

### 문제

```c
int arr[4] = {10, 20, 30, 40};
int *ptr = arr;
int **dptr = &ptr;
printf("%d\n", **dptr + *(ptr+2) - *(arr+1));       // ①
ptr += 1;                                            // ptr → arr[1]
printf("%d\n", *(arr+3) - **dptr + (ptr - arr));     // ②
printf("%d\n", (int)(*dptr - arr) * sizeof(int) + 1000); // ③
```

### 코드 모든 줄 해석 + 포인터 관계 그림

```c
int arr[4] = {10, 20, 30, 40};   // 값이 든 실제 배열
int *ptr = arr;                  // ptr → arr[0] (배열 첫 주소)
int **dptr = &ptr;               // dptr → ptr → arr[0]  (이중 포인터: ptr의 주소)
```

```
그림:  dptr ──▶ ptr ──▶ arr[0]=10  arr[1]=20  arr[2]=30  arr[3]=40
별 벗기기:  dptr(=ptr의 주소)  *dptr(=ptr)  **dptr(=*ptr=ptr가 가리키는 값)
포인터 뺄셈:  (ptr - arr) = "몇 칸 떨어졌나"(인덱스 차). 바이트 아님!
```

### 출력 한 줄씩

**① `**dptr + *(ptr+2) - *(arr+1)`** — (아직 ptr는 arr[0])

| 조각 | 뜻 | 값 |
| -- | -- | -- |
| `**dptr` | *ptr = arr[0] | 10 |
| `*(ptr+2)` | arr[2] | 30 |
| `*(arr+1)` | arr[1] | 20 |

→ 10 + 30 − 20 = **20**

```c
ptr += 1;   // ptr가 한 칸 전진 → 이제 ptr → arr[1]
```

**② `*(arr+3) - **dptr + (ptr - arr)`** — (ptr는 이제 arr[1])

| 조각 | 뜻 | 값 |
| -- | -- | -- |
| `*(arr+3)` | arr[3] | 40 |
| `**dptr` | *ptr = arr[1] (ptr가 이동했으니!) | 20 |
| `(ptr - arr)` | 1칸 차이 | 1 |

→ 40 − 20 + 1 = **21**

**③ `(int)(*dptr - arr) * sizeof(int) + 1000`**

| 조각 | 뜻 | 값 |
| -- | -- | -- |
| `*dptr` | ptr (현재 arr[1] 주소) | — |
| `(*dptr - arr)` | ptr − arr = 1칸 차이 | 1 |
| `sizeof(int)` | int 크기 | 4 |

→ 1 × 4 + 1000 = **1004**

> 🎯 함정: ②③은 `ptr += 1` **이후**라 `**dptr`가 arr[1]=20, 포인터 차가 1. ①과 값이 달라짐.

### ✅ 정답

```
20
21
1004
```
> ⚠️ ②③은 `ptr+=1` 이후라 `**dptr`가 arr[1]=20, 포인터차가 1. sizeof(int)=4.

## Q04. C언어 — 원형 큐 enqueue/dequeue

### 문제

```c
#define MAX_SIZE 5
int queue[MAX_SIZE]; int front=0, rear=0;
int isFull(){ return (rear+1)%MAX_SIZE == front; }
int isEmpty(){ return front == rear; }
void enqueue(int v){ if(!isFull()){ rear=(rear+1)%MAX_SIZE; queue[rear]=v; } }
int dequeue(){ if(!isEmpty()){ front=(front+1)%MAX_SIZE; return queue[front]; } return -1; }
```

### 🤔 핵심 — 원형 큐, MAX_SIZE=5지만 **실제 저장 최대 4개**(한 칸 비움)

isFull은 `(rear+1)%5==front`라 한 칸을 항상 비운다.

### 한 연산씩 따라가기 (인덱스는 %5)

```
초기: front=0, rear=0 (빈 큐)

enqueue(10): rear=1, q[1]=10        [10]
enqueue(20): rear=2, q[2]=20        [10,20]
enqueue(30): rear=3, q[3]=30        [10,20,30]
dequeue():   front=1, return q[1]=10  (버림)  남은 [20,30]
enqueue(40): rear=4, q[4]=40        [20,30,40]
enqueue(50): (rear+1)%5=0==front(1)? 아니오 → rear=0, q[0]=50  [20,30,40,50]
enqueue(60): (rear+1)%5=1==front(1)? 예 → isFull → 무시     [20,30,40,50]
dequeue():   front=2, return q[2]=20 → 출력 "20"   남은 [30,40,50]
enqueue(70): (0+1)%5=1==front(2)? 아니오 → rear=1, q[1]=70  [30,40,50,70]
dequeue():   front=3, return q[3]=30 → 출력 "30"   남은 [40,50,70]
enqueue(80): (1+1)%5=2==front(3)? 아니오 → rear=2, q[2]=80  [40,50,70,80]
enqueue(90): (2+1)%5=3==front(3)? 예 → isFull → 무시        [40,50,70,80]
while !isEmpty:
   dequeue front=4 q[4]=40 → "40"
   dequeue front=0 q[0]=50 → "50"
   dequeue front=1 q[1]=70 → "70"
   dequeue front=2 q[2]=80 → "80"
   front(2)==rear(2)? 예 → 종료
```
출력: 20 30 40 50 70 80

### ✅ 정답

```
20 30 40 50 70 80 
```
> ⚠️ MAX=5지만 한 칸 비워 최대 4개. 60·90은 isFull로 버려짐.

## Q06. Python — 라운드 로빈 이터레이터 병합

### 문제

```python
def func(*args):
    iterators = [iter(arg) for arg in args]
    result = []
    while iterators:
        for it in iterators[:]:      # 복사본 순회
            try:
                result.append(next(it))
            except StopIteration:
                iterators.remove(it)  # 끝난 이터레이터 제거
    return result
a = [1,2,3]; b = ('a','b'); c = "XYZ"
output = func(a, b, c)
```

### 🤔 이 함수가 하는 일

여러 이터러블을 **번갈아 하나씩(round-robin)** 뽑아 합친다. 짧은 것이 끝나면 그 이터레이터만 제거.

### 한 라운드씩 따라가기

```
iterators = [iter([1,2,3]), iter(('a','b')), iter("XYZ")]

라운드1: a→1, b→'a', c→'X'   result=[1,'a','X']
라운드2: a→2, b→'b', c→'Y'   result=[1,'a','X',2,'b','Y']
라운드3: a→3,
         b→ StopIteration → b 제거,
         c→'Z'             result=[1,'a','X',2,'b','Y',3,'Z']
라운드4: a→ StopIteration → a 제거,
         c→ StopIteration → c 제거
         iterators 비어짐 → 종료
```

### ✅ 정답

```
[1, 'a', 'X', 2, 'b', 'Y', 3, 'Z']
```
> ⚠️ 3번째 라운드에서 b는 소진(제거)되고 a·c만 값 배출 → 3, 'Z' 순.

## Q07. 능동적(Active) 공격 분류

### 핵심 개념 — 수동적 vs 능동적

```
수동적(Passive): 정보 수집만, 변경 X → 도청, 트래픽 분석, 감시
능동적(Active):  데이터/시스템에 영향 → 변조, 재전송, 위장, 서비스 거부
```

| ------------- | ------- |
| ㄱ. 도청         | 수동      |
| ㄴ. 트래픽 분석     | 수동      |
| **ㄷ. 재전송**    | **능동**  |
| **ㄹ. 서비스 거부** | **능동**  |
| **ㅁ. 위장**     | **능동**  |
| **ㅂ. 변조**     | **능동**  |
| ㅅ. 감시         | 수동      |

### ✅ 정답

ㄷ, ㄹ, ㅁ, ㅂ

## Q08. Java — 참조 swap + 메서드 결합

### 문제

```java
class Box {
    int width, height;
    Box(int w, int h){ width=w; height=h; }
    void swap(Box other){
        int temp = this.width;
        this.width = other.height;
        other.height = temp;
    }
    int area(){ return width * height; }
    int combine(Box other){
        this.swap(other);
        return this.area() + other.area();
    }
}
box1=(5,4), box2=(3,2);
int result1 = box1.combine(box2);
int result2 = box2.combine(box1);
```

### 한 줄씩 따라가기

시작: box1(w5,h4), box2(w3,h2)

**result1 = box1.combine(box2)** (this=box1, other=box2)
```
swap(box2): temp = box1.width = 5
            box1.width = box2.height = 2   → box1(w2,h4)
            box2.height = temp = 5         → box2(w3,h5)
area 합 = box1.area()(2*4=8) + box2.area()(3*5=15) = 23
→ result1=23,  box1(2,4), box2(3,5)
```

**result2 = box2.combine(box1)** (this=box2, other=box1)
```
swap(box1): temp = box2.width = 3
            box2.width = box1.height = 4   → box2(w4,h5)
            box1.height = temp = 3         → box1(w2,h3)
area 합 = box2.area()(4*5=20) + box1.area()(2*3=6) = 26
→ result2=26,  box1(2,3), box2(4,5)
```

### ✅ 정답

```
23, 26
2x3
4x5
```

## Q12. C언어 — switch fall-through + 시프트

### 문제

```c
int num = 2, result = 0;
for (int i = 0; i < 3; i++) {
    switch (num << i) {          // 2<<0=2, 2<<1=4, 2<<2=8
        case 2:  result += 10;   // break 없음 → fall-through
        case 4:  result += 20; break;
        case 8:  result += 30;   // break 없음
        case 16: result += 40; break;
        default: result += 50;
    }
}
```

### 🤔 핵심 — break 없는 case는 아래로 흘러내림(fall-through)

### 한 반복씩 따라가기

```
[i=0] num<<0 = 2 → case 2 진입
      result += 10 → 10
      (break 없음) → case 4로 흘러 result += 20 → 30
      break
[i=1] num<<1 = 4 → case 4 진입
      result += 20 → 50
      break
[i=2] num<<2 = 8 → case 8 진입
      result += 30 → 80
      (break 없음) → case 16으로 흘러 result += 40 → 120
      break
```
result = 120

### ✅ 정답

```
120
```
> ⚠️ case 2는 break 없어 case 4까지 실행(10+20). case 8도 break 없어 case 16까지(30+40).

## Q13. Java — String 메서드 체인

### 문제

```java
String str = "Hello Java";
String result = str.substring(6).replace('a','o').concat("!").toUpperCase();
int idx = result.indexOf('O', 2);
char ch = result.charAt(idx - 1);
System.out.println(result + ", " + idx + ", " + ch);
```

### 한 줄씩 따라가기

```
str.substring(6) = "Java"       (인덱스 6부터: H0 e1 l2 l3 o4 _5 J6 a7 v8 a9)
.replace('a','o') = "Jovo"      (a→o)
.concat("!") = "Jovo!"
```

## Q14. 조건 커버리지 추가 케이스

### 문제

```c
if( x < 7 || y > 0 ) ...
if( x > 0 && y < 0 ) ...
```

### 🤔 핵심 — 각 개별 조건이 True/False 모두 나와야

### 기존 케이스 분석

| 케이스 | x<7 | y>0 | x>0 | y<0 |

| t1(-2,-3) | T (-2<7) | F (-3>0아님) | F (-2>0아님) | T (-3<0) |
| t2(-9,10) | T (-9<7) | T (10>0) | F | F (10<0아님) |

①(x=-5,y=12), ③(x=5,y=-5)는 x<7=T라 `x<7`의 F를 못 채움.

## Q17. Python — for-else + continue/break

### 문제

```python
data = [
    {"id":1, "values":(10,20,30)},
    {"id":2, "values":(5,15)},
    {"id":3, "values":(25,35,45)},
    {"id":4, "values":(40,)}
]
result = []
for item in data:
    total = 0
    for num in item["values"]:
        if num % 20 == 0: continue      # 20 배수 건너뜀
        if num > 30: break              # 30 초과면 중단(else 미실행)
        total += num
    else:                               # break 없이 정상 종료 시만
        result.append((item["id"], total))
print(result)
```

### 🤔 핵심 — `for-else`의 else는 **break 없이** 루프가 끝나야 실행

### 한 항목씩 따라가기

```
id=1 values(10,20,30):
   10: 10%20≠0, 10>30? no → total=10
   20: 20%20==0 → continue
   30: 30%20≠0, 30>30? no → total=40
   break 없이 종료 → else 실행 → result += (1, 40)

id=2 values(5,15):
   5: total=5
   15: total=20
   break 없음 → else → result += (2, 20)

id=3 values(25,35,45):
   25: total=25
   35: 35>30 → break → else 실행 안 됨 (추가 X)

id=4 values(40,):
   40: 40%20==0 → continue
   break 없이 종료 → else → result += (4, 0)   (total 0)
```
result = [(1,40), (2,20), (4,0)]

### ✅ 정답

```
[(1, 40), (2, 20), (4, 0)]
```
> ⚠️ id=3은 break로 else 미실행 → 제외. id=4는 40이 continue돼 total=0이지만 break 없어 포함.

## Q18. SQL — COUNT + NULL + HAVING(<>)

### 문제

```sql
SELECT DEPT, COUNT(PHONE) AS PHONE, COUNT(EMAIL) AS EMAIL, COUNT(*) AS TOTAL
FROM EMPLOYEE
GROUP BY DEPT
HAVING COUNT(PHONE) <> COUNT(EMAIL);
```
데이터(부서별 PHONE/EMAIL NULL 여부):
영업부: 김민수(둘 다 있음), 이지연(둘 다 NULL)
인사부: 박현준(둘 다 있음), 최수빈(PHONE만)
개발부: 윤소희(EMAIL만), 강태오(둘 다 NULL)

### 🤔 핵심 — COUNT(컬럼)은 NULL 제외, HAVING으로 PHONE≠EMAIL 필터

### 부서별 집계

```
영업부: PHONE=1(김민수), EMAIL=1(김민수), TOTAL=2 → 1=1 → 제외
인사부: PHONE=2(김민수... 아니 박현준,최수빈), EMAIL=1(박현준), TOTAL=2 → 2≠1 → 포함
개발부: PHONE=0, EMAIL=1(윤소희), TOTAL=2 → 0≠1 → 포함
```

### ✅ 정답

| DEPT | PHONE | EMAIL | TOTAL |
| ---- | ----- | ----- | ----- |
| 인사부  | 2     | 1     | 2     |
| 개발부  | 0     | 1     | 2     |

> ⚠️ COUNT(PHONE)/COUNT(EMAIL)은 NULL 제외 카운트. 영업부는 1=1이라 HAVING 탈락.

## Q20. Java — ArrayList 순회 중 삽입/수정

### 문제

```java
ArrayList<Integer> nums = new ArrayList<>();
nums.add(5); nums.add(10); nums.add(15);
nums.add(1, 20);   // 인덱스1에 20 삽입
nums.add(3, 25);   // 인덱스3에 25 삽입
int total = 0;
for(int i=0; i<nums.size(); i++){
    if(nums.get(i) % 10 == 0){          // 10 배수
        nums.add(i, nums.get(i) / 2);   // 절반을 앞에 삽입
        i++;                             // 삽입한 것 건너뜀
    } else if(nums.get(i) > 15){        // 15 초과
        nums.set(i, nums.get(i) - 5);   // 5 감소
        total += nums.get(i);
    } else {
        total -= nums.get(i);           // 그 외 차감
    }
}
```

### 코드 규칙 요약

```
get(i)가 10의 배수  → 그 절반을 "i 위치 앞에" 삽입 + i++ (방금 넣은 것 건너뜀)
get(i) > 15         → 그 값을 -5로 set + total에 더함
그 외                → total에서 뺌
```
> ⚠️ 리스트에 삽입(add)하면 **뒤 원소들의 인덱스가 한 칸씩 밀린다**. 그래서 순회 중 인덱스가 꼬이기 쉬움.

### 1단계 — 초기 리스트 만들기

```
add 5,10,15 → [5, 10, 15]
add(1, 20) → 인덱스1에 20 삽입 → [5, 20, 10, 15]
add(3, 25) → 인덱스3에 25 삽입 → [5, 20, 10, 25, 15]
```

### 2단계 — 루프 매 스텝 추적표 (i, get(i), 동작, 리스트, total)

시작: `nums = [5, 20, 10, 25, 15]`, total=0

| i | nums.get(i) | 분기 | 동작 | 이후 리스트 | total | 다음 i |
| - | ---------- | -- | -- | -------- | ----- | ---- |
| 0 | 5 | else | total-=5 | [5,20,10,25,15] | **-5** | 1 |
| 1 | 20 | 10배수 | add(1,10) 후 i++ | [5,**10**,20,10,25,15] | -5 | 3 |
| 3 | 10 | 10배수 | add(3,5) 후 i++ | [5,10,20,**5**,10,25,15] | -5 | 5 |
| 5 | 25 | >15 | set(5,20), total+=20 | [5,10,20,5,10,**20**,15] | **15** | 6 |
| 6 | 15 | else | total-=15 | (동일) | **0** | 7 |
| 7 | — | 종료 (7 < size 7? 아니오) | — | — | 0 | — |

> 각 행의 "다음 i"는 **10배수 분기의 i++ 한 번 + for문의 i++ 한 번**이 겹쳐 2 증가한 것(예: 1 → 3).

최종: `total = 0`, `nums = [5, 10, 20, 5, 10, 20, 15]`

### ✅ 정답

```
total=0, nums=[5, 10, 20, 5, 10, 20, 15]
```
> 🎯 함정: 20을 만나 절반(10)을 앞에 넣으면 리스트가 [5,**10**,20,...]로 커지고 인덱스가 밀림. i++로 방금 넣은 10은 건너뛰어 무한루프를 방지. 마지막에 15는 15초과가 아니라(>15 거짓) else로 빠져 total-=15.
