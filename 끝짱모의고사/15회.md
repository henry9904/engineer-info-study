## Q02. C언어 — 재귀 + 비트 시프트

### 문제

```c
int func(int *arr, int size, int index) {
    if (index >= size) return 0;
    if (arr[index] % 2 == 0)
        return (arr[index] >> 1) + func(arr, size, index + 1);   // 짝수: /2 후 더함
    else
        return (arr[index] << 1) - func(arr, size, index + 1);   // 홀수: *2 후 뺌
}
int arr[] = {5, 8, 3, 6, 2};
```

### 🤔 이 함수가 하는 일

배열을 재귀로 순회하며, 짝수는 `>>1`(÷2) 더하고, 홀수는 `<<1`(×2)에서 다음 결과를 뺀다.

### 한 호출씩 따라가기 (뒤에서부터 계산)

```
func(index=5): 5>=5 → 0
func(index=4, arr[4]=2 짝): (2>>1)=1 + func(5)=0 → 1
func(index=3, arr[3]=6 짝): (6>>1)=3 + func(4)=1 → 4
func(index=2, arr[2]=3 홀): (3<<1)=6 - func(3)=4 → 2
func(index=1, arr[1]=8 짝): (8>>1)=4 + func(2)=2 → 6
func(index=0, arr[0]=5 홀): (5<<1)=10 - func(1)=6 → 4
```

### ✅ 정답

```
4
```

## Q05. C언어 — 단락 평가 + 함수 부작용

### 문제

```c
int check(int val) { printf("%d ", val); return val % 2; }
int main() {
    int x = 3, y = 0, z = -1;
    if ((x > 0 && check(10)) || (y > 0 && check(20)) || check(30)) printf("A\n");
    if ((y < 0 && check(40)) || (z < 0 && check(50))) printf("B\n");
    if ((x < 0 && check(60)) && (z > 0 || check(70))) printf("C\n");
}
```

### 🤔 핵심 — check()는 호출되면 값 출력. 단락 평가로 호출 여부가 결정됨

### 한 조건씩 따라가기

```
초기: x=3, y=0, z=-1

if1: (x>0 && check(10)) || (y>0 && check(20)) || check(30)
   x>0=참 → check(10) 호출 → "10 ", 반환 10%2=0(거짓)
   → 좌항 거짓. (y>0)=0>0 거짓 → check(20) 호출 안 됨
   → check(30) 호출 → "30 ", 반환 0(거짓)
   전체 거짓 → "A" 출력 안 됨
   출력: "10 30 "

if2: (y<0 && check(40)) || (z<0 && check(50))
   y<0=0<0 거짓 → check(40) 안 됨
   z<0=-1<0 참 → check(50) 호출 → "50 ", 반환 0(거짓)
   전체 거짓 → "B" 안 됨
   출력: "50 "

if3: (x<0 && check(60)) && (z>0 || check(70))
   x<0=3<0 거짓 → check(60) 안 됨, && 좌항 거짓 → 우항 평가 안 함(check(70) 안 됨)
   전체 거짓 → "C" 안 됨
   출력: 없음
```
전체 출력: "10 30 50 "

### ✅ 정답

```
10 30 50 
```
> ⚠️ 단락 평가로 check(20), check(40), check(60), check(70)은 호출 안 됨. 10, 30, 50만 출력.

## Q06. Java — 분할정복 홀짝 조건

### 문제

```java
static int func(int[] arr, int start, int end) {
    if (start >= end) return arr[start];
    int mid = (start + end) / 2;
    int left = func(arr, start, mid);
    int right = func(arr, mid + 1, end);
    if (left % 2 == 0 && right % 2 == 0) return left + right;    // 둘 다 짝 → 합
    else if (left % 2 != 0 && right % 2 != 0) return left - right; // 둘 다 홀 → 차
    else return Math.max(left, right);                            // 섞임 → 최댓값
}
int[] numbers = {7, 2, 9, 4, 5, 8, 1, 6};
```

### 🤔 이 함수가 하는 일

반씩 나눠 두 결과의 홀짝에 따라 합/차/최댓값. (재귀)

### 한 단계씩 따라가기 (핵심 경로)

```
[7,2,9,4,5,8,1,6] → 왼쪽[7,2,9,4], 오른쪽[5,8,1,6]

왼쪽[7,2,9,4]:
  [7,2]: left=7(홀),right=2(짝) → 섞임 → max(7,2)=7
  [9,4]: 9(홀),4(짝) → max=9
  7(홀),9(홀) → 둘 다 홀 → 7-9 = -2
오른쪽[5,8,1,6]:
  [5,8]: 5,8 → max=8
  [1,6]: 1,6 → max=6
  8(짝),6(짝) → 둘 다 짝 → 8+6 = 14
최종: left=-2(짝: -2%2==0), right=14(짝) → 둘 다 짝 → -2+14 = 12
```

### ✅ 정답

```
12
```

## Q09. Python — 트리 후위 순회 조건 출력

### 문제

```python
def traverse(node):
    if node is None: return 0
    left_sum = traverse(node.left)
    right_sum = traverse(node.right)
    if left_sum > right_sum:      # 왼쪽 합 > 오른쪽 합이면 출력
        print(node.data, end=' ')
    return left_sum + right_sum + node.data
# 트리: 10(root), 7(L),3(R), 5,1(7의 자식), 2,1(3의 자식)
```

### 🤔 이 함수가 하는 일

후위 순회하며, 각 노드에서 **왼쪽 서브트리 합 > 오른쪽 서브트리 합**이면 노드값 출력. 반환은 서브트리 전체 합.

### 한 노드씩 따라가기 (후위: 자식 먼저)

```
트리:
         10
       /    \
      7      3
     / \    / \
    5   1  2   1

노드 5: left=0, right=0 → 0>0 거짓. return 5
노드 1(7의 우): 0,0 → return 1
노드 7: left_sum=5, right_sum=1 → 5>1 참 → print 7. return 5+1+7=13
노드 2(3의 좌): return 2
노드 1(3의 우): return 1
노드 3: left_sum=2, right_sum=1 → 2>1 참 → print 3. return 2+1+3=6
노드 10: left_sum=13, right_sum=6 → 13>6 참 → print 10. return 13+6+10=29
```
출력 순서: 7, 3, 10

### ✅ 정답

```
7 3 10 
```

## Q10. 정규형 판별 (BCNF)

### 문제

> R: 원자값 O, 완전 함수 종속 O(부분종속 없음), 이행 종속 없음, **후보키가 아닌 속성도 결정자가 될 수 있음**

### 🤔 핵심 — 조건 해석

```
원자값 → 1NF
부분종속 제거 → 2NF
이행종속 제거 → 3NF
"후보키 아닌 속성도 결정자" → 3NF는 만족하지만 BCNF는 위반
   (BCNF는 모든 결정자가 후보키여야 함)
```
따라서 만족하는 가장 높은 정규형 = **3NF** (BCNF는 못 감).

### ✅ 정답

제3정규형(3NF)
> ⚠️ "후보키 아닌 속성이 결정자가 될 수 있다"는 BCNF 위반 조건 → 최고 3NF.

## Q11. C언어 — 후위 표기식 계산(스택)

### 문제

```c
// "34+52-*" 후위 표기식을 스택으로 계산
char exp[] = "34+52-*";
```

### 🤔 핵심 — 후위 표기(postfix): 피연산자는 push, 연산자는 pop 2개 계산 후 push

### 한 문자씩 따라가기

```
'3': push 3      스택: [3]
'4': push 4      스택: [3,4]
'+': pop 4,3 → 3+4=7, push  스택: [7]
'5': push 5      스택: [7,5]
'2': push 2      스택: [7,5,2]
'-': pop 2,5 → 5-2=3, push  스택: [7,3]
'*': pop 3,7 → 7*3=21, push 스택: [21]
결과: pop → 21
```
> ⚠️ 연산자에서 `val2=pop()` 먼저, `val1=pop()` 나중 → val1 (연산자) val2 순서. 5-2 (5가 먼저 push된 val1).

### ✅ 정답

```
21
```

## Q14. 디스크 스케줄링 식별 (SSTF)

### 문제

> 헤드 초기 50, 요청 큐: 95,180,40,119,11,123,48,65
> 처리 순서: 48 → 40 → 65 → 95 → 119 → 123 → 180 → 11

### 🤔 핵심 — 현재 위치에서 **가장 가까운(탐색시간 최소)** 요청부터 처리

### 순서 검증

```
50에서 가장 가까움: 48(거리2) → 이동
48에서: 40(8) vs 65(17)... → 40이 더 가까움? 48→40(8), 48→65(17) → 40
40에서: 남은 중 가장 가까움 65(25) vs 11(29)... → 65
... 이런 식으로 매번 최근접 선택 → SSTF
```
초기에 50→48→40(가까운 것 우선)으로 움직이는 패턴 = **SSTF**.
(SCAN이면 한 방향으로 끝까지 갔다가 반대로 감 — 여기선 48,40 후 다시 65로 올라가므로 SCAN 아님)

### ✅ 정답

SSTF(Shortest Seek Time First)

## Q15. Java — 필드 은닉 + super.name

### 문제 (요지)

```java
class A { String name="A"; A(){ print(name+" "); } void print(){ print(name+" "); } }
class B extends A { String name="B"; }
class C extends B { String name="C"; C(){ print(name+" "); }
    void print(){ print(super.name+" "); print(this.name+" "); } }
A obj1 = new B();  A obj2 = new C();
obj1.print();  obj2.print();
```

### 🤔 핵심 — name은 A.name/B.name/C.name이 각자 존재(은닉). "어느 name이냐"가 전부

```
A(){ print(name+" "); } 안의 name → A 코드 안이므로 항상 A.name = "A"
C.print()의 super.name → B.name = "B",  this.name → C.name = "C"
print()(생성자가 부름)는 실제 객체 타입의 것 → B는 오버라이드 없어 A.print(), C는 C.print()
```

### 실행을 순서대로 (출력마다 "어디서 나온 것인지")

| 시점 | 실행 | 출력 |
| -- | -- | -- |
| `A obj1=new B()` | B 기본생성자 → super() A() → A() 안 print(name), name=**A.name** | `A ` |
| `A obj2=new C()` | C()→B()→A(): A() print(A.name) | `A ` |
| (이어서) | C() 본문 print(name), name=**C.name** | `C ` |
| `obj1.print()` | obj1=B객체, B는 print 없음 → **A.print()** → A.name | `A ` |
| `obj2.print()` | obj2=C객체 → **C.print()**: super.name(B.name) | `B ` |
| (이어서) | C.print(): this.name(C.name) | `C ` |

→ 전체: **`A A C A B C `**

### ✅ 정답

```
A A C A B C 
```
> 🎯 함정: 생성자 A()의 `name`은 실제 객체가 B든 C든 **항상 A.name("A")**(코드가 A에 있으니까). obj.print()의 실행 메서드는 실제 타입(B→A.print, C→C.print). C.print()에서 super.name=B, this.name=C.

## Q16. 벨-라파둘라 / 비바 모델

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |
| --- | ------------------------------ | --------- |
| ①  | **기밀성**, No Read Up/No Write Down | **벨-라파둘라(Bell-LaPadula)** |
| ②  | **무결성**, No Read Down/No Write Up | **비바(Biba)** |

### 메커니즘

```
벨-라파둘라(기밀성): 상위 정보를 하위로 유출 방지
   NRU(상위 못 읽음), NWD(하위로 못 씀)
비바(무결성): 하위 정보가 상위를 오염 방지
   NRD(하위 못 읽음), NWU(상위로 못 씀)
```

### ✅ 정답

① 벨-라파둘라(Bell-LaPadula) 모델  ② 비바(Biba) 모델

## Q17. Python — 재귀 분할 계산

### 문제

```python
def fnc(nums):
    if len(nums) == 1: return nums[0]
    if len(nums) % 2 == 0:
        return fnc(nums[:len(nums)//2]) - fnc(nums[len(nums)//2:])   # 짝수: 좌−우
    else:
        return fnc(nums[1:]) + nums[0]                                # 홀수: 나머지+첫원소
numbers = [4, 3, 1, 2, 5]
```

### 한 단계씩 따라가기

```
fnc([4,3,1,2,5]) — 길이5 홀 → fnc([3,1,2,5]) + 4
  fnc([3,1,2,5]) — 길이4 짝 → fnc([3,1]) - fnc([2,5])
    fnc([3,1]) — 짝 → fnc([3]) - fnc([1]) = 3 - 1 = 2
    fnc([2,5]) — 짝 → fnc([2]) - fnc([5]) = 2 - 5 = -3
    → 2 - (-3) = 5
  → 5 + 4 = 9
```

### ✅ 정답

```
9
```

## Q18. 커버리지 달성률 계산

### 문제

```c
if (a > 0 || b <= 5)   // 문장①
    c = a + b;         // 문장②
else
    c = a - b;         // 문장③
// T = {(1,6), (-1,3)}, short-circuit 미수행
```

### 🤔 핵심 — 문장/결정/조건 커버리지 각각 계산

**테스트 데이터 분석**

| (a,b) | a>0 | b<=5 | 전체(OR) | 실행 문장 |
| ----- | --- | ---- | ------- | ----- |
| (1,6) | T | F | T | ②(if) |
| (-1,3) | F | T | T | ②(if) |

**문장 커버리지**: 실행된 문장 = ①, ② (③은 else라 미실행) → 2/3 = **66%**

**결정 커버리지**: 전체 조건 결과가 T, T만 나옴 (F 안 나옴) → 2가지 중 1 → **50%**

**조건 커버리지**: 각 개별 조건 T/F
- a>0: T(1,6), F(-1,3) → 둘 다 나옴 ✓
- b<=5: F(1,6), T(-1,3) → 둘 다 나옴 ✓
→ 모든 개별 조건 T/F 만족 → **100%**

### ✅ 정답

② (문장 66%, 결정 50%, 조건 100%)
> ⚠️ 결정(전체 분기)은 T만 나와 50%지만, 개별 조건은 각각 T/F 다 나와 조건 100%.

## Q20. EAI 구축 방식

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |
| --- | ------------------------------ | --------- |
| ①  | 시스템 간 **직접 연결**, 복잡성 증가         | **포인트 투 포인트(Point-to-Point)** |
| ②  | **중앙 집중형**, 단일 연결 지점, 확장성↑      | **허브 앤 스포크(Hub&Spoke)** |
| ③  | **미들웨어(메시지 큐)** 비동기, 낮은 결합도     | **메시지 버스(Message Bus)** |

### 메커니즘 — EAI 구축 유형

```
Point-to-Point: 1:1 직접 (연결 수 급증)
Hub&Spoke: 중앙 허브 경유 (관리 용이, 허브 장애 취약)
Message Bus(ESB): 버스로 비동기 통신 (확장성·유연성↑)
Hybrid: Hub&Spoke + Message Bus 혼합
```

### ✅ 정답

① 포인트 투 포인트  ② 허브 앤 스포크  ③ 메시지 버스
