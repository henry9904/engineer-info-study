## Q01. 반이중 통신

### 지문 키워드

> 동시에 주고받지 못함, 한쪽 전송 후 다른 쪽 수신·응답, 명령-응답/트랜잭션 시스템

### 왜 이게 답인가?

| 지문 키워드                       | 정답        |
| ---------------------------- | --------- |
| 양방향이나 **동시 불가**(교대)          | **반이중(Half-Duplex)** |

### 메커니즘 — 통신 방식

```
단방향(Simplex): 한 방향만 (방송)
반이중(Half-Duplex): 양방향이나 교대 (무전기)  ← 이 문제
전이중(Full-Duplex): 동시 양방향 (전화)
```

### ✅ 정답

반이중 통신(Half-Duplex)

## Q02. C언어 — 포인터 배열 선택 정렬 + 부작용

### 문제

```c
int nums[] = {4, 6, 3, 5, 2};
int *ptrs[5];  // 각 원소 주소
// 3번 반복: 최소값 원소를 골라 result에 더하고, 좌우 이웃 값을 조정
for (i=0; i<3; i++) {
    min_idx 찾기 (방문 안 한 것 중 최소)
    visited[min_idx]=1; result += *ptrs[min_idx];
    if (min_idx+1<5) (*ptrs[min_idx+1])--;    // 오른쪽 이웃 -1
    if (min_idx-1>=0) (*ptrs[min_idx-1]) += 2; // 왼쪽 이웃 +2
}
```

### 🤔 이 함수가 하는 일 + 포인터 포인트

```
ptrs[i] = &nums[i]  →  ptrs는 nums 각 원소의 "주소"를 담음.
(*ptrs[k])++ / +=2  →  ptrs[k]가 가리키는 "원본 nums"를 직접 바꿈!
즉 최솟값을 고를 때마다 그 좌우 이웃의 nums 값이 실제로 변한다(부작용).
```

### 한 반복씩 — nums 상태와 result 추적 (`_`=방문 완료)

| 반복 | 방문 안 한 것 중 최소 | result | 좌우 이웃 조정 | 이후 nums |
| -- | ------------ | ----- | -------- | ------- |
| 초기 | — | 0 | — | [4, 6, 3, 5, 2] |
| 1 | idx4=**2** | 2 | 왼(idx3) +2 → 7 | [4, 6, 3, **7**, _] |
| 2 | idx2=**3** | 5 | 오(idx3) −1 → 6, 왼(idx1) +2 → 8 | [4, **8**, _, **6**, _] |
| 3 | idx0=**4** | 9 | 오(idx1) −1 → 7 (왼쪽 없음) | [_, **7**, _, 6, _] |

→ 고른 값 2 + 3 + 4 = **9**

### ✅ 정답

```
9
```
> 🎯 함정: 이웃 값이 계속 바뀌지만, **고른 값 자체는 고를 당시의 값**(2,3,4)을 result에 더함. nums[3]은 7→6으로 변하지만 이미 방문 처리라 다시 안 골림. 포인터로 원본을 건드리는 게 핵심.

## Q04. Java — 생성자 가상 호출 + 필드 은닉 3단계

### 문제 (요지)

```java
class X { String val="X"; X(){ print(); } void print(){ println(val); } }
class Y extends X { String val="Y"; Y(){ super(); val="YY"; print(); } void print(){ println(val); } }
class Z extends Y { String val="Z"; Z(){ val="ZZ"; print(); } void print(){ println(val); } }
X obj = new Z();  println(obj.val);  obj.print();
Y obj2 = new Z();  println(obj2.val);  obj2.print();
```

### 🤔 핵심 — 생성 중 print()는 항상 "실제 객체 Z"의 것, 그때 Z.val은 아직 null

```
new Z() 생성 순서: X() → Y() → Z() 본문 (super가 위로 먼저)
그 안에서 print()는 실제 객체 Z → 항상 Z.print() → Z.val을 봄.
Z.val은 Z 필드 초기화가 끝나기 전엔 null. (Y()가 YY로 바꾸는 건 Y.val이지 Z.val 아님!)
obj.val(직접 접근)은 필드라 선언 타입(X면 X.val, Y면 Y.val).
```

### new Z() 생성 과정 — 매번 Z.print()가 Z.val을 봄

| 단계 | 실행 | Z.val 상태 | 출력 |
| -- | -- | --------- | -- |
| X()→print() | Z.print() → Z.val | 미초기화 null | `null` |
| Y() 본문 val="YY"(Y.val) → print() | Z.print() → **Z.val**(YY 아님!) | 여전히 null | `null` |
| Z() 본문 val="ZZ"(Z.val) → print() | Z.print() → Z.val | "ZZ" | `ZZ` |

**`X obj = new Z()`** → 생성 출력 `null null ZZ`
- `println(obj.val)`: 필드, 선언타입 X → **X.val** = `X`
- `obj.print()`: 실제타입 Z → Z.val = `ZZ`

**`Y obj2 = new Z()`** → 생성 출력 `null null ZZ`
- `println(obj2.val)`: 필드, 선언타입 Y → **Y.val** = `YY`
- `obj2.print()`: 실제타입 Z → `ZZ`

### ✅ 정답

```
null
null
ZZ
X
ZZ
null
null
ZZ
YY
ZZ
```
> ⚠️ 생성 중 print()는 Z.print()지만 Z.val 미초기화 null. obj.val은 선언타입(X→"X", Y→"YY"), obj.print()는 실제타입 Z→"ZZ".

## Q05. SQL — 스칼라 서브쿼리 + NULL

### 문제

```sql
SELECT
  (SELECT COUNT(*) FROM students WHERE score >= 60) AS A,
  (SELECT COUNT(*) FROM students WHERE score < 60) AS B,
  (SELECT COUNT(*) FROM students WHERE score IS NULL) AS C,
  (SELECT COUNT(*) FROM students WHERE score != 100) AS D,
  (SELECT COUNT(*) FROM students) AS E
```
score: 80, NULL, 50, NULL, 100

### 단계별 트레이싱

```
A: score>=60 → 80, 100 = 2  (NULL은 비교 결과 unknown → 제외)
B: score<60 → 50 = 1
C: score IS NULL → 2
D: score!=100 → 80, 50 = 2  (NULL은 !=100도 unknown → 제외)
E: COUNT(*) 전체 = 5
```

### ✅ 정답

| A | B | C | D | E |
| - | - | - | - | - |
| 2 | 1 | 2 | 2 | 5 |

> ⚠️ NULL은 비교(>=, <, !=) 결과가 unknown이라 A,B,D에서 제외. COUNT(*)만 NULL 포함(E=5).

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |

| ①  | 전송 속도/신호 전압 등 물리·전기 특성          | **구문(Syntax)**... 아니, **타이밍?** |

> ⚠️ 정정: 프로토콜 3요소는 구문(Syntax), 의미(Semantics), 타이밍(Timing).
> ① 물리·전기 특성(신호 전압, 속도) = **구문(Syntax)** — 데이터 형식·부호화·신호 레벨
> ② 데이터 구조/형식·오류 정보의 **의미** = **의미(Semantics)**
> ③ 오류·흐름 제어, 연결 설정/해제 **절차/순서** = **타이밍(Timing)**

### 메커니즘 — 프로토콜 3요소

```
구문(Syntax): 데이터 형식, 부호화, 신호 레벨
의미(Semantics): 각 비트의 의미, 제어 정보, 오류 처리
타이밍(Timing): 속도 조절(흐름 제어), 순서 제어
```

### ✅ 정답

① 구문(Syntax)  ② 의미(Semantics)  ③ 타이밍(Timing)

## Q08. SQL — CTAS + EXISTS

### 문제

```sql
CREATE TABLE high_employees ...;
INSERT INTO high_employees SELECT ... FROM employees WHERE salary >= 5000;
SELECT e.* FROM employees e
WHERE EXISTS (
    SELECT 1 FROM (
        SELECT dept, AVG(salary) AS avg_salary FROM high_employees
        GROUP BY dept HAVING AVG(salary) >= 5500
    ) h WHERE h.dept = e.dept
);
```
employees: A(Sales,5000), B(HR,4500), C(Sales,6000), D(IT,5000), E(HR,4800)

### 단계별 트레이싱

```
high_employees (salary>=5000): A(Sales,5000), C(Sales,6000), D(IT,5000)
부서별 평균(high) HAVING >=5500:
   Sales: (5000+6000)/2 = 5500 → >=5500 ✓
   IT: 5000 → ✗
→ h = {Sales}
employees 중 dept가 Sales인 직원: A(101), C(103)
```

### ✅ 정답

| id | name | dept | salary |
| -- | ---- | ---- | ------ |
| 101 | A | Sales | 5000 |
| 103 | C | Sales | 6000 |

> ⚠️ high_employees에는 5000 이상만 들어감. Sales 평균 5500이 기준 충족 → 원본 Sales 직원(A,C) 반환.

## Q11. Java — 재귀 combine

### 문제

```java
static int calculate(int n) {
    if (n <= 1) return n;
    return combine(calculate(n-1), calculate(n-2));
}
static int combine(int a, int b) { return a*a + b*b; }
// calculate(4)
```

### 한 호출씩 따라가기

```
calculate(0)=0, calculate(1)=1
calculate(2) = combine(cal(1), cal(0)) = combine(1,0) = 1²+0² = 1
calculate(3) = combine(cal(2), cal(1)) = combine(1,1) = 1+1 = 2
calculate(4) = combine(cal(3), cal(2)) = combine(2,1) = 2²+1² = 4+1 = 5
```

### ✅ 정답

```
5
```


## Q14. Java — 생성자 체인 + 필드 은닉

### 문제 (요지)

```java
class X { int num; X(){ this(10); println("A"); } X(int num){ this.num=num; println(num); } }
class Y extends X { int num=20; void printNum(){ println(num); } }
class Z extends Y { int num; Z(){ super(); println("B"); }
    Z(int num){ this(); this.num=num; println("C"); println(num); } void printNum(){ println(num); } }
Z obj = new Z(50);  obj.printNum();
```

### 한 줄씩 따라가기

```
new Z(50):
   Z(50) → this() → Z():
      super() → Y()(기본) → X():
         X() → this(10) → X(int 10){ this.num=10; println(10) } → "10"
         X() 본문: println("A") → "A"
      (Y 기본 생성자, 출력 없음)
      Z() 본문: println("B") → "B"
   Z(50) 본문 이어서: this.num=50 (Z.num), println("C") → "C", println(num=50) → "50"
obj.printNum(): 실제타입 Z → Z.printNum() → Z.num=50 → "50"
```

### ✅ 정답

```
10
A
B
C
50
50
```

## Q17. Python — filter/map/reduce 체인

### 문제

```python
data = [10, 3, 7, 2, 5, 8, 1, 4, 9, 6]
filtered = filter(lambda x: x % 3 != 0, data)        # 3의 배수 제외
mapped = map(lambda x: x*2 if x%2 else x+1, filtered) # 홀수 *2, 짝수 +1
mapped_list = [x - 1 for x in mapped]                 # 각 -1
sliced = mapped_list[1:-2]
result = reduce(lambda x, y: x+y if y%2 else x-y, sliced)  # 홀수 +, 짝수 -
```

### 한 단계씩 따라가기

```
filtered(3배수 제외): 10,7,2,5,8,1,4  (3,9,6 제외)
mapped: 10짝→11, 7홀→14, 2짝→3, 5홀→10, 8짝→9, 1홀→2, 4짝→5
   → [11, 14, 3, 10, 9, 2, 5]
mapped_list(-1): [10, 13, 2, 9, 8, 1, 4]
sliced[1:-2]: 인덱스1~4 = [13, 2, 9, 8]
reduce (첫값 13, 이후 홀수면 +y 짝수면 -y):
   acc=13
   y=2(짝): 13-2=11
   y=9(홀): 11+9=20
   y=8(짝): 20-8=12
```

### ✅ 정답

```
12
```

## Q18. SQL — HAVING > ALL 서브쿼리

### 문제

```sql
SELECT category, SUM(price*qty) AS total
FROM products p JOIN orders o ON p.pid = o.pid
GROUP BY category
HAVING SUM(price*qty) > ALL (
    SELECT AVG(price*qty) FROM products p2 JOIN orders o2 ON p2.pid=o2.pid GROUP BY p2.category
);
```

### 단계별 트레이싱

```
카테고리별 SUM(price*qty):
   A: 100*2 + 200*5 = 200+1000 = 1200
   B: 150*1 + 120*3 = 150+360 = 510
   C: 120*4 + 180*2 = 480+360 = 840
카테고리별 AVG(price*qty) (그룹 내 평균):
   A: (200+1000)/2 = 600
   B: (150+360)/2 = 255
   C: (480+360)/2 = 420
> ALL {600, 255, 420} → 최댓값 600보다 커야
   A(1200) > 600 ✓
   B(510) > 600 ✗
   C(840) > 600 ✓
```

### ✅ 정답

| category | total |
| -------- | ----- |
| A | 1200 |
| C | 840 |

> ⚠️ `> ALL`은 서브쿼리 결과의 **최댓값(600)보다 커야** 함. A(1200), C(840)만 통과.

## Q20. C언어 — 연결 리스트 홀짝 누적

### 문제

```c
// 앞쪽 삽입: 4,7,2,5,8 → 리스트 8→5→2→7→4
int confuse(Node* head) {
    if (!head || !head->next) return 0;
    int result = head->value;          // 첫 노드값
    Node* temp = head->next;
    while (temp) {
        result = (result % 2 == 0) ? result + temp->value : result - temp->value;
        temp = temp->next;
    }
    return result;
}
```

### 🤔 이 함수가 하는 일

첫 값에서 시작, 현재 result가 **짝수면 더하고 홀수면 뺀다**.

### 1단계 — 리스트 구성 (앞쪽 삽입 역순)

```
ins 4,7,2,5,8 → 8 → 5 → 2 → 7 → 4
```

### 2단계 — 한 노드씩 따라가기

```
result = 8 (첫 노드)
temp=5: result 8 짝 → 8+5 = 13
temp=2: result 13 홀 → 13-2 = 11
temp=7: result 11 홀 → 11-7 = 4
temp=4: result 4 짝 → 4+4 = 8
```

### ✅ 정답

```
8
```
> ⚠️ 앞쪽 삽입으로 8→5→2→7→4. result 짝/홀에 따라 +/− 교대.
