## Q02. C언어 — 2차원 배열 포인터 산술

### 문제

```c
void compute(int (*arr)[2], int *res) {   // 열 크기를 2로 받음(함정)
    int *row1 = *(arr + 1);
    int val1 = *(*(arr + 0) + 4);
    int val2 = *(row1 + 2);
    int val3 = *(*(arr + 2) + (val2 - val1));
    *res = val1 + val2 + val3;
}
int mat[3][3] = {{10,11,12},{20,21,22},{30,31,32}};
compute(mat, &result);
```

### 🤔 핵심 함정 — 파라미터 `int (*arr)[2]`인데 실제는 3×3

`arr`은 "열 크기 2"인 배열 포인터로 선언 → `arr+1`은 **2개(int) 만큼** 이동. 하지만 실제 메모리는 3×3이 연속 배치. 결국 **선형 메모리를 2칸 단위로 재해석**한다.

메모리 선형: `10,11,12,20,21,22,30,31,32` (인덱스 0~8)

### 단계별 트레이싱

`arr`은 int[2] 단위 포인터. `*(arr+k)`는 선형 인덱스 `2k`부터 시작하는 행.

```
val1 = *(*(arr+0) + 4) = 선형[0 + 4] = 선형[4] = 21
row1 = *(arr+1)        = 선형 인덱스 2부터 (12,20,...)
val2 = *(row1 + 2)     = 선형[2 + 2] = 선형[4] = 21
val3 = *(*(arr+2) + (val2-val1))
     = *(선형인덱스 4 시작 + (21-21))
     = 선형[4 + 0] = 선형[4] = 21
*res = 21 + 21 + 21 = 63
```

| 변수   | 선형 인덱스 | 값 |
| ---- | ------ | - |
| val1 | 4      | 21 |
| val2 | 2+2=4  | 21 |
| val3 | 4+0=4  | 21 |

### ✅ 정답

```
63
```
> ⚠️ `int(*)[2]`로 받아 2칸 단위로 이동 → 세 값 모두 선형[4]=21 → 합 63.

## Q03. Java — 생성자 체인(this/super) + 정적 바인딩

### 문제 (요지)

```java
class A { A(){print("A");} static void f(){print("B");} void g(){print("C");} }
class B extends A { B(){super();print("D");} static void f(){print("E");} void g(){print("F");} }
class C extends B { C(int x){print("G");} C(){this(10);print("H");} void g(){print("J");} }

A obj1 = new C();  obj1.f();  obj1.g();   // 첫 줄
B obj2 = new B();  obj2.f();  obj2.g();   // 둘째 줄
```

### 🤔 핵심 — static 메서드는 **선언 타입**, instance 메서드는 **실제 타입**

- `this(10)`: 같은 클래스 다른 생성자 호출. `super()`: 부모 생성자.
- `f()`는 static → 호출 변수의 **선언 타입** 기준(다형성 X).
- `g()`는 instance → **실제 객체 타입** 기준(오버라이딩).

### 단계별 트레이싱

**첫 줄: `A obj1 = new C();`**
```
new C() → C() → this(10) → C(int) → (암묵적 super) B() → super() A()
  A() "A" → B() "D" → C(int) "G" → C() "H"    ⇒ "ADGH"
obj1.f() → static, 선언타입 A → A.f() "B"
obj1.g() → instance, 실제타입 C → C.g() "J"
⇒ "ADGH" + "B" + "J" = ADGHBJ
```

**둘째 줄: `B obj2 = new B();`**
```
new B() → B() → super() A() : "A" → "D"      ⇒ "AD"
obj2.f() → static, 선언타입 B → B.f() "E"
obj2.g() → instance, 실제타입 B → B.g() "F"
⇒ "AD" + "E" + "F" = ADEF
```

### ✅ 정답

```
ADGHBJ
ADEF
```
> ⚠️ `f()`(static)는 선언 타입(A/B) 기준, `g()`(instance)는 실제 타입 기준.

## Q06. 즉시 갱신 + redo/undo

### 지문 키워드

> 변경 내용을 DB에 **바로 반영**하는 회복 기법(①)
> 커밋된 트랜잭션엔 (②) 연산, 비정상 종료엔 (③) 연산

### 왜 이게 답인가?

| 빈칸 | 지문 키워드              | 정답           |
| --- | ------------------- | ------------ |
| ①  | 변경을 즉시 DB 반영        | **즉시 갱신(Immediate Update)** |
| ②  | 커밋된 것 → 다시 반영       | **redo**     |
| ③  | 비정상 종료 → 취소         | **undo**     |

### 메커니즘 — 로그 기반 회복

```

  장애 시:
    - 커밋 완료 트랜잭션 → REDO (재실행하여 반영 보장)

지연 갱신(Deferred): 커밋 전까지 DB 미반영 → UNDO 불필요, REDO만
```
> ⚠️ 즉시 갱신은 **redo + undo 모두** 필요. 지연 갱신은 redo만.

### ✅ 정답

① 즉시 갱신  ② redo  ③ undo

## Q07. C언어 — switch fall-through + 포인터 증감

### 문제

```c
int a = 2, b = 3, c = 4;
int *p = &a, *q = &b;
switch (*q) {                 // *q = 3
    case 2: *p += (*q)++;
    case 3: *q *= (*p)++;
    case 4: c += (*p + *q);
    default: *q = c - (*p)--;
}
printf("%d, %d, %d\n", a, b, c);   // a, b, c
```

### 🤔 핵심 함정 — `break` 없음 → **fall-through**

`switch(*q)`에서 `*q=3` → `case 3`부터 시작, 이후 **break가 없어 아래 case들을 전부 실행**한다. `*p`는 a, `*q`는 b를 가리킴.

### 단계별 트레이싱

시작: a=2, b=3, c=4. `*q=3` → case 3 진입.

| case      | 식                  | 계산                        | a, b, c 상태     |
| --------- | ------------------ | ------------------------- | -------------- |
| case 3    | `*q *= (*p)++`     | b = 3 × a(2) = 6, 그 후 a→3 | a=3, b=6, c=4  |
| case 4    | `c += (*p + *q)`   | c = 4 + (3 + 6) = 13      | a=3, b=6, c=13 |
| default   | `*q = c - (*p)--`  | b = 13 − a(3) = 10, 그 후 a→2 | a=2, b=10, c=13 |

> 💡 `(*p)++`는 **사용 후 증가** → 식에는 증가 전 값 사용. default의 `(*p)--`도 감소 전 값(3) 사용 후 a가 2로.

### ✅ 정답

```
2, 10, 13
```

## Q08. Python — 문자열 슬라이싱(중간 글자 추출)

### 문제

```python
if len(string) ① 8:          # 8자 이상
    mid = len(string) ② 2    # 중간 인덱스
    if len(string) % 2 == 0:
        m = ( ③ )            # 짝수: 정확히 중간 두 글자
    else:
        m = string[:2] + string[mid] + string[-2:]  # 홀수: 가운데 한 글자
    print(m)
```
입력 "GoodMorning!"(12자) → "Goorg!" / "Fantastic"(9자) → "Faaic"

### 왜 이렇게 채우나?

| 빈칸 | 필요 동작              | 정답              |
| --- | ----------------- | --------------- |
| ①  | 8자 **이상** 조건      | **>=**          |
| ②  | 중간 인덱스 = 길이//2    | **//**          |
| ③  | 짝수: 앞2 + 중간2 + 끝2 | **string[:2] + string[mid-1:mid+1] + string[-2:]** |

### 검산 — "GoodMorning!" (len 12, 짝수)

```
mid = 12 // 2 = 6
앞2 = string[:2] = "Go"
중간2 = string[5:7] = "or"   ← mid-1:mid+1 = 5:7
끝2 = string[-2:] = "g!"
m = "Go" + "or" + "g!" = "Goorg!" ✅
```

### ✅ 정답

① `>=`  ② `//`  ③ `string[:2] + string[mid-1:mid+1] + string[-2:]`

## Q09. 스토리지 — DAS/NAS/SAN + RAID/RAID1

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                              | 정답         |
| --- | ---------------------------------- | ---------- |
| ①  | 서버에 **직접 연결**, 단일 서버 로컬 디스크        | **DAS**    |
| ②  | **파일 단위** 공유, SMB/NFS 프로토콜         | **NAS**    |
| ③  | 고속 **블록 단위**, 파이버 채널 공유            | **SAN**    |
| ④  | 여러 디스크를 **하나의 논리 장치**로 묶음          | **RAID**   |
| ⑤  | 디스크 **복제(미러링)**, 빠른 복구             | **RAID 1** |

### 메커니즘

```
DAS: 서버 ─직접연결─ 디스크 (확장/공유 약함)
NAS: 서버들 ─네트워크(파일)─ 스토리지 (SMB/NFS, 파일 단위)
SAN: 서버들 ─고속망(블록)─ 스토리지 (Fibre Channel, 블록 단위)

RAID 0: 스트라이핑(속도↑, 안정성X)
RAID 1: 미러링(복제, 안정성↑)
RAID 5: 패리티 분산(1개 장애 복구)
```
> 💡 NAS=파일, SAN=블록. RAID1=미러링(복제).

### ✅ 정답

① DAS  ② NAS  ③ SAN  ④ RAID  ⑤ RAID 1

## Q11. Java — try-catch-finally + finally의 return

### 문제

```java
try {
    System.out.print("A");
    System.out.print(method());
} catch (Exception e) { System.out.print("B"); }
finally { System.out.print("C"); }

static String method() throws Exception {
    try { System.out.print("D"); throw new RuntimeException("1"); }
    catch (RuntimeException e) { System.out.print("E"); throw new Exception("2"); }
    finally { System.out.print("F"); return "G"; }   // ← 핵심
}
```

### 🤔 핵심 함정 — finally의 `return`이 예외를 **삼킨다**

`finally`에서 `return`하면, catch에서 던진 예외(`throw new Exception`)가 **무효화**되고 정상 반환된다.

### 단계별 트레이싱

```
main: print "A"
method() 호출:
  try: print "D", RuntimeException 발생
  catch: print "E", Exception("2") 던짐 (아직 전파 대기)
  finally: print "F", return "G"  ← 예외 취소하고 "G" 반환
  → method는 예외 없이 "G" 반환
main: print(method()) → print "G"
      (예외 안 나왔으므로 catch "B" 실행 X)
finally: print "C"
```
출력 누적: A → (D E F) → G → C

### ✅ 정답

```
ADEFGC
```
> ⚠️ finally의 return이 catch의 throw를 덮어써 예외가 사라짐 → catch("B") 미실행.

## Q12. 럼바우(Rumbaugh) OMT 3모델

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |
| --- | ------------------------------ | --------- |
| ①  | **정적 구조**, 객체 다이어그램            | **객체 모델** |
| ②  | 상태 전이/이벤트, **상태도**             | **동적 모델** |
| ③  | 데이터 **흐름/처리**                  | **기능 모델** |
| ④  | 기능 모델의 산출물                     | **DFD**   |

## Q14. C언어 — 부분 문자열 뒤집기

### 문제

```c
void twist(char *str, int x) {
    int len = strlen(str);
    int mid = (len + x) / 2;
    for (int i = x; i < mid; i++) {
        char t = str[i];
        str[i] = str[len-1-(i-x)];
        str[len-1-(i-x)] = t;
    }
}
// "mirror"(x=1), "string"(x=2), "puzzle"(x=3) 각각 twist 후 출력
```

### 🤔 이 함수가 하는 일

`twist`는 앞의 x글자는 그대로 두고, **인덱스 x부터 끝까지**를 좌우 대칭으로 교환(뒤집기)한다. 교환 짝은 `str[i]` ↔ `str[len-1-(i-x)]`.

### 한 줄씩 따라가기 — "mirror", x=1 (len=6, mid=(6+1)/2=3)

```
루프 i는 x(1)부터 mid(3) 전까지 → i=1, 2

i=1: 짝 = str[6-1-(1-1)] = str[5]
     str[1]('i') ↔ str[5]('r')  →  "mrrror"? 아니, 정확히:
     "mirror" = m0 i1 r2 r3 o4 r5
     str[1]↔str[5]:  i↔r  →  m r r r o i  = "mrrroi"
i=2: 짝 = str[6-1-(2-1)] = str[4]
     str[2]('r') ↔ str[4]('o')  →  m r o r r i = "mrorri"
i=3: 3 < 3 거짓 → 종료
```
결과: **mrorri** (앞1글자 'm' 고정, 나머지 "irror"→"rorri")

### 한 줄씩 따라가기 — "string", x=2 (len=6, mid=4)

```
루프 i = 2, 3
"string" = s0 t1 r2 i3 n4 g5
i=2: 짝 str[6-1-(2-2)]=str[5]  → str[2]('r')↔str[5]('g') → s t g i n r
i=3: 짝 str[6-1-(3-2)]=str[4]  → str[3]('i')↔str[4]('n') → s t g n i r
i=4: 4<4 거짓 → 종료
```
결과: **stgnir** (앞2글자 "st" 고정, "ring"→"gnir")

### 한 줄씩 따라가기 — "puzzle", x=3 (len=6, mid=4)

```
루프 i = 3
"puzzle" = p0 u1 z2 z3 l4 e5
i=3: 짝 str[6-1-(3-3)]=str[5]  → str[3]('z')↔str[5]('e') → p u z e l z
i=4: 4<4 거짓 → 종료
```
결과: **puzelz** (앞3글자 "puz" 고정, "zle"→"elz")

### ✅ 정답

```
mrorri
stgnir
puzelz
```
> 💡 x = "고정할 앞 글자 수". 그 뒤부터 끝까지 뒤집기.

SELECT c.name, COUNT(o.amount) AS cnt

customers: Kim(1),Lee(2),Park(3),Choi(4)

| name | cnt | 이유                |

## Q20. 가상주소 — VPN / 페이지 오프셋 비트

### 문제

> 페이지 크기 1,024바이트, 가상 주소 총 20비트. VPN과 오프셋 비트 수는?

### 단계별 계산

**페이지 오프셋 비트** = 페이지 크기를 표현하는 비트
```
1,024 = 2^10 → 오프셋 = 10비트
```

**가상 페이지 번호(VPN) 비트** = 전체 − 오프셋
```
20 − 10 = 10비트
```

| 항목      | 계산          | 비트    |
| ------- | ----------- | ----- |
| 오프셋     | 2^10 = 1024 | 10비트  |
| VPN     | 20 − 10     | 10비트  |

### ✅ 정답

오프셋: 10비트, 가상 페이지 번호(VPN): 10비트

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트

1. **Q02 배열포인터 타입**: `int(*)[2]`로 받으면 `arr+1`이 2칸(int) 이동. 3×3을 2칸 단위로 재해석 → 모두 선형[4]=21.
2. **Q03 static vs instance**: 정적 메서드 `f()`는 선언 타입 기준, 인스턴스 `g()`는 실제 타입 기준. this()/super() 체인 순서 주의.
3. **Q07 switch fall-through**: break 없으면 매칭 case부터 아래 전부 실행. `(*p)++`는 사용 후 증가.
4. **Q08 슬라이싱**: 짝수 중간 두 글자 = `string[mid-1:mid+1]` (mid=len//2).
5. **Q11 finally return**: finally의 return이 catch의 throw를 덮어써 예외 소멸 → catch 미실행.
6. **Q13 substring**: `substring(a,b)`는 인덱스 a~b-1. 공백도 길이에 포함.
7. **Q17 LEFT JOIN + COUNT(컬럼)**: 매칭 없거나 amount가 NULL이면 COUNT=0. cust_id NULL 주문은 매칭 실패.

### 자주 나오는 용어 (이 회차 등장)

- **보안 요소**: 식별/인증/인가, 기밀성/무결성/가용성, 부인방지.
- **웹서비스**: SOAP(전송)/WSDL(설명)/UDDI(검색), HTTP/HTTPS.
- **회복 기법**: 즉시 갱신(redo+undo)/지연 갱신(redo만).
- **스토리지**: DAS(직접)/NAS(파일)/SAN(블록), RAID1(미러링).
- **럼바우 OMT**: 객체/동적/기능(DFD).
- **접근 제어**: MAC(강제)/DAC(소유자)/RBAC(역할).
- **AI**: 딥러닝(신경망)/강화학습(보상 기반).
