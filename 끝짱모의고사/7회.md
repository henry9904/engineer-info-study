## Q02. 커버리지 포함 관계 (참/거짓 판별)

### 문제

> (가) 결정 커버리지 100% → 문장 커버리지 100%?
> (나) 조건/결정 100% → 결정 100%?
> (다) 조건 100% → 결정 100%?
> (라) 조건/결정 100% → 조건 100%?

### 🤔 핵심 — 커버리지 강도 포함 관계

```
문장 ⊂ 결정(분기) ⊂ 조건/결정 ⊂ 변경조건결정(MC/DC) ⊂ 다중조건
조건 커버리지는 결정을 반드시 포함하지 않음(예외)
```

### 각 보기 판별

| 보기 | 명제                     | 판정 | 이유                          |
| --- | ---------------------- | --- | --------------------------- |
| 가  | 결정 100% → 문장 100%      | ✅   | 결정이 문장을 포함(모든 분기 실행=모든 문장 실행) |
| 나  | 조건/결정 100% → 결정 100%   | ✅   | 조건/결정은 결정을 포함               |
| 다  | 조건 100% → 결정 100%      | ❌   | 조건만족이 결정을 보장 못 함(반례 존재)     |
| 라  | 조건/결정 100% → 조건 100%   | ✅   | 조건/결정은 조건도 포함               |

> ⚠️ (다) 함정: 각 조건이 T/F를 가져도 **전체 결정(분기) 결과**의 T/F가 모두 나오지 않을 수 있음.

### ✅ 정답

가, 나, 라

## Q04. C언어 — 이진 탐색 트리 전위 순회

### 문제

```c
Node* insert(Node* root, int data){   // BST 삽입
    if(root==NULL){ ... return newNode; }
    if(data < root->data) root->left = insert(root->left, data);
    else                  root->right = insert(root->right, data);
    return root;
}
void order(Node* root){                // 전위 순회(root→left→right)
    if(root){ printf("%d ", root->data); order(root->left); order(root->right); }
}
int arr[] = {15, 6, 23, 4, 7, 71, 5};  // 순서대로 삽입
```

### 🤔 이 함수가 하는 일

BST에 값을 삽입(작으면 왼쪽, 크면 오른쪽) 후 **전위 순회**(루트 먼저) 출력.

### 1단계 — BST 구성 (한 값씩 삽입)

```
15 삽입 → 루트 15
6  → 6<15 → 15의 왼쪽
23 → 23>15 → 15의 오른쪽
4  → 4<15 왼→ 4<6 → 6의 왼쪽
7  → 7<15 왼→ 7>6 → 6의 오른쪽
71 → 71>15 오→ 71>23 → 23의 오른쪽
5  → 5<15 왼→ 5<6 왼→ 5>4 → 4의 오른쪽
```
트리 모양:
```
          15
        /    \
       6      23
      / \       \
     4   7      71
      \
       5
```

### 2단계 — 전위 순회 (루트 → 왼쪽 → 오른쪽)

```
15 → (왼)6 → (6의 왼)4 → (4의 왼 없음)(4의 오)5 → (7)7
   → (15의 오)23 → (23의 왼 없음)(23의 오)71
방문 순서: 15, 6, 4, 5, 7, 23, 71
```

### ✅ 정답

```
15 6 4 5 7 23 71 
```

## Q05. C언어 — 2차원 배열 포인터 + 교대 부호합

### 문제

```c
int arr[][3] = { 3,5,7, 2,4,6, 1,8,9 };   // 실제 3×3
int sum = 0;
int *p = &arr[0][0];
for(int i=0;i<9;i++){
    if(i%2==0) sum += *(p+i);   // 짝수 인덱스 +
    else       sum -= *(p+i);   // 홀수 인덱스 -
}
```

### 한 줄씩 따라가기

선형 배열: `3,5,7,2,4,6,1,8,9` (인덱스 0~8)

```
i=0(+): +3   sum=3
i=1(-): -5   sum=-2
i=2(+): +7   sum=5
i=3(-): -2   sum=3
i=4(+): +4   sum=7
i=5(-): -6   sum=1
i=6(+): +1   sum=2
i=7(-): -8   sum=-6
i=8(+): +9   sum=3
```

### ✅ 정답

```
결과 : 3
```

## Q06. Python — sorted/filter/map/any/all

### 문제

```python
nums = [3, -6, 9, -12, 15, 0]
a = sorted(nums, key=lambda x: -x)          # 내림차순
b = list(filter(lambda x: x%3==0 and x!=0, nums))  # 3배수 & 0 아님
c = sum(map(abs, nums))                     # 절댓값 합
d = any(x > 10 for x in nums)               # 하나라도 10 초과?
e = all(x % 3 == 0 for x in nums)           # 전부 3배수?
```

### 한 줄씩 따라가기

```
a: key=-x → 큰 값이 앞 → [15, 9, 3, 0, -6, -12]
b: 3배수이고 0 아닌 것 → 3, -6, 9, -12, 15 → [3, -6, 9, -12, 15]
c: |3|+|−6|+|9|+|−12|+|15|+|0| = 3+6+9+12+15+0 = 45
d: 15>10 → True (하나라도 만족)
e: 0도 3의 배수(0%3==0), 모두 3의 배수 → True
```

### ✅ 정답

```
a = [15, 9, 3, 0, -6, -12]
b = [3, -6, 9, -12, 15]
c = 45
d = True
e = True
```
> ⚠️ e: 0%3==0이라 0도 통과 → 전부 True. d: any는 하나만 만족해도 True.

## Q07. Java — 싱글턴 패턴

### 문제 (요지)

```java
class CounterManager {
    private static CounterManager manager = null;
    private int value = 0;
    private CounterManager() {}
    public static CounterManager getManager() {
        if (manager == null) manager = new CounterManager();
        return manager;   // 항상 같은 인스턴스
    }
    public void inValue() { value++; }
}
c1 = getManager();  c2 = getManager();  // c1, c2 동일 객체
c1.inValue(); c1.inValue(); c2.inValue();
```

### 🤔 핵심 — 싱글턴: 인스턴스가 하나뿐

`c1`과 `c2`는 **같은 객체**를 가리킨다 → value도 공유.

### 한 줄씩 따라가기

```
c1 = getManager() → manager 최초 생성, c1이 그것을 가리킴
c2 = getManager() → manager 이미 존재 → c2도 같은 객체
c1.inValue() → value 0→1
c1.inValue() → value 1→2
c2.inValue() → value 2→3  (같은 객체이므로)
c1.getValue() → 3
c2.getValue() → 3
c1 == c2 → 같은 참조 → true
```

### ✅ 정답

```
3
3
true
```

## Q11. C언어 — 이중 포인터 문자열 배열

### 문제

```c
char arr[3][10] = {"hello", "world", "good"};
char *pArr[3] = {arr[2], arr[1], arr[0]};   // 역순 포인터 배열
char **p = pArr;
printf("%s\n", *(p+1));       // ①
printf("%c\n", *(*(p)+1));    // ②
printf("%s\n", *(p+2)+2);     // ③
```

### 핵심 — pArr 매핑

```
pArr[0] = arr[2] = "good"
pArr[1] = arr[1] = "world"
pArr[2] = arr[0] = "hello"
p = pArr
```

### 출력 한 줄씩

```
① *(p+1) = pArr[1] = "world" → "world"
② *(*(p)+1):
   *(p) = pArr[0] = "good"
   "good"+1 = 'o'의 주소, *(...)  = 'o' → 'o'
③ *(p+2)+2:
   *(p+2) = pArr[2] = "hello"
   "hello"+2 = "llo" (인덱스 2부터) → "llo"
```

### ✅ 정답

```
world
o
llo
```

## Q14. C언어 — 역순 순회 최솟값 갱신 카운트

### 문제

```c
int bars[] = {7, 6, 9, 7, 6, 4, 6};
int size = 7;
int *p = bars + size - 1;   // 마지막 원소(인덱스6=6)
int cnt = 1;
for(int i = size-2; i >= 0; i--){   // 뒤에서 두 번째부터 앞으로
    if(*(bars+i) > *p){ cnt++; p = bars + i; }
}
```

### 🤔 이 함수가 하는 일

배열을 **오른쪽에서 왼쪽으로** 보며, 현재 기준(*p)보다 **큰 값**을 만날 때마다 카운트 증가하고 기준 갱신. (오른쪽에서 볼 때 "더 높은 막대" 세기)

### 한 반복씩 따라가기

```
초기: p → bars[6]=6, cnt=1

i=5: bars[5]=4 > *p(6)? 거짓 → 그대로
i=4: bars[4]=6 > 6? 거짓 → 그대로
i=3: bars[3]=7 > 6? 참 → cnt=2, p→bars[3]=7
i=2: bars[2]=9 > 7? 참 → cnt=3, p→bars[2]=9
i=1: bars[1]=6 > 9? 거짓
i=0: bars[0]=7 > 9? 거짓
```
cnt = 3

### ✅ 정답

```
결과 : 3
```
> 💡 오른쪽 끝에서 볼 때 보이는 막대: 6 → 7 → 9 (점점 커지는 것만) = 3개.

## Q19. Java — 참조 전달 + 메서드 상호 호출

### 문제

```java
class Counter {
    int value;
    Counter(int init){ value=init; }
    int exchange(Counter other){
        int temp = this.value;
        this.value = other.value * 2;
        other.value = temp / 2;
        return this.value - other.value;
    }
    int compute(Counter other){
        int result = this.exchange(other);
        result += other.exchange(this);
        return result;
    }
}
cnt1 = new Counter(8);  cnt2 = new Counter(3);
int result = cnt1.compute(cnt2);
```

### 한 줄씩 따라가기

시작: cnt1.value=8, cnt2.value=3. `cnt1.compute(cnt2)`.

```
result = cnt1.exchange(cnt2):   (this=cnt1, other=cnt2)
   temp = cnt1.value = 8
   cnt1.value = cnt2.value*2 = 3*2 = 6
   cnt2.value = temp/2 = 8/2 = 4
   return 6 - 4 = 2
   → result=2,  cnt1=6, cnt2=4

result += cnt2.exchange(cnt1):  (this=cnt2, other=cnt1)
   temp = cnt2.value = 4
   cnt2.value = cnt1.value*2 = 6*2 = 12
   cnt1.value = temp/2 = 4/2 = 2
   return 12 - 2 = 10
   → result = 2 + 10 = 12,  cnt1=2, cnt2=12
```
result=12, cnt1.value=2, cnt2.value=12

### ✅ 정답

```
12, 2, 12
```

## Q20. Python — 중첩 try-finally return

### 문제

```python
def fnc():
    try:
        print('A', end=''); x = 5/0; print('B', end='')   # ZeroDivision
    except ZeroDivisionError:
        print('C', end='')
        try:
            y = int('abc')                                  # ValueError
        except ValueError:
            print('D', end=''); return '1'
        finally:
            print('E', end=''); return '2'                  # finally return
    finally:
        print('F', end=''); return '3'                      # 바깥 finally return
```

### 🤔 핵심 — finally의 return이 앞선 return을 덮어씀

### 한 줄씩 따라가기

```
try: print 'A' → "A"
     5/0 → ZeroDivisionError 발생 (B 출력 안 됨)
except ZeroDivisionError: print 'C' → "C"
   내부 try: int('abc') → ValueError
   내부 except: print 'D' → "D", return '1' 준비
   내부 finally: print 'E' → "E", return '2' → 내부 return '1' 덮어씀 → '2' 반환 시도
바깥 finally: print 'F' → "F", return '3' → 앞의 '2' 덮어씀 → 최종 '3' 반환
```
출력 누적: A C D E F, 반환값 '3'

### ✅ 정답

```
ACDEF3
```
> ⚠️ finally의 return이 항상 최종 승자. 내부 '1'→'2'→바깥 '3'으로 덮어써짐.
