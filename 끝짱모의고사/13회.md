## Q02. C언어 — 변수 스코프 + static

### 문제

```c
int x = 5;                 // 전역
void f() {
    static int x = 2;      // 정적 지역(유지)
    int y = 3;
    x += ++y * 2;          // ++y=4 → x += 8
    printf("%d ", x);
}
int main() {
    int x = 4;             // main 지역
    {
        int x = 7;         // 블록 지역
        f();               // static x: 2→10
        x += x + 1;        // 블록 x: 7 → 7+7+1=15
        printf("%d ", x);
    }
    f();                   // static x: 10→18
    printf("%d", x);       // main x = 4
}
```

### 🤔 핵심 — 블록마다 다른 x, f()의 static x는 유지

### 한 줄씩 따라가기

```
main x=4, 블록 진입 x=7
f() 1회: static x=2, y=3 → ++y=4 → x += 4*2=8 → x=10 → print "10 "
블록 x += x+1 → 7 + (7+1) = 15 → print "15 "
블록 종료 (블록 x 소멸)
f() 2회: static x=10 (유지!), y=3 → ++y=4 → x += 8 → x=18 → print "18 "
main x = 4 → print "4"
```

### ✅ 정답

```
10 15 18 4
```
> ⚠️ f()의 static x는 호출 간 유지(2→10→18). main/블록/전역 x는 각각 별개.

## Q03. C언어 — 원형 큐 회전 처리

### 문제

```c
#define MAX 10
void enqueue(int item){ queue[rear]=item; rear=(rear+1)%MAX; }
int dequeue(){ int item=queue[front]; front=(front+1)%MAX; return item; }
void processQueue(int n) {
    for (int i=1; i<=n; i++) enqueue(i);        // 1~6 삽입
    while ((rear+MAX-front)%MAX > 1) {          // 큐 크기 > 1인 동안
        dequeue();
        enqueue(dequeue());                     // 하나 버리고, 다음을 꺼내 뒤로
    }
}
// processQueue(6) 후 dequeue 2번 출력
```

### 🤔 이 함수가 하는 일

큐 [1,2,3,4,5,6]에서, 크기가 1 초과인 동안: **앞을 버리고, 그다음을 꺼내 맨 뒤로 보냄**. (한 개 건너뛰며 소거 → 마지막 생존자 찾기, 요세푸스 유사)

### 한 반복씩 따라가기

```
큐: [1,2,3,4,5,6] (크기 6)
반복1: dequeue()→1(버림), enqueue(dequeue()→2) → 2를 뒤로  → [3,4,5,6,2]
반복2: dequeue()→3(버림), enqueue(dequeue()→4)          → [5,6,2,4]
반복3: dequeue()→5(버림), enqueue(dequeue()→6)          → [2,4,6]
반복4: dequeue()→2(버림), enqueue(dequeue()→4)          → [6,4]
반복5: dequeue()→6(버림), enqueue(dequeue()→4)          → [4]
크기 1 → while 종료
```
큐에 [4] 남음... 그런데 출력은 dequeue 2번. 마지막 상태를 재확인.

**정정 트레이싱** (크기 계산 `(rear+MAX-front)%MAX`):
```
[1,2,3,4,5,6]
버림1, 2뒤로 → [3,4,5,6,2]
버림3, 4뒤로 → [5,6,2,4]
버림5, 6뒤로 → [2,4,6]
버림2, 4뒤로 → [6,4]
버림6, 4뒤로 → [4]   크기1 종료
```
큐 [4]만 남지만 main은 dequeue를 2번 호출 → 4, 그리고 그 뒤 큐의 잔여 슬롯값 2를 출력.

> 실제 실행 결과: **4, 2** (첫 dequeue=4, 두 번째 dequeue는 큐가 비었지만 front 위치의 잔존값 2를 반환 — 원형 큐에서 이전에 남은 값)

### ✅ 정답

```
4,2
```
> ⚠️ 한 칸 건너 소거로 최종 생존자는 4. 두 번째 dequeue는 빈 큐에서 잔존 슬롯값(2) 반환 → "4,2".

## Q08. Python — 다익스트라 최단경로

### 문제

```python
graph = {'A':{'B':3,'C':1}, 'B':{'A':3,'C':7,'D':5}, 'C':{'A':1,'B':7,'D':2}, 'D':{'B':5,'C':2}}
def dijkstra(graph, start):
    distances = {node: inf for node in graph}
    distances[start] = 0
    queue = [start]
    while queue:
        current = queue.pop(0)
        for a, w in graph[current].items():
            d = distances[current] + w
            if d < distances[a]:
                distances[a] = d
                queue.append(a)
    return distances
result = dijkstra(graph, 'A')
```

### 🤔 이 함수가 하는 일

start('A')에서 각 노드까지 **최단 거리**를 구한다(완화 반복).

### 한 단계씩 따라가기

```
초기: A=0, B=inf, C=inf, D=inf, queue=[A]
A 처리: B=0+3=3, C=0+1=1 → queue=[B,C]
B 처리: A(3+3=6>0 skip), C(3+7=10>1 skip), D(3+5=8<inf → D=8) → queue=[C,D]
C 처리: A(1+1=2>0 skip), B(1+7=8>3 skip), D(1+2=3<8 → D=3) → queue=[D,D]
D 처리: B(3+5=8>3 skip), C(3+2=5>1 skip) → 갱신 없음
D 처리(중복): 동일 → 없음
```
최종: A=0, B=3, C=1, D=3

### ✅ 정답

```
{'A': 0, 'B': 3, 'C': 1, 'D': 3}
```
> 💡 D는 처음 B경유 8이었다가 C경유(1+2=3)로 갱신됨.

## Q10. 순환 복잡도 (Cyclomatic Complexity)

### 문제

```c
if (num > 0) {
    if (num % 2 == 0) printf("양수 짝수");
    else printf("양수 홀수");
} else if (num < 0) printf("음수");
else printf("0");
```

### 🤔 핵심 — 순환 복잡도 = 분기(결정점) 수 + 1

### 계산

```
결정점(조건 판단): if(num>0), if(num%2==0), else if(num<0) → 3개
순환 복잡도 V(G) = 판단 노드 수 + 1 = 3 + 1 = 4
```
(또는 V(G) = E − N + 2 로도 계산 가능, 결과 4)

### ✅ 정답

```
4
```

## Q11. Python — 가변 기본 인자 함정

### 문제

```python
def append_item(item, my_list=[]):    # 기본값 리스트는 한 번만 생성(공유!)
    my_list.append(item)
    return my_list
list1 = append_item(1)      # 기본 리스트 사용
list2 = append_item(2)      # 같은 기본 리스트 재사용
list3 = append_item(3, [])  # 새 리스트 전달
list4 = append_item(4)      # 또 같은 기본 리스트
```

### 🤔 핵심 함정 — 기본 인자 `[]`는 **함수 정의 시 한 번만 생성**되어 호출 간 공유됨

### 한 줄씩 따라가기

```
list1 = append_item(1): 기본리스트 = [1], list1이 그것을 참조
list2 = append_item(2): 같은 기본리스트 = [1,2], list2도 그것 참조
   → list1, list2 모두 [1,2] (같은 객체!)
list3 = append_item(3, []): 새 리스트 [3]
list4 = append_item(4): 기본리스트 = [1,2,4]
   → list1, list2, list4 모두 [1,2,4]
```

### ✅ 정답

```
[1, 2, 4]
[1, 2, 4]
[3]
[1, 2, 4]
```
> ⚠️ 가변 기본 인자는 공유됨 → list1·list2·list4가 같은 [1,2,4]. list3만 별도 [3].

## Q12. Java — 예외 계층 catch 순서

### 문제 (요지)

```java
try { check(value); print("X"); }
catch (ArithmeticException e) { print("Y"); }
catch (Exception e) { print("Z"); }
finally { print("W"); }
print("V");
// check: value==0 → ArithmeticException, value<0 → Exception
// new ExceptionPuzzle(1), (0), (-1)
```

### 한 객체씩 따라가기

```
new ExceptionPuzzle(1): check(1) 예외 없음 → "X", finally "W", "V" → "XWV"
"-"
new ExceptionPuzzle(0): check(0) ArithmeticException → catch(Arithmetic) "Y", "W", "V" → "YWV"
"-"
new ExceptionPuzzle(-1): check(-1) Exception → catch(Exception) "Z", "W", "V" → "ZWV"
```

### ✅ 정답

```
XWV-YWV-ZWV
```
> ⚠️ ArithmeticException은 Exception의 하위라 먼저 잡힘. finally "W"와 "V"는 항상 실행.

## Q15. C언어 — 이중 포인터 문자열 배열

### 문제

```c
char *arr[] = {"pointer", "array", "language", "complexity"};
char **p = arr;
printf("%c\n", *(*(p + 2) + 1));   // ①
p++;                                // p → arr[1]
printf("%s\n", *(p + 1));          // ②
printf("%c\n", *(*p + 2));         // ③
printf("%s\n", *(p - 1) + 4);      // ④
```

### 출력 한 줄씩 따라가기

```
① *(*(p+2)+1): p+2=arr[2]="language", +1 → 'a'의 위치, * → 'a'
p++ → p = &arr[1] (arr[1]="array")
② *(p+1): arr[2] = "language" → "language"
③ *(*p+2): *p=arr[1]="array", +2 → 'r'의 위치, * → 'r'
④ *(p-1)+4: p-1=arr[0]="pointer", +4 → "ter" (인덱스 4부터)
```

### ✅ 정답

```
a
language
r
ter
```

## Q19. Java — static/instance 필드 은닉

### 문제 (요지)

```java
class Parent { static int total=0; int value=0; Parent(){total++;value++;} void add(){total++;value++;} static void resetTotal(){total=0;} void display(){println(total+","+value);} }
class Child extends Parent { static int total=10; int value=10; Child(){total++;value++;} void add(){total++;value++;} static void resetTotal(){total=10;} void display(){println(total+","+value);} }

Parent obj1 = new Parent();  Child obj2 = new Child();
obj1.add(); obj2.add();
obj1.display(); obj2.display();
Parent.resetTotal(); Child.resetTotal();
Child obj3 = new Child();  obj3.add();
obj1.display(); obj3.display();
```

### 🤔 핵심 — Parent.total과 Child.total은 **별개의 정적 필드**

### 한 줄씩 따라가기

```
Parent.total=0, Child.total=10 (각각 별개)

obj1 = new Parent(): Parent() → Parent.total=1, obj1.value=1
obj2 = new Child(): Child() → super Parent()(Parent.total=2, obj2의 Parent.value=1)
   → Child() 본문: Child.total=11, obj2.Child.value=11
obj1.add(): Parent.add() → Parent.total=3, obj1.value=2
obj2.add(): Child.add() → Child.total=12, obj2.Child.value=12
obj1.display(): Parent.display() → Parent.total(3)+","+obj1.value(2) → "3,2"
obj2.display(): Child.display() → Child.total(12)+","+obj2.Child.value(12) → "12,12"
Parent.resetTotal() → Parent.total=0
Child.resetTotal() → Child.total=10
obj3 = new Child(): super Parent()(Parent.total=1) → Child() Child.total=11, obj3.Child.value=11
obj3.add(): Child.total=12, obj3.Child.value=12
obj1.display(): Parent.display() → Parent.total(1)+","+obj1.value(2) → "1,2"
obj3.display(): Child.display() → Child.total(12)+","+obj3.Child.value(12) → "12,12"
```

### ✅ 정답

```
3,2
12,12
1,2
12,12
```
> ⚠️ Parent.total과 Child.total은 이름이 같아도 별개 정적 변수. new Child()는 super로 Parent.total도 증가시킴.

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트
