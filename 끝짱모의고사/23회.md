## Q02. C언어 — 포인터 배열 양끝 교환

### 문제

```c
int a[5] = {2, 7, 3, 8, 1};
int *p = a;
for (i=0; i<2; i++) {          // i=0,1
    temp = *(p+i);
    *(p+i) = *(p+4-i);
    *(p+4-i) = temp;
}
```

### 🤔 이 함수가 하는 일

양끝을 안쪽으로 2번 교환 → 배열 뒤집기(가운데 제외).

### 한 반복씩 따라가기

```
초기: [2, 7, 3, 8, 1]
i=0: a[0]↔a[4] → [1, 7, 3, 8, 2]
i=1: a[1]↔a[3] → [1, 8, 3, 7, 2]
(i=2에서 종료, 가운데 3 유지)
```

### ✅ 정답

```
1 8 3 7 2 
```

## Q06. Python — dict 정렬 + 조건 필터

### 문제

```python
data = {'a':3, 'b':1, 'c':4, 'd':1, 'e':5}
result = sorted(data.items(), key=lambda x: (x[1], x[0]), reverse=True)
print([k for k, v in result if v % 2 == 1])
print(sum(v for k, v in result if k < 'd'))
```

### 한 단계씩 따라가기

```
items: [('a',3),('b',1),('c',4),('d',1),('e',5)]
정렬 key=(값, 키), reverse=True → 값 내림차순, 같으면 키 내림차순:
   ('e',5), ('c',4), ('a',3), ('d',1), ('b',1)
   (값 1인 d,b는 키 내림차순 → d 먼저)
① 홀수 값(v%2==1)인 키: e(5), a(3), d(1), b(1) → ['e','a','d','b']
② k<'d' 인 값 합: a(3), b(1), c(4) → 3+1+4 = 8
```

### ✅ 정답

```
['e', 'a', 'd', 'b']
8
```
> ⚠️ 정렬은 (값,키) 튜플 내림차순 → 동값 d,b는 키 역순으로 d가 앞. k<'d'는 a,b,c.

## Q08. SQL — GROUP BY + HAVING

### 문제

> 학과별 평균 총점 ≥ 85인 학과와 평균총점 조회

### 단계별 트레이싱

```
학과별 평균:
   컴퓨터: (85+78+90)/3 = 84.33 → <85 제외
   전자: (92+88)/2 = 90 → ≥85 ✓
```

### 정답 SQL

```sql
SELECT 학과, AVG(총점) AS 평균총점
FROM 학생
GROUP BY 학과
HAVING AVG(총점) >= 85;
```
결과: 전자 90

### ✅ 정답 → 위 SQL (결과: 전자 90)

## Q09. Java — 필드 은닉 + super/캐스팅

### 문제 (요지)

```java
class A { int x=10; void show(){ print(x); } }
class B extends A { int x=20; void show(){ print(x); print(super.x); } void print2(){ show(); print(x); } }
class C extends B { int x=30; void show(){ print(x); print(super.x); print(((A)this).x); } }
B obj = new C();
print(obj.x);  obj.show();  obj.print();
```
(코드상 B의 메서드명은 print())

### 🤔 이 문제의 전부 = "필드냐 메서드냐" 한 줄 규칙

```
┌─────────────┬──────────────────────────────────────┐
│ 필드(x)     │ "어느 타입 이름으로 접근했나"(정적)     │
│             │  obj.x → obj의 선언타입,  super.x→부모, │
│             │  ((A)this).x → 캐스팅한 A 기준          │
├─────────────┼──────────────────────────────────────┤
│ 메서드(show)│ "실제 객체가 무엇이냐"(동적/가상)       │
│             │  obj가 C 객체면 → 항상 C.show()        │
└─────────────┴──────────────────────────────────────┘
```
> 세 개의 x가 각 클래스에 따로 존재(은닉): A.x=10, B.x=20, C.x=30. 객체 하나(C) 안에 셋 다 들어있고, "어떤 이름으로 부르냐"에 따라 골라잡는다.

### obj 정보

```
B obj = new C();
   선언타입 = B  (필드 접근 기준)
   실제타입 = C  (메서드 호출 기준)
```

### 한 줄씩 따라가기

| 호출 | 무엇 | 기준 | 결과 |
| -- | -- | -- | -- |
| obj.x | 필드 | 선언타입 B → B.x | **20** |
| obj.show() | 메서드 → 실제타입 C → **C.show()** | | |
| ├ x | 필드(C 안) | C.x | **30** |
| ├ super.x | 필드 | B.x | **20** |
| └ ((A)this).x | 필드 | 캐스팅 A → A.x | **10** |
| obj.print() | 메서드 → B.print() (C에 없음, 상속) | | |
| ├ show() | 메서드 → 실제타입 C → C.show() | 30 20 10 | **30 20 10** |
| └ print(x) | print2 본문의 x | B에서 실행 → B.x | **20** |

전체: 20 · (30 20 10) · (30 20 10) · 20 = **20 30 20 10 30 20 10 20**

### ✅ 정답

```
20 30 20 10 30 20 10 20 
```
> ⚠️ 필드는 참조/캐스팅 타입 기준(obj.x=B.x=20, ((A)this).x=A.x=10). show()는 실제타입 C.

## Q11. C언어 — 포인터 이동 + static 누적

### 문제

```c
int* move(int *p, int n) { return p + n; }
int calc(int *p) { static int s = 1; s += 2; return *p + s; }
int a[] = {5, 10, 15, 20, 25};
int *p = a;  int *q = move(p, 2);   // q = &a[2]
printf("%d ", *p);              // a[0]
printf("%d ", *q);              // a[2]
printf("%d ", calc(p));         // a[0] + s
printf("%d ", calc(q));         // a[2] + s
printf("%d\n", *(move(a, 4)));  // a[4]
```

### 🤔 두 가지 핵심 개념

```
① move(p,n)=p+n : 포인터 산술 → 주소를 n칸 옮김 (값 복사 아님, "가리키는 위치" 이동)
② static int s : 함수가 끝나도 "죽지 않고" 값을 유지 → 다음 calc 호출에 s가 이어짐
   (일반 지역변수라면 매번 1로 초기화되지만, static은 최초 1회만 =1)
```
> s는 calc 호출을 거듭할수록 1 → 3 → 5 로 누적. 이게 이 문제의 함정.

### 배열·포인터 그림

```
a = [ 5 , 10 , 15 , 20 , 25 ]
      ▲              ▲          ▲
      p            q=move(p,2) move(a,4)
     a[0]           a[2]        a[4]
```

### 한 줄씩 따라가기 (s 값 추적)

| 줄 | 계산 | s | 출력 |
| -- | -- | -- | -- |
| *p | a[0] | 1 | **5** |
| *q | a[2] (q=&a[2]) | 1 | **15** |
| calc(p) | s: 1→3, a[0]+s = 5+3 | 3 | **8** |
| calc(q) | s: 3→5, a[2]+s = 15+5 | 5 | **20** |
| *(move(a,4)) | a[4] | 5 | **25** |

### ✅ 정답

```
5 15 8 20 25
```
> ⚠️ static s는 호출 간 유지(1→3→5). calc(p)=8, calc(q)=20.

## Q12. Python — 클로저 + nonlocal(독립성)

### 문제

```python
def outer(x):
    y = x * 2
    def inner(z):
        nonlocal y
        y += z
        return y
    return inner
func1 = outer(3)   # y=6
func2 = outer(5)   # y=10 (별도 클로저)
print(func1(1), func1(2), func2(1), func1(1))
```

### 🤔 핵심 — 각 outer 호출마다 **독립적인 y**(클로저) 생성

### 한 호출씩 따라가기

```
func1 = outer(3): func1의 y = 6
func2 = outer(5): func2의 y = 10 (func1과 독립)

func1(1): y = 6+1 = 7 → 7
func1(2): y = 7+2 = 9 → 9
func2(1): y = 10+1 = 11 → 11  (func2 독립 클로저)
func1(1): y = 9+1 = 10 → 10  (func1의 y는 9에서 이어짐)
```

### ✅ 정답

```
7, 9, 11, 10
```
> ⚠️ func1과 func2는 각자 y를 가진 별도 클로저. func1의 y는 6→7→9→10으로 누적.

## Q14. C언어 — 재귀 거듭제곱(분할)

### 문제

```c
int func(int n, int m) {
    if (m == 0) return 1;
    if (m == 1) return n;
    if (m % 2 == 0) return func(n*n, m/2);     // 짝수 지수: (n²)^(m/2)
    else return n * func(n, m-1);               // 홀수: n × n^(m-1)
}
// func(2,10), func(3,4)
```

### 🤔 이 함수가 하는 일

**빠른 거듭제곱(분할정복)** — n^m 계산.

### 한 호출씩 따라가기

```
func(2,10): m 짝 → func(4,5)
   func(4,5): m 홀 → 4*func(4,4)
      func(4,4): 짝 → func(16,2)
         func(16,2): 짝 → func(256,1) = 256
      → 256
   → 4*256 = 1024
→ func(2,10) = 1024  (2^10)

func(3,4): 짝 → func(9,2)
   func(9,2): 짝 → func(81,1) = 81
→ 81  (3^4)
```

### ✅ 정답

```
1024
81
```
> 💡 2^10=1024, 3^4=81. 분할정복 거듭제곱.

## Q16. SQL — SUM + NULL 처리

### 문제

```sql
SELECT SUM(C1 + C2 + C3 + C4) AS R1,
       SUM(C1) + SUM(C2) + SUM(C3) + SUM(C4) AS R2
FROM TBL;
```
데이터: (1,2,NULL,4),(3,NULL,5,6),(NULL,2,3,1),(4,5,6,NULL),(7,8,9,1)

### 🤔 핵심 함정 — `C1+C2+C3+C4`에 NULL이 하나라도 있으면 그 **행 전체가 NULL**

### 단계별 트레이싱

**R1 = SUM(C1+C2+C3+C4)** — 행별 합, NULL 포함 행은 NULL
```
행1: 1+2+NULL+4 = NULL (제외)
행2: 3+NULL+5+6 = NULL (제외)
행3: NULL+2+3+1 = NULL (제외)
행4: 4+5+6+NULL = NULL (제외)
행5: 7+8+9+1 = 25  ← 유일하게 NULL 없음
R1 = 25
```

**R2 = SUM(C1)+SUM(C2)+SUM(C3)+SUM(C4)** — 열별 합(NULL 제외)
```
SUM(C1) = 1+3+4+7 = 15  (NULL 제외)
SUM(C2) = 2+2+5+8 = 17
SUM(C3) = 5+3+6+9 = 23
SUM(C4) = 4+6+1+1 = 12
R2 = 15+17+23+12 = 67
```

### ✅ 정답

| R1 | R2 |
| -- | -- |
| 25 | 67 |

> ⚠️ R1은 행 단위 덧셈이라 NULL 있는 행 전체가 NULL(4행 제외) → 25. R2는 열별 SUM(NULL 개별 제외) → 67. 결과가 다름!

## Q18. Java — try-catch-finally + 참조

### 문제 (요지)

```java
static int calc(Box a, Box b, int n) {
    try {
        a.add(n);
        if (n % 2 == 0) throw new RuntimeException();
        b.add(a.value);
        return a.value + b.value;
    } catch (RuntimeException e) {
        b.add(10);
        return b.value;
    } finally {
        a.add(100);
        b.add(1);
    }
}
Box x = new Box(5), y = new Box(3);
int r1 = calc(x, y, 2);   // x,y
int r2 = calc(y, x, 3);   // 순서 바뀜
```

### 🤔 헷갈리는 2가지 규칙

```
① return 값 "동결": return이 실행되면 반환값은 그 순간 확정(예약)됨.
   그 뒤 finally가 돌며 필드를 더 바꿔도 → 이미 예약된 반환값은 안 바뀐다.
   (단, Box처럼 "객체 참조"를 반환하면 필드 변화가 보이지만, 여기선 int라 값 동결)
② 별칭(aliasing): calc(x,y) 안의 a·b는 바깥 x·y와 "같은 객체"(참조 전달).
   → a.add()는 실제로 x를 바꾸고 함수 종료 후에도 x에 남는다. 호출마다 값 누적!
```
> 그래서 r2 호출은 r1이 바꿔놓은 x=107, y=14 상태에서 이어서 시작한다.

### r1 = calc(x, y, 2) — a=x, b=y (시작 x=5, y=3)

| 단계 | 실행 | x | y | 반환예약 |
| -- | -- | -- | -- | -- |
| try a.add(2) | x=5+2 | 7 | 3 | |
| n%2==0 → throw | 예외 발생 | 7 | 3 | |
| catch b.add(10) | y=3+10 | 7 | 13 | |
| return b.value | **13 동결** | 7 | 13 | 13 |
| finally a.add(100) | x=7+100 | 107 | 13 | 13 |
| finally b.add(1) | y=13+1 | 107 | 14 | 13 |

→ r1=**13**, 출력 **13,107,14**

### r2 = calc(y, x, 3) — a=y, b=x (시작 x=107, y=14, 순서 바뀜!)

| 단계 | 실행 | x | y | 반환예약 |
| -- | -- | -- | -- | -- |
| try a.add(3) | y=14+3 | 107 | 17 | |
| n%2==0? 아니오 | 예외 없음 | 107 | 17 | |
| b.add(a.value) | x=107+17 | 124 | 17 | |
| return a+b | 17+124=**141 동결** | 124 | 17 | 141 |
| finally a.add(100) | y=17+100 | 124 | 117 | 141 |
| finally b.add(1) | x=124+1 | 125 | 117 | 141 |

→ r2=**141**, 출력 **141,125,117**

### ✅ 정답

```
13,107,14
141,125,117
```
> ⚠️ return 값은 finally 전에 확정되지만, finally에서 Box(참조) 필드는 계속 변함 → 출력되는 x,y 값은 finally 반영된 값.
