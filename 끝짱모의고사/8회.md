## Q01. C언어 — 괄호 짝 검사(스택 카운트)

### 문제

```c
int func(const char *expr) {
    int cnt = 0;
    while (*expr) {
        if (*expr == '(') cnt++;
        else if (*expr == ')') {
            cnt--;
            if (cnt < 0) return 0;   // 닫는 괄호가 먼저 나오면 실패
        }
        expr++;
    }
    return cnt == 0;                 // 최종 짝 맞으면 1
}
// arr = {"()(()())", "())(", "(()(()))(", NULL}
```

### 코드 모든 줄 해석

```c
int func(const char *expr) {     // expr = 문자열 시작 주소. const = 내용 안 바꿈
    int cnt = 0;                 // cnt = 아직 안 닫힌 '(' 개수(열림 깊이)
    while (*expr) {              // *expr(현재 글자)가 '\0'(끝) 아닐 때까지
        if (*expr == '(') cnt++; //   '(' 만나면 깊이 +1
        else if (*expr == ')') { //   ')' 만나면
            cnt--;               //     깊이 -1
            if (cnt < 0) return 0;//    음수 = 짝 없는 ')'가 먼저 나옴 → 즉시 실패(0)
        }
        expr++;                  //   다음 글자로 포인터 전진
    }
    return cnt == 0;             // 끝에서 깊이 0이면 균형(1=True), 아니면 0(False)
}
int main() {
    const char *arr[] = {"()(()())", "())(", "(()(()))(", NULL};  // 문자열 4개(끝은 NULL)
    const char **ptr = arr;      // ptr = 문자열 배열을 가리키는 이중 포인터
    while (*ptr) {               // *ptr(현재 문자열)이 NULL 아닐 때까지
        printf("%s\n", func(*ptr) ? "True" : "False");  // 검사 결과 출력
        ptr++;                   //   다음 문자열로
    }
}
```

### 🤔 이 함수가 하는 일

`(`면 깊이 +1, `)`면 −1. **도중에 음수가 되면 그 즉시 실패**(닫힘이 먼저 나옴). 끝까지 갔을 때 깊이 0이면 균형.

### 한 문자열씩 — cnt 값 추적표

**① "()(()())"**

| 글자 | ( | ) | ( | ( | ) | ( | ) | ) | 끝 |
| -- | - | - | - | - | - | - | - | - | - |
| cnt | 1 | 0 | 1 | 2 | 1 | 2 | 1 | 0 | **0** |

→ 끝에서 cnt=0 → **True**

**② "())("**

| 글자 | ( | ) | ) | ← 여기서 |
| -- | - | - | - | ---- |
| cnt | 1 | 0 | **-1** | cnt<0 → 즉시 return 0 |

→ 3번째 `)`에서 음수 → **False** (뒤의 `(`는 볼 필요도 없음)

**③ "(()(()))("**

| 글자 | ( | ( | ) | ( | ( | ) | ) | ) | ( | 끝 |
| -- | - | - | - | - | - | - | - | - | - | - |
| cnt | 1 | 2 | 1 | 2 | 3 | 2 | 1 | 0 | **1** | **1** |

→ 끝에서 cnt=1 ≠ 0 → **False** (마지막 `(`가 안 닫힘)

### ✅ 정답

```
True
False
False
```
> 🎯 포인트: ②는 중간에 음수(닫힘 먼저)라 실패, ③은 마지막 `(`가 안 닫혀 끝에서 cnt≠0이라 실패. 실패 사유가 다름.

## Q03. C언어 — 8진수 + 비트/증감 복합 루프

### 문제

```c
int x = 012;   // 8진수 → 10진수 10
int y = 8;
int result = 0;
for(int i = 0; i <= x; i += 3) {          // i = 0,3,6,9
    if((i & y) && (++y % 2 == 0))  result += y;
    else if((i | x) == (y++))      result -= i;
    else                           result ^= (--y);
}
printf("%d, %d, %d\n", x, y, result);
```

### 🤔 핵심 — `012`는 8진수(=10), 조건마다 y가 증감

### 한 반복씩 따라가기

`x=10`(012). i는 0,3,6,9.

```
초기: x=10, y=8, result=0

[i=0] i&y = 0&8 = 0 → 첫 조건 거짓(&& 단락, ++y 실행 안 됨)
      else if: (i|x)=(0|10)=10, y++ → 비교엔 8, 10==8? 거짓, y→9
      else: result ^= (--y) → y→8, result = 0^8 = 8
      (y=8, result=8)
[i=3] i&y = 3&8 = 0 → 거짓
      else if: (3|10)=11, y++ → 비교 8, 11==8? 거짓, y→9
      else: --y → y→8, result = 8^8 = 0
      (y=8, result=0)
[i=6] i&y = 6&8 = 0 → 거짓
      else if: (6|10)=14, y++ → 비교 8, 14==8? 거짓, y→9
      else: --y → y→8, result = 0^8 = 8
      (y=8, result=8)
[i=9] i&y = 9&8 = 8 (0이 아님, 참) && (++y%2==0):
         ++y → y→9, 9%2==0? 거짓 → 첫 조건 전체 거짓
      else if: (9|10)=11, y++ → 비교 9, 11==9? 거짓, y→10
      else: --y → y→9, result = 8^9 = 1
      (y=9, result=1)
```
최종: x=10, y=9, result=1

### 비트 계산 참고

```
6 & 8: 0110 & 1000 = 0000 = 0
9 & 8: 1001 & 1000 = 1000 = 8
8^8=0, 0^8=8, 8^9: 1000^1001=0001=1
```

### ✅ 정답

```
10, 9, 1
```
> ⚠️ `012`=8진수 10. 각 반복마다 else if의 y++ 또는 else의 --y로 y가 오르내림.

## Q04. Java — 전위 증감 + 문자열 결합 조건

### 문제

```java
int x = 3, y = 2;
for (int i = 1; i <= 3; i++) {
    x += i;
    y += ++x;
    if ((x + y) % 2 == 0) System.out.println(x + y + "Java");
    else                  System.out.println("Java" + x + y);
}
```

### 🤔 핵심 — `x+y+"Java"`는 산술 후 문자열, `"Java"+x+y`는 전부 문자열

### 한 반복씩 따라가기

```
초기: x=3, y=2

[i=1] x += 1 → x=4
      y += ++x → ++x는 x→5, y = 2+5 = 7
      x+y = 12, 짝수 → println(x+y+"Java")
         12 + "Java" → "12Java"     ← 12는 int로 먼저 계산
[i=2] x += 2 → x=7
      y += ++x → ++x→8, y = 7+8 = 15
      x+y = 23, 홀수 → println("Java"+x+y)
         "Java"+8 → "Java8", +15 → "Java815"
[i=3] x += 3 → x=11
      y += ++x → ++x→12, y = 15+12 = 27
      x+y = 39, 홀수 → println("Java"+x+y)
         "Java"+12 → "Java12", +27 → "Java1227"
```

### ✅ 정답

```
12Java
Java815
Java1227
```
> ⚠️ 짝수 분기 `x+y+"Java"`는 `(x+y)` 산술 먼저(12) → "12Java". 홀수 분기는 "Java"부터라 문자열 연결(8,15 그대로 붙음).

## Q05. Python — 메모이제이션 재귀(피보나치)

### 문제

```python
memo = {}
def func(n):
    if n in memo: return memo[n]
    if n <= 2: memo[n] = 1
    else: memo[n] = func(n-1) + func(n-2)
    print(n, memo[n])
    return memo[n]
result = func(5)
print("결과:", result)
```

### 코드 모든 줄 해석

```python
memo = {}                        # 계산 결과를 저장하는 딕셔너리(캐시)
def func(n):
    if n in memo: return memo[n] #   이미 계산했으면 바로 반환 (print 없음!)
    if n <= 2: memo[n] = 1        #   1,2는 값 1 (피보나치 시작)
    else: memo[n] = func(n-1) + func(n-2)  # 없으면 재귀로 계산해 저장
    print(n, memo[n])            #   ★ "처음 계산할 때만" 여기 도달 → 출력
    return memo[n]
```

### 🤔 핵심 — print는 **처음 계산될 때만** 실행. 이미 memo에 있으면 첫 줄에서 return하므로 print 안 됨.

### 한 호출씩 — 호출 트리 + memo 상태

```
func(5) [memo 없음] → func(4) + func(3)
├ func(4) [없음] → func(3) + func(2)
│ ├ func(3) [없음] → func(2) + func(1)
│ │ ├ func(2): n<=2 → memo[2]=1 → 📢print "2 1"
│ │ └ func(1): n<=2 → memo[1]=1 → 📢print "1 1"
│ │ memo[3]=1+1=2 → 📢print "3 2"
│ └ func(2): ✅memo에 있음(1) → print 없이 1 반환
│ memo[4]=2+1=3 → 📢print "4 3"
├ func(3): ✅memo에 있음(2) → print 없이 2 반환
memo[5]=3+2=5 → 📢print "5 5"
result=5 → print "결과: 5"
```

**memo 딕셔너리 변화 (📢출력 시점마다)**

| 시점 | memo 상태 | 출력 |
| -- | ------- | -- |
| func(2) 계산 | {2:1} | `2 1` |
| func(1) 계산 | {2:1, 1:1} | `1 1` |
| func(3) 계산 | {2:1,1:1,3:2} | `3 2` |
| (func(2) 재사용) | 변화 없음 | (없음) |
| func(4) 계산 | {...,4:3} | `4 3` |
| (func(3) 재사용) | 변화 없음 | (없음) |
| func(5) 계산 | {...,5:5} | `5 5` |

### ✅ 정답

```
2 1
1 1
3 2
4 3
5 5
결과: 5
```
> 🎯 포인트: func(2)는 처음(2 1) 계산 후 두 번째 호출(func(4) 안)에선 memo 히트라 **print 안 됨**. func(3)도 마찬가지. 그래서 출력이 2,1,3,4,5 순.

## Q11. C언어 — 분할 정복 최댓값 재귀

### 문제

```c
int find(int arr[], int start, int end) {
    if (start == end) return arr[start];
    int mid = (start + end) / 2;
    int res1 = find(arr, start, mid);       // 왼쪽 최댓값
    int res2 = find(arr, mid + 1, end);     // 오른쪽 최댓값
    return (res1 > res2) ? res1 : res2;     // 둘 중 큰 값
}
int data[] = {3, 17, 9, 12, 5, 15, 8, 20};  // 8개
```

### 코드 모든 줄 해석

```c
int find(int arr[], int start, int end) {   // arr의 start~end 구간 최댓값
    if (start == end) return arr[start];     // 원소 1개면 그게 최댓값(재귀 종료)
    int mid = (start + end) / 2;             // 구간 중앙 인덱스
    int res1 = find(arr, start, mid);        // 왼쪽 절반의 최댓값(재귀)
    int res2 = find(arr, mid + 1, end);      // 오른쪽 절반의 최댓값(재귀)
    return (res1 > res2) ? res1 : res2;      // 둘 중 큰 값 반환
}
// data = {3,17,9,12,5,15,8,20} (인덱스 0~7), find(data, 0, 7) 호출
```

### 🤔 이 함수가 하는 일

배열을 **절반씩 계속 쪼개** 원소 1개가 될 때까지 내려간 뒤, 올라오며 두 절반의 최댓값을 비교. (분할 정복 = 나눠서 → 각각 풀고 → 합침)

### 재귀 호출 트리 (인덱스 [start,end] 표기)

```
find[0,7] {3,17,9,12,5,15,8,20}  mid=3
├ find[0,3] {3,17,9,12}  mid=1
│ ├ find[0,1] {3,17}  mid=0
│ │ ├ find[0,0] = 3        (원소1개)
│ │ └ find[1,1] = 17       (원소1개)
│ │ → max(3,17) = 17
│ └ find[2,3] {9,12}  mid=2
│   ├ find[2,2] = 9
│   └ find[3,3] = 12
│   → max(9,12) = 12
│ → max(17,12) = 17          ← 왼쪽 절반 최댓값
└ find[4,7] {5,15,8,20}  mid=5
  ├ find[4,5] {5,15} → max(5,15) = 15
  └ find[6,7] {8,20} → max(8,20) = 20
  → max(15,20) = 20           ← 오른쪽 절반 최댓값
→ max(17, 20) = 20            ← 전체 최댓값
```
> 💡 원소 1개까지 내려갔다가(맨 안쪽), 올라오며 max로 합친다. 결국 전체 최댓값(20)이 나옴.

### ✅ 정답

```
20
```
> 💡 결국 전체 배열의 최댓값(20)을 반환. 분할 정복도 결과는 단순 최댓값.

## Q15. Java — super 체인 + 필드 은닉 심화

### 문제 (요지)

```java
class A { int num=100; A(){ num+=50; } void show(){ print(num+","); } }   // A.num→150
class B extends A { int num=200; int val; B(){ val=num+30; }              // val=200+30=230
    void show(){ print(val+","); } void superShow(){ super.show(); } }
class C extends B { int num=300; int val=400;
    C(){ super(); num+=50; val+=100; }                                    // C.num→350, C.val→500
    void show(){ print(val+","); }
    void display(){ show(); super.show(); super.superShow(); print(num+","); println(super.num); }
}
C c = new C();  c.display();
```

### 필드 초기화 정리

```
A.num: 100 → A() num+=50 → 150
B.num: 200,  B.val: num(=B.num 200)+30 = 230
C.num: 300 → C() num+=50 → 350
C.val: 400 → C() val+=100 → 500
```

### display() 한 줄씩 따라가기

```
show()          → C.show() → C.val = 500 → "500,"
super.show()    → B.show() → B.val = 230 → "230,"
super.superShow() → B.superShow() → super.show() = A.show() → A.num = 150 → "150,"
print(num+",")  → display는 C에 있음 → C.num = 350 → "350,"
println(super.num) → super = B → B.num = 200 → "200"
```

### ✅ 정답

```
500,230,150,350,200
```
> ⚠️ 필드는 정적 타입/접근 위치 기준. super.superShow()가 A.show()까지 내려가 A.num(150)을 출력.

## Q16. Python — 집합 연산

### 문제

```python
A = {1,2,3,4,5}; B = {4,5,6,7,8}; C = {1,8,9}
result1 = A.union(B).intersection(C)          # (A∪B)∩C
result2 = A.difference(B).union(C)            # (A−B)∪C
result3 = (B - C).symmetric_difference(A)     # (B−C)△A
result1.add(3); result1.add(9)
result2.remove(1); result2.discard(10)
result3.add(4); result3.discard(2)
```

### 한 줄씩 따라가기

```
result1 = (A∪B)∩C:
   A∪B = {1,2,3,4,5,6,7,8}
   ∩C({1,8,9}) = {1,8}
   .add(3) → {1,3,8};  .add(9) → {1,3,8,9}
   → sorted = [1,3,8,9]

result2 = (A−B)∪C:
   A−B = {1,2,3}
   ∪C = {1,2,3,8,9}
   .remove(1) → {2,3,8,9};  .discard(10)(없음) → 그대로
   → sorted = [2,3,8,9]

result3 = (B−C)△A:
   B−C = {4,5,6,7}
   △A({1,2,3,4,5}) = 대칭차(양쪽 중 한쪽에만) = {1,2,3,6,7}
      (공통 4,5 제외)
   .add(4) → {1,2,3,4,6,7};  .discard(2) → {1,3,4,6,7}
   → sorted = [1,3,4,6,7]
```

### ✅ 정답

```
result1: [1, 3, 8, 9]
result2: [2, 3, 8, 9]
result3: [1, 3, 4, 6, 7]
```

## Q17. 개체 무결성 위반 원인 — 서술

### 문제

```sql
CREATE TABLE student ( id INT PRIMARY KEY, name VARCHAR(20), grade INT );
INSERT ... VALUES (NULL, 'Chris', 88);   -- ①
INSERT ... VALUES (1, 'David', 92);      -- ② (id=1 이미 존재)
```

### 왜 오류인가? — 개체 무결성 제약

**개체 무결성**: 기본키(PK)는 **NULL 불가 + 중복 불가**.

```
① id=NULL: 기본키는 NULL을 가질 수 없음 → 위반
② id=1 중복: 기본키는 유일해야 함(이미 id=1 존재) → 위반
```

### ✅ 정답 (서술)

① 기본키로 설정된 컬럼은 NULL 값을 허용하지 않는다.
② 기본키 컬럼에는 중복된 값이 삽입될 수 없다(유일해야 한다).

## Q19. C언어 — strstr + strncpy 문자열 조작

### 문제

```c
char arr[] = "AB12CD34EF";
char *p1, *p2, *p3;
char temp[20] = "XYZ";
p1 = &arr[2];                     // "12CD34EF" 시작
p2 = strstr(arr, "12");           // "12"의 위치 = &arr[2]
if(p2) strncpy(temp, p2, 4);      // temp에 4글자 복사
printf("%s\n", temp);             // ①
arr[4] = 'C';                     // arr = "AB12CD34EF" (arr[4]는 원래 'C', 그대로)
p3 = strstr(p1, "CD");            // "CD" 위치
if(p3) strncpy(p3, "789", 2);     // "CD"를 "78"로 교체(2글자)
printf("%s\n", arr);              // ②
strncpy(temp, arr+1, 3);          // temp 앞 3글자를 arr[1..3]로
printf("%s\n", temp);             // ③
```

### 한 줄씩 따라가기

**① temp 갱신**
```
p2 = strstr(arr,"12") → arr의 "12" 위치(인덱스2)
strncpy(temp, p2, 4) → temp 앞 4글자 = "12CD"
temp = "12CD" + 나머지 "XYZ"의 잔여? strncpy(4)는 4글자만 덮음.
  원래 temp="XYZ\0..." → "12CD"로 앞 4칸 덮음 → "12CD"(그 뒤 '\0' 있어 "12CD")
출력 ① → "12CD"
```

**② arr 갱신**
```
arr[4]='C' → arr[4]는 원래 'C'(AB12[C]D...) → 변화 없음, arr="AB12CD34EF"
p3 = strstr(p1,"CD") → p1="12CD34EF"에서 "CD" 위치 = arr의 인덱스4
strncpy(p3, "789", 2) → "CD"를 "78"로 (2글자만) → arr[4]='7', arr[5]='8'
arr = "AB127834EF"
출력 ② → "AB127834EF"
```

**③ temp 재갱신**
```
strncpy(temp, arr+1, 3) → arr[1..3] = "B12" → temp 앞 3글자 덮음
temp 원래 "12CD" → 앞 3글자 "B12"로 → "B12D" (4번째 'D' 남음)
출력 ③ → "B12D"
```

### ✅ 정답

```
12CD
AB127834EF
B12D
```
> ⚠️ strncpy(n)은 n글자만 덮고 널을 자동으로 안 붙임 → 남은 기존 문자가 이어질 수 있음(③의 'D').
