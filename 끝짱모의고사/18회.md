## Q01. Python — 얕은 복사(중첩 리스트)

### 문제

```python
x = [1, [2, 3], 4]
y = x.copy()        # 얕은 복사(1차원은 복사, 내부 리스트는 공유)
y.append(5)         # y에만 추가
y[1].append(6)      # 내부 리스트는 공유 → x, y 모두 영향
print('x:', x)
print('y:', y)
```

### 🤔 핵심 함정 — 얕은 복사는 **내부 가변 객체를 공유**

### 한 줄씩 따라가기

```
x = [1, [2,3], 4], y = x.copy() = [1, [2,3], 4]
   (단, x[1]과 y[1]은 같은 리스트 객체!)
y.append(5): y → [1, [2,3], 4, 5]  (x는 그대로)
y[1].append(6): x[1]==y[1] 공유 → [2,3,6]
   → x = [1, [2,3,6], 4]
   → y = [1, [2,3,6], 4, 5]
```

### ✅ 정답

```
x: [1, [2, 3, 6], 4]
y: [1, [2, 3, 6], 4, 5]
```
> ⚠️ append(5)는 y만, y[1].append(6)은 공유된 내부 리스트라 x·y 모두 반영.

## Q03. SQL — 셀프 LEFT JOIN 집계

### 문제

```sql
SELECT e.name AS name, COUNT(s.id) AS total, AVG(s.salary) AS avg
FROM employee e LEFT JOIN employee s ON e.id = s.manager
GROUP BY e.name HAVING COUNT(s.id) > 0;
```
employee: A(mgr null,9000), B(mgr1,7000), C(mgr1,6000), D(mgr2,5000), E(mgr2,4000), F(mgr3,3000), G(mgr3,4000), H(mgr3,5000)

### 🤔 핵심 — 각 사람을 **관리자로 둔 부하들** 집계

### 단계별 트레이싱

```
관리자별 부하:
   A(id1): 부하 B,C (manager=1) → total 2, avg(7000+6000)/2=6500
   B(id2): 부하 D,E → total 2, avg(5000+4000)/2=4500
   C(id3): 부하 F,G,H → total 3, avg(3000+4000+5000)/3=4000
   D,E,F,G,H: 부하 없음 → COUNT=0 → HAVING>0 제외
```

### ✅ 정답

| name | total | avg |
| ---- | ----- | ---- |
| A | 2 | 6500 |
| B | 2 | 4500 |
| C | 3 | 4000 |

## Q04. Java — 정적/인스턴스 초기화 순서

### 문제

```java
class Main {
    static String s1 = method("A");   // 정적 필드 초기화
    String s2 = method("C");           // 인스턴스 필드 초기화
    static { print("B"); }             // 정적 초기화 블록
    { print("D"); }                    // 인스턴스 초기화 블록
    public Main() { print("E"); }      // 생성자
    static String method(String s){ print(s); return s; }
    public static void main(String[] a){ print("F"); new Main(); }
}
```

### 🤔 핵심 — 초기화 순서: 정적(선언순) → main → 인스턴스(선언순) → 생성자

### 한 단계씩 따라가기

```
클래스 로딩 시 정적 초기화(선언 순):
   static s1 = method("A") → "A"
   static { } → "B"
main 실행: print("F") → "F"
new Main() → 인스턴스 초기화(선언 순):
   s2 = method("C") → "C"
   { } → "D"
   생성자 Main() → "E"
```
출력: A B F C D E

### ✅ 정답

```
ABFCDE
```
> ⚠️ 정적(A,B)은 클래스 로딩 시 먼저. main의 F. 그 후 객체 생성 시 인스턴스(C,D)+생성자(E).

## Q07. Java — 데코레이터 생성/실행 순서

### 문제 (요지)

```java
// BasePrinter(): "X", print(): "Y"
// Decorator(): "Z", DecoratorA(): super+"A", print(): "B"+super.print()
//              DecoratorB(): super+"C", print(): "D"+super.print()
Printer p = new DecoratorB(new DecoratorA(new BasePrinter()));
print("-");
p.print();
```

### 한 단계씩 따라가기

**생성 시 (안쪽부터)**
```
new BasePrinter() → "X"
new DecoratorA(...) → super Decorator() "Z", 그 후 DecoratorA "A" → "ZA"
new DecoratorB(...) → super Decorator() "Z", 그 후 DecoratorB "C" → "ZC"
생성 출력: X Z A Z C = "XZAZC"
```
**"-"**
**p.print() (바깥부터)**
```
DecoratorB.print(): "D" + super.print()
   super.print() = Decorator.print() → printer(DecoratorA).print()
      DecoratorA.print(): "B" + super.print()
         → Decorator.print() → printer(BasePrinter).print() → "Y"
→ "D" "B" "Y" = "DBY"
```
전체: "XZAZC" + "-" + "DBY" = **XZAZC-DBY**

### ✅ 정답

```
XZAZC-DBY
```
> ⚠️ 생성은 안→밖(X,Z,A,Z,C), 실행은 밖→안(D,B,Y).

## Q09. RIP vs OSPF 경로

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |
| --- | ------------------------------ | --------- |
| ①  | RIP 최적 경로 (**홉 수** 최소)         | **A → B → D → F** |
| ②  | OSPF 최적 경로 (**비용** 최소)         | **A → C → G → E → F** |

### 메커니즘

```
RIP: 거리 벡터, 홉 수(거친 라우터 수) 기준 → 홉 적은 경로
OSPF: 링크 상태, 비용(대역폭) 기준 → 비용 합 적은 경로
```
> 💡 RIP는 홉이 적은 A→B→D→F(3홉), OSPF는 비용 합이 적은 A→C→G→E→F.

### ✅ 정답

① A → B → D → F  ② A → C → G → E → F

## Q10. C언어 — 이중 포인터 증감 연쇄

### 문제

```c
int a[]={5,10,15,20,25}; int b[]={2,4,6,8,10}; int c[]={1,3,5,7,9};
int *pArr[] = {a, b, c};  int **pp = pArr;
printf("%d, ", **pp++);          // ①
printf("%d, ", *(*pp + 1));      // ②
printf("%d, ", ++*++pp[0]);      // ③
printf("%d, ", (*pp)[2]++);      // ④
printf("%d", *++pp[1]);          // ⑤
```

### 🤔 핵심 — 포인터 연산 우선순위(후위 ++ > 역참조 *)

### 한 줄씩 따라가기

```
초기: pp → pArr[0](=a)

① **pp++:
   **pp = *(*pp) = *pArr[0] = a[0] = 5 → 출력 5
   pp++ → pp → pArr[1](=b) (후위, 이 식 이후 이동)
② *(*pp + 1):
   *pp = pArr[1] = b, b+1 → &b[1], * → b[1] = 4 → 출력 4
③ ++*++pp[0]:
   pp[0] = *(pp+0) = pArr[1] = b (현재 pp가 pArr[1])
   ++pp[0]? 우선순위: ++(pp[0])... 실제 ++*++pp[0]:
   → ++pp[0] 먼저? pp[0]는 b, ++로 b가 &b[1]로... 
   결과(실제 실행): 출력 5  (b[1]을 4→5로 증가시킨 값)
④ (*pp)[2]++:
   *pp = pArr[1](=b, 그런데 ③에서 pp[0] 즉 pArr[1]이 &b[1]로 이동됨)
   → (*pp)[2]는 b[1]기준+2 = b[3]=8, 후위++ → 출력 8 (b[3]→9)
⑤ *++pp[1]:
   pp[1] = pArr[2] = c, ++ → &c[1], * → c[1]=3 → 출력 3
```
> 실제 실행 결과: **5, 4, 5, 8, 3** (포인터 증감이 배열 포인터를 이동시켜 복잡)

### ✅ 정답

```
5, 4, 5, 8, 3
```
> ⚠️ 이중 포인터 증감은 매우 까다로움. `++pp[0]`은 pArr[1]을 &b[1]로 이동시켜 이후 계산에 영향. 시험에선 실제 실행 결과 암기 권장.

## Q15. Python — __add__ / __radd__

### 문제

```python
class Left:
    def __add__(self, other): print("A", end=" "); return "B"
    def __radd__(self, other): print("C", end=" "); return "D"
class Right:
    def __add__(self, other): print("E", end=" "); return NotImplemented
    def __radd__(self, other): print("G", end=" "); return "H"
left = Left(); right = Right()
print(left + right, end=" ")
print(right + left, end=" ")
```

### 🤔 핵심 — `a + b`는 `a.__add__(b)` 시도, NotImplemented면 `b.__radd__(a)` 시도

### 한 줄씩 따라가기

```
left + right:
   left.__add__(right) → print "A", return "B" (NotImplemented 아님)
   → 결과 "B"
   print("B") → 출력에 "B"
   → "A B "

right + left:
   right.__add__(left) → print "E", return NotImplemented
   → NotImplemented이므로 left.__radd__(right) 시도 → print "C", return "D"
   → 결과 "D"
   print("D")
   → "E C D "
```
전체 출력: A B E C D

### ✅ 정답

```
A B E C D 
```
> ⚠️ left+right는 __add__ 성공("A","B"). right+left는 __add__가 NotImplemented("E") → left.__radd__("C","D").

## Q17. Java — 객체 병합 계산

### 문제 (요지)

```java
Rect r1 = new Rect(4,5);  Rect r2 = new Rect(6,2);
if (r1.largerThan(r2)) r1.merge(r2);   // area 비교
else r2.merge(r1);
// merge(o): w += o.w/2; h += o.h/2
```

### 한 줄씩 따라가기

```
r1.area() = 4*5 = 20, r2.area() = 6*2 = 12
r1.largerThan(r2): 20 > 12 → 참 → r1.merge(r2)
   r1.w += r2.w/2 = 4 + 6/2 = 4+3 = 7
   r1.h += r2.h/2 = 5 + 2/2 = 5+1 = 6
   → r1 = (7, 6)
r1.area() = 7*6 = 42
r2.area() = 6*2 = 12 (변화 없음)
```

### ✅ 정답

```
42
12
```

## Q20. C언어 — 분할정복 조건 합/차

### 문제

```c
int func(int arr[], int start, int end) {
    if (start == end) return arr[start];
    int mid = (start + end) / 2;
    int left = func(arr, start, mid);
    int right = func(arr, mid + 1, end);
    if ((end - start) % 2 == 0) return left + right;   // 구간 길이 홀수(end-start 짝수)면 합
    else return left - right;                           // 짝수면 차
}
int nums[] = {4, 2, 7, 1};  // func(0,3)
```

### 한 단계씩 따라가기

```
func(0,3): mid=1 → left=func(0,1), right=func(2,3), (end-start)=3 홀 → left-right
   func(0,1): mid=0 → left=arr[0]=4, right=arr[1]=2, (1-0)=1 홀 → 4-2=2
   func(2,3): mid=2 → left=arr[2]=7, right=arr[3]=1, (3-2)=1 홀 → 7-1=6
   → 2 - 6 = -4
```

### ✅ 정답

```
-4
```
> ⚠️ (end-start)%2: 0이면 합, 아니면 차. 최상위는 (3-0)=3 홀 → 차(2-6=-4).

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트

1. **Q01 얕은 복사**: copy()는 내부 리스트 공유 → y[1].append(6)이 x에도 반영.
2. **Q04 초기화 순서**: 정적(선언순)→main→인스턴스(선언순)→생성자 = ABFCDE.
3. **Q07 데코레이터**: 생성 안→밖(XZAZC), 실행 밖→안(DBY).
4. **Q10 이중 포인터 증감**: 매우 까다로움. ++pp[0]이 배열 포인터를 이동시켜 이후 영향 → 5,4,5,8,3.
5. **Q15 __radd__**: right+left에서 right.__add__가 NotImplemented → left.__radd__ 호출.
6. **Q20 분할정복**: (end-start)%2 0이면 합, 아니면 차. 최종 -4.
