## Q01. C언어 — 2의 보수 + XOR 연쇄

### 문제

```c
int func(int a, int b){ return ((~a + 1) & 0xFF) ^ ((~b + 1) & 0xFF); }
int main(){
    char m = -32;  unsigned char n = 127;  int k = 16;
    m = ~m;        // ~(-32) = 31
    n = ~n;        // ~127 = 128 (unsigned char)
    int res1 = func(m, n);
    int res2 = func(res1, k);
    printf("%d", res2);
}
```

### 🤔 핵심 — `~val+1`=2의 보수(-val), `&0xFF`=하위 8비트

### 한 단계씩 따라가기

```
m = ~(-32) = 31
n = ~127 = 128 (unsigned char: 01111111 → 10000000 = 128)

func(m=31, n=128):
   (~31+1)&0xFF = (-31)&0xFF = 225   (256-31)
   (~128+1)&0xFF = (-128)&0xFF = 128
   225 ^ 128 = 11100001 ^ 10000000 = 01100001 = 97
res1 = 97

func(res1=97, k=16):
   (~97+1)&0xFF = (-97)&0xFF = 159   (256-97)
   (~16+1)&0xFF = (-16)&0xFF = 240   (256-16)
   159 ^ 240 = 10011111 ^ 11110000 = 01101111 = 111
res2 = 111
```

### ✅ 정답

```
111
```
> ⚠️ 2의 보수(-val)를 &0xFF로 하위 8비트만. XOR은 이진수로 계산.

## Q03. SQL — JOIN + 서브쿼리 + 정렬

### 문제

```sql
SELECT d.dept_name, d.location, COUNT(e.emp_id) AS cnt, AVG(e.salary) AS avg
FROM department d JOIN employee e ON d.dept_id = e.dept_id
WHERE d.location IN ('서울', '부산')
AND e.salary > (SELECT AVG(salary) FROM employee WHERE dept_id IN ('D01', 'D02'))
GROUP BY d.dept_name, d.location
HAVING COUNT(e.emp_id) >= 1
ORDER BY avg DESC;
```
employee: 김철수(D01,5000), 이영희(D02,6000), 박민수(D01,7000), 정혜진(D03,4500), 강태호(D02,8000)

### 단계별 트레이싱

```
서브쿼리 AVG(D01,D02 급여) = (5000+6000+7000+8000)/4 = 26000/4 = 6500
조건: 위치 서울/부산 AND 급여 > 6500
   김철수 D01 서울 5000 → 급여 미달 ✗
   이영희 D02 부산 6000 → 미달 ✗
   박민수 D01 서울 7000 → ✓
   정혜진 D03 개발부(서울) 4500 → 미달 ✗
   강태호 D02 부산 8000 → ✓
그룹화: 영업부(서울) 박민수 → cnt1, avg7000 / 인사부(부산) 강태호 → cnt1, avg8000
정렬 avg DESC: 인사부(8000), 영업부(7000)
```

### ✅ 정답

| dept_name | location | cnt | avg |
| --------- | -------- | --- | ---- |
| 인사부 | 부산 | 1 | 8000 |
| 영업부 | 서울 | 1 | 7000 |

## Q04. 조건-결정 커버리지 데이터 선택

### 문제

```c
if ((x < 10 || y == 20) && z != 0) result = result/2;
else result = result+10;
```

### 🤔 핵심 — 조건-결정 커버리지: 각 개별 조건 T/F 모두 + 전체 결정 T/F 모두

개별 조건: `x<10`, `y==20`, `z!=0`. 전체 결정도 T/F 필요.

### 보기 ③ 검증: (5,10,1), (15,20,0)

```
(x=5,y=10,z=1):
   x<10=T, y==20=F, z!=0=T
   결정: (T||F)&&T = T
(x=15,y=20,z=0):
   x<10=F, y==20=T, z!=0=F
   결정: (F||T)&&F = F
개별 조건 커버:
   x<10: T(5), F(15) ✓
   y==20: F(10), T(20) ✓
   z!=0: T(1), F(0) ✓
전체 결정: T, F 모두 ✓
→ 조건-결정 커버리지 만족
```

### ✅ 정답

③

## Q05. Java — 오버라이딩 vs 필드/정적 은닉

### 문제 (요지)

```java
class A { String val="A"; void m(){println("A");} static void sm(){println("SA");} }
class B extends A { String val="B"; void m(){println("B");} static void sm(){println("SB");} }
class C extends B { String val="C"; static void sm(){println("SC");} }
A obj = new C();
obj.m();  System.out.println(obj.val);  obj.sm();
```

### 한 줄씩 따라가기

```
A obj = new C(): obj 선언타입 A, 실제타입 C
obj.m(): 인스턴스 메서드 → 실제타입 C. C는 m() 오버라이드 안 함 → 상속받은 B.m() → "B"
obj.val: 필드 → 선언타입 A → A.val = "A" → "A"
obj.sm(): 정적 메서드 → 선언타입 A → A.sm() → "SA"
```

### ✅ 정답

```
B
A
SA
```
> ⚠️ m()은 오버라이딩(실제타입 C→상속된 B.m()), val/sm()은 선언타입 A 기준.

## Q06. Python — 분할정복 곱

### 문제

```python
def multiply(arr, left, right):
    if left == right: return arr[left]
    mid = (left + right) // 2
    return multiply(arr, left, mid) * multiply(arr, mid+1, right)
numbers = [2, -3, 4, -1]
```

### 🤔 이 함수가 하는 일

배열을 반씩 나눠 전체 곱을 계산 (분할정복).

### 한 단계씩 따라가기

```
multiply(0,3): mid=1 → multiply(0,1) * multiply(2,3)
   multiply(0,1): mid=0 → arr[0](2) * arr[1](-3) = -6
   multiply(2,3): mid=2 → arr[2](4) * arr[3](-1) = -4
   → -6 * -4 = 24
```

### ✅ 정답

```
24
```

## Q10. Python — 짝수 앞으로 재배열

### 문제

```python
def update(arr):
    temp = []
    for i in range(len(arr)):
        if arr[i] % 2 == 0: temp.insert(0, arr[i])   # 짝수 → 맨 앞
        else: temp.append(arr[i])                     # 홀수 → 맨 뒤
    for i in range(len(arr)): arr[i] = temp[i]
    return arr
nums = [1, 2, 3, 4, 5]
```

### 한 원소씩 따라가기

```
temp=[]

2(짝): insert(0) → [2, 1]
3(홀): append → [2, 1, 3]
4(짝): insert(0) → [4, 2, 1, 3]
5(홀): append → [4, 2, 1, 3, 5]
```

### ✅ 정답

```
[4, 2, 1, 3, 5]
```
> ⚠️ 짝수는 앞에 삽입되어 역순(4,2), 홀수는 뒤에 추가되어 순서 유지(1,3,5).

## Q11. Java — 제네릭 소거 + 오버로딩 선택

### 문제 (요지)

```java
class Processor {
    void process(Object obj){print("X");}
    void process(Double obj){print("Y");}
    void process(Integer obj){print("Z");}
}
class Wrapper<T> { T item; ... void execute(){ new Processor().process(item); } }
Wrapper w1 = new Wrapper(10);
Wrapper<Integer> w2 = new Wrapper<>(20);
Wrapper<Double> w3 = new Wrapper<>(10.5);
w1.execute(); w2.execute(); w3.execute();
```

### 🤔 핵심 함정 — 제네릭 타입 소거(erasure): `T`는 컴파일 시 **Object**로 처리

`execute()` 안에서 item의 정적 타입은 T = Object → 오버로딩은 **컴파일 시점 정적 타입**으로 결정 → 항상 `process(Object)` = "X".

### 한 줄씩 따라가기

```
w1.execute(): item 타입 T(=Object) → process(Object) → "X"
w2.execute(): item 타입 T(=Object) → process(Object) → "X"
w3.execute(): item 타입 T(=Object) → process(Object) → "X"
```

### ✅ 정답

```
XXX
```
> ⚠️ 제네릭은 타입 소거로 T가 Object가 됨. 런타임 실제 타입(Integer/Double)과 무관하게 process(Object) 선택.

## Q15. Python — filter/map/reduce 체인

### 문제

```python
data = [1,2,...,10]
filtered = filter(lambda x: x % 2 != 0, data)         # 홀수
mapped = map(lambda x: x**2 if x < 7 else x-2, filtered)  # <7이면 제곱, 아니면 -2
result = reduce(lambda acc, x: acc + (x if x % 3 else -x), mapped, 0)  # 3배수면 -x
```

### 한 단계씩 따라가기

```
filtered(홀수): 1,3,5,7,9
mapped:
   1<7 → 1²=1
   3<7 → 9
   5<7 → 25
   7≥7 → 7-2=5
   9≥7 → 9-2=7
   → [1, 9, 25, 5, 7]
reduce(acc + (x if x%3 else -x), 초기 0):
   x=1: 1%3≠0 → +1 → acc=1
   x=9: 9%3==0 → -9 → acc=-8
   x=25: 25%3≠0 → +25 → acc=17
   x=5: 5%3≠0 → +5 → acc=22
   x=7: 7%3≠0 → +7 → acc=29
```

### ✅ 정답

```
29
```
> ⚠️ 9만 3의 배수라 -9. 나머지는 그대로 더함.

## Q18. Java — 복합 증감 + 단락 평가

### 문제

```java
int a=5, b=10, c=3;
boolean flag;
int temp = ++a * b-- + c * (--b + a--) - ++c;
flag = (temp > 90 && ++a < b--) || (--c + a > 5 && temp < 100);
```

### 한 단계씩 따라가기 — temp

```
초기: a=5, b=10, c=3
temp = ++a * b-- + c * (--b + a--) - ++c
   ++a → a=6, 값6
   b-- → 값10, b→9
   6 * 10 = 60
   c=3 (아직)
   (--b + a--): --b → b=8, 값8;  a-- → 값6, a→5;  8+6=14
   c * 14 = 3 * 14 = 42
   60 + 42 = 102
   ++c → c=4, 값4
   102 - 4 = 98
temp = 98
상태: a=5, b=8, c=4
```

### flag

```

   → || 단락(우항 평가 안 함)

```

## Q20. C언어 — 하노이 탑 재귀

### 문제

```c
void hanoi(int n, char from, char temp, char to) {
    if (n == 0) return;
    hanoi(n-1, from, to, temp);
    printf("%d: %c - %c\n", n, from, to);
    hanoi(n-1, temp, from, to);
}
// hanoi(3, 'A', 'B', 'C')
```

### 🤔 이 함수가 하는 일

원반 n개를 from에서 to로 옮기는 하노이 탑. n-1개를 보조로 옮기고, n번 옮기고, n-1개를 목적지로.

### 한 단계씩 따라가기 (3개)

```
hanoi(3,A,B,C):
   hanoi(2,A,C,B):
      hanoi(1,A,B,C): hanoi(0)무시, print "1: A - C", hanoi(0)무시
      print "2: A - B"
      hanoi(1,C,A,B): print "1: C - B"
   print "3: A - C"
   hanoi(2,B,A,C):
      hanoi(1,B,C,A): print "1: B - A"
      print "2: B - C"
      hanoi(1,A,B,C): print "1: A - C"
```

### ✅ 정답

```
1: A - C
2: A - B
1: C - B
3: A - C
1: B - A
2: B - C
1: A - C
```
> 💡 원반 3개 → 2³−1 = 7번 이동.
