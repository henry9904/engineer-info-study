## Q01. 트랜잭션 — 원자성 / COMMIT / ROLLBACK

### 지문 키워드

> 은행에서 A 출금 + B 입금을 **하나의 트랜잭션**으로 처리
> ① 모두 성공 or 모두 실패 보장 특성  ② 결과를 **영구 반영**하는 명령어  ③ **이전 상태로 되돌리는** 명령어

### 왜 이게 답인가?

| 빈칸 | 지문 핵심 키워드          | 정답            |
| --- | ------------------ | ------------- |
| ①  | 모두 성공 or 모두 실패     | **원자성(Atomicity)** |
| ②  | 결과를 영구 반영          | **COMMIT**    |
| ③  | 이전 상태로 되돌림         | **ROLLBACK**  |

### 메커니즘 — 트랜잭션 ACID

| 특성                | 의미            |
| ----------------- | ------------- |
| **A**tomicity(원자성) | 전부 반영 or 전부 취소 |
| **C**onsistency(일관성) | 무결성 유지     |
| **I**solation(격리성) | 동시 실행 간섭 없음  |
| **D**urability(지속성) | 커밋 결과 영구 보존 |

> ⚠️ 원자성 위반 상황(출금만 됨) → ROLLBACK으로 출금까지 취소해야 원자성 보장.

### ✅ 정답

① 원자성(Atomicity)  ② COMMIT  ③ ROLLBACK

## Q03. C언어 — 소수 판별 + 삼항 증가

### 문제

```c
int temp = 2, flag;
while (temp < 8) {
    flag = 2;
    while (flag < temp && temp % flag != 0) flag++;   // 나눠지는 수 탐색
    if (flag == temp || temp == 2) printf("%d ", temp); // 소수면 출력
    temp += (temp % 3 == 0) ? 1 : 2;                    // 3배수면 +1, 아니면 +2
}
```

### 🤔 이 코드가 하는 일

`temp`가 소수면 출력. 안쪽 while로 2부터 나눠보다가 `flag==temp`까지 가면 소수. 증가폭은 3배수 여부로 달라짐.

### 한 줄씩 따라가기

**temp=2**
```
flag=2
안쪽 while: flag(2) < temp(2)? → 거짓, 바로 종료. flag=2
if: flag(2)==temp(2)? O 또는 temp==2? O → 출력 "2"
증가: 2%3≠0 → +2 → temp=4
```
**temp=4**
```
flag=2
안쪽 while: flag(2)<4 참 && 4%2!=0? → 4%2==0이라 거짓 → 종료. flag=2
if: flag(2)==4? X, temp==2? X → 출력 안 함
증가: 4%3≠0 → +2 → temp=6
```
**temp=6**
```
flag=2
안쪽 while: 2<6 참 && 6%2!=0? → 6%2==0 거짓 → 종료. flag=2
if: 2==6? X → 출력 안 함
증가: 6%3==0 → +1 → temp=7
```
**temp=7**
```
flag=2
안쪽 while:
  flag=2: 2<7 && 7%2!=0(참) → flag=3
  flag=3: 3<7 && 7%3!=0(참) → flag=4
  flag=4: 4<7 && 7%4!=0(참) → flag=5
  flag=5: 5<7 && 7%5!=0(참) → flag=6
  flag=6: 6<7 && 7%6!=0(참) → flag=7
  flag=7: 7<7? 거짓 → 종료. flag=7
if: flag(7)==temp(7)? O → 출력 "7"
증가: 7%3≠0 → +2 → temp=9
```
**temp=9** → `9 < 8` 거짓 → 루프 종료

### ✅ 정답

```
2 7 
```
> ⚠️ 4·6은 flag=2에서 바로 나눠져 flag가 2에 머묾(소수 아님). 7만 끝까지 돌아 flag=7(소수).

## Q04. Java — 생성자 순서 + 오버라이딩 + 필드 초기화 시점

### 문제

```java
class Parent {
    public Parent() { System.out.println("1"); display(); }   // ← 여기가 함정
    public void display() { System.out.println("Super"); }
}
class Child extends Parent {
    private String info = "Child Info";
    private int count = 100;
    public Child() { System.out.println("2"); }
    public void display() { System.out.println(info); System.out.println(count); }
}
Parent obj1 = new Parent();
Parent obj2 = new Child();
```

### 🤔 이 문제의 3가지 함정

1. 생성자 속 `display()`는 **실제 객체가 Child면 Child.display()가 실행**(동적 바인딩).
2. 객체 생성 순서: **① super() → ② 자식 필드 초기화 → ③ 자식 생성자 본문**. 부모 생성자 시점엔 자식 필드가 **아직 초기화 전**(info=null, count=0).
3. obj1(진짜 Parent)과 obj2(Child)는 다르게 동작.

### 한 줄씩 따라가기

**`obj1 = new Parent()`**
```
Parent() 진입 → println("1")
display() 호출 → obj1 실제타입 Parent → Parent.display() → "Super"
```
출력: `1` / `Super`

**`obj2 = new Child()`**
```
Child() 호출 → 먼저 super() = Parent() 실행
  Parent() 진입 → println("1")
  display() 호출 → obj2 실제타입 Child → Child.display() 실행
     println(info)  → info 아직 초기화 전 → null
     println(count) → count 아직 초기화 전 → 0
  (Parent 생성자 끝)
  이제 Child 필드 초기화: info="Child Info", count=100
  Child 생성자 본문: println("2")
```
출력: `1` / `null` / `0` / `2`

### ✅ 정답

```
1
Super
1
null
0
2
```
> ⚠️ 부모 생성자가 돌 때 자식 필드는 아직 null/0 → "Child Info/100" 아님.

## Q05. SQL — GROUP BY + HAVING + 집계함수

### 문제 (요구 조건)

- GROUP BY·HAVING 필수, WHERE 금지, 별칭 필수, 집계함수 필수
- **최고−최저 ≥ 10** 인 과목만 출력

### 왜 이렇게 쓰나?

| 조건        | 구현                              |
| --------- | ------------------------------- |
| 과목별 묶기    | `GROUP BY 과목이름`               |
| 최고/최저     | `MAX(점수)`, `MIN(점수)`          |
| 차 10 이상   | `HAVING MAX(점수)-MIN(점수) >= 10` |

> ⚠️ "차 10 이상"은 집계 결과 조건 → **HAVING**(WHERE 아님). WHERE=그룹화 전 개별행, HAVING=그룹화 후 집계값.

### 정답 SQL

```sql
SELECT 과목이름,
       MAX(점수) AS 최고점수,
       MIN(점수) AS 최저점수
FROM GRADE
GROUP BY 과목이름
HAVING MAX(점수) - MIN(점수) >= 10;
```
검산: JAVA(85,88,91) 차 6→제외 / DB(92,97,83) 차 14→포함 / OS(75,60) 차 15→포함

### ✅ 정답 → 위 SQL

## Q06. 서브넷 비트 수 계산

### 문제

> 10.0.0.0/8, 최소 **200개 서브넷**, 각 서브넷 **1,000대 이상**. 최소 서브넷 비트 수?

### 핵심 개념 — 비트 배분

`/8`이라 호스트로 쓸 비트 = 32−8 = **24비트**. 이를 서브넷(s) + 호스트(h)로 분배.

```
① 호스트 조건: 2^h - 2 ≥ 1000
   h=9 → 510 (부족),  h=10 → 1022 (충족) → h=10
② 서브넷 조건: 2^s ≥ 200
   s=7 → 128 (부족),  s=8 → 256 (충족) → s=8
③ 확인: s+h = 8+10 = 18 ≤ 24 ✅
```

### ✅ 정답

```
8
```
> ⚠️ 묻는 건 **서브넷 비트 s=8** (전체 프리픽스로는 /16).

## Q08. Python — isinstance / type + bool 함정

### 문제

```python
def check(v):
    if isinstance(v, str):     return len(v)
    elif isinstance(v, bool):  return 10
    elif type(v) == int:       return 1
    elif type(v) == float:     return 2
    elif isinstance(v, (list, tuple)): return len(v) * 2
    elif v is None:            return 0
    else:                      return -1
a="AI"; b=True; c=3.14; d=[1,2,3]; e=(10,20); f=None
print(check(a)+check(b)+check(c)+check(d)+check(e)+check(f))
```

### 🤔 핵심 함정 — bool은 int의 하위 클래스

`isinstance(True,int)`도 True지만, 분기 순서상 `isinstance(v,bool)`이 `type(v)==int`보다 **위**라 bool이 먼저 걸림.

### 한 값씩 따라가기

```
check("AI"):    isinstance str → len("AI")=2
check(True):    isinstance str? X → isinstance bool? O → 10
check(3.14):    str?X bool?X type==int?X type==float?O → 2
check([1,2,3]): ... isinstance (list,tuple)? O → len*2 = 3*2 = 6
check((10,20)): isinstance (list,tuple)? O → 2*2 = 4
check(None):    ... v is None? O → 0
합계 = 2 + 10 + 2 + 6 + 4 + 0 = 24
```

### ✅ 정답

```
24
```

## Q11. CPM — 임계경로 + ES / LS / Slack

### 문제

| 작업 | 선행작업 | 소요 기간 |
| --- | ---- | ----- |
| A  | -    | 12    |
| B  | A    | 15    |
| C  | A    | 14    |
| D  | B, C | 16    |
| E  | D    | 13    |
| F  | C    | 11    |

### 전진 계산(ES/EF)

```
ES = 선행 EF의 최댓값,  EF = ES + 기간
A: ES0  EF12
B: ES12 EF27
C: ES12 EF26
D: 선행 B(27),C(26) 중 max=27 → ES27 EF43
E: ES43 EF56
F: 선행 C(26) → ES26 EF37
프로젝트 종료 = max EF = 56 (E)
```

### 임계경로

```
가장 긴 경로 = A→B→D→E = 12+15+16+13 = 56 (Slack 0)
```

### F 작업 후진 계산

```
F는 후행 없음 → LF = 종료일 56
LS(F) = LF - 기간 = 56 - 11 = 45
Slack(F) = LS - ES = 45 - 26 = 19
```

### ✅ 정답

① A → B → D → E  ② ES=26  ③ LS=45  ④ Slack=19

## Q14. C언어 — 배열 노드 값 swap + 포인터 연결

### 문제

```c
struct Node { int value; struct Node* next; };
void swapPairs(struct Node arr[], int size) {
    for (int i = 0; i + 1 < size; i += 2) {   // 0,2,4... 인접 쌍
        int temp = arr[i].value;
        arr[i].value = arr[i + 1].value;
        arr[i + 1].value = temp;              // value만 교환 (next는 그대로)
    }
}
int main() {
    struct Node nodes[3] = {
        {1, &nodes[2]},   // nodes[0]: value=1, next→nodes[2]
        {2, NULL},        // nodes[1]: value=2, next→NULL
        {3, &nodes[1]}    // nodes[2]: value=3, next→nodes[1]
    };
    swapPairs(nodes, 3);
    struct Node* curr = &nodes[0];
    while (curr != NULL) { printf("%d ", curr->value); curr = curr->next; }
}
```

### 🤔 이 함수가 하는 일

`swapPairs`는 **인접한 쌍의 value만 교환**한다. next 포인터(연결 구조)는 절대 건드리지 않음. 출력은 배열 순서가 아니라 **next를 따라간 순서**.

### 1단계 — 초기 상태 (배열 3칸)

```
nodes[0]: value=1, next→nodes[2]
nodes[1]: value=2, next→NULL
nodes[2]: value=3, next→nodes[1]
```

### 2단계 — swapPairs 한 줄씩

```
for i=0 (i+1=1 < 3 → 실행):
    temp = arr[0].value = 1
    arr[0].value = arr[1].value = 2     → nodes[0].value = 2
    arr[1].value = temp = 1             → nodes[1].value = 1
i += 2 → i=2:
for i=2 (i+1=3 < 3? 거짓 → 루프 종료)
    → nodes[2].value = 3 그대로
```
swap 후 value: nodes[0]=**2**, nodes[1]=**1**, nodes[2]=**3**
(next 연결은 변하지 않음)

### 3단계 — while로 next 따라 출력

```
curr = &nodes[0]  → value=2 출력 → curr = next = &nodes[2]
curr = &nodes[2]  → value=3 출력 → curr = next = &nodes[1]
curr = &nodes[1]  → value=1 출력 → curr = next = NULL
curr == NULL → 종료
```

### 연결 순서 시각화

```
출력 경로:  nodes[0] ──next──▶ nodes[2] ──next──▶ nodes[1] ──next──▶ NULL
value:         2                  3                  1
```

### ✅ 정답

```
2 3 1 
```
> ⚠️ 배열 인덱스 순서(2 1 3)가 아니라 **next 연결 순서(2 3 1)**로 출력. value만 바뀌고 연결은 그대로인 게 핵심.

## Q15. C언어 — 포인터 산술 도수분포 + 최빈값

### 문제

```c
#define SIZE 11
int analyze(int* arr, int len) {
    int temp[11] = {0};
    for (int i = 0; i < len; i++) (*(temp + *(arr + i)))++;  // temp[arr[i]]++
    int peak = 0;
    for (int i = 0; i < 11; i++) if (*(temp + i) > peak) peak = *(temp + i);
    printf("결과 : ");
    for (int i = 0; i < 11; i++) if (*(temp + i) == peak) printf("%d ", i);
    printf("\n");
    return peak;
}
int main() {
    int source[SIZE] = {1, 2, 3, 2, 2, 3, 3, 3, 1, 4, 2};
    analyze(source, SIZE);
}
```

### 🤔 포인터 산술 해석

`(*(temp + *(arr+i)))++` = `temp[arr[i]]++` → **등장한 숫자를 인덱스로 세는 도수분포표**.

### 1단계 — 도수 세기 (한 원소씩)

source = {1,2,3,2,2,3,3,3,1,4,2}

```
1 → temp[1]=1
2 → temp[2]=1
3 → temp[3]=1
2 → temp[2]=2
2 → temp[2]=3
3 → temp[3]=2
3 → temp[3]=3
3 → temp[3]=4
1 → temp[1]=2
4 → temp[4]=1
2 → temp[2]=4
```
→ temp = [0, 2, 4, 4, 1, 0,...]  (값1:2회, 값2:4회, 값3:4회, 값4:1회)

### 2단계 — peak(최대 빈도) 찾기

```
temp 순회하며 최댓값 → peak = 4  (값 2와 3이 각각 4회)
```

### 3단계 — 빈도가 peak(4)인 인덱스 출력

```
temp[2]==4 → "2 " 출력
temp[3]==4 → "3 " 출력
```

### ✅ 정답

```
결과 : 2 3 
```
> 💡 최빈값이 공동일 때 모두 출력. 값 2, 3이 각 4회로 공동 최빈.

## Q16. Java — String `==` vs `equals` + `intern()`

### 문제

```java
String[] arr1 = {"apple", "banana", new String("apple"), "Banana".toLowerCase()};
String[] arr2 = new String[4];
arr2[0] = "apple";
arr2[1] = "banana";
arr2[2] = arr1[2].intern();
arr2[3] = new String("banana");
for (int i = 0; i < 4; i++) {
    System.out.print((arr1[i] == arr2[i]) + ",");
    System.out.println((arr1[i].equals(arr2[i])));
}
```

### 🤔 핵심 개념

- 리터럴 `"apple"`은 **String Pool**에 저장 → 같은 리터럴은 같은 객체.
- `new String(...)`은 **Heap**에 새 객체 → 리터럴과 다른 객체.
- `.intern()`은 같은 내용의 **Pool 객체**를 반환.
- `.toLowerCase()`처럼 런타임 생성 문자열은 Pool이 아닌 **새 객체**.
- `==`는 참조(주소) 비교, `.equals()`는 내용 비교.

### 한 쌍씩 따라가기

```
i=0: arr1[0]="apple"(Pool)  vs arr2[0]="apple"(Pool)
     == : 같은 Pool 객체 → true   / equals : 내용 같음 → true
i=1: arr1[1]="banana"(Pool) vs arr2[1]="banana"(Pool)
     == : true   / equals : true
i=2: arr1[2]=new String("apple")(Heap) vs arr2[2]=arr1[2].intern()(Pool)
     == : Heap ≠ Pool → false   / equals : true
i=3: arr1[3]="Banana".toLowerCase()(런타임 Heap) vs arr2[3]=new String("banana")(Heap)
     == : 서로 다른 Heap 객체 → false   / equals : true
```

### ✅ 정답

```
true,true
true,true
false,true
false,true
```
> ⚠️ equals는 4개 모두 true(내용 동일). `==`만 Pool 공유 여부에 따라 다름.

## Q19. C언어 — 연결 리스트 노드 건너뛰기 삭제

### 문제

```c
typedef struct Node { int data; struct Node* next; } Node;
void ins_node(Node** head, int data) {         // 맨 앞 삽입
    Node* newNode = malloc(sizeof(Node));
    newNode->data = data;
    newNode->next = *head;
    *head = newNode;
}
void del_node(Node* head) {
    if (head == NULL) return;
    Node* prev = head;
    Node* now = head->next;
    while (prev != NULL && now != NULL) {
        prev->next = now->next;   // now 건너뛰기
        free(now);
        prev = prev->next;        // 다음 살아있는 노드로
        if (prev != NULL) now = prev->next;
    }
}
int main() {
    Node* head = NULL;
    for (int i = 1; i <= 5; i++) ins_node(&head, i);
    del_node(head);
    printList(head);
}
```

### 1단계 — 리스트 구성 (앞쪽 삽입 → 역순)

`ins_node`는 맨 앞에 삽입하므로 1,2,3,4,5를 넣으면 역순:
```
ins 1: head→1
ins 2: head→2→1
ins 3: head→3→2→1
ins 4: head→4→3→2→1
ins 5: head→5→4→3→2→1
```
→ `5 → 4 → 3 → 2 → 1 → NULL`

### 2단계 — del_node 한 줄씩 (한 칸 걸러 삭제)

```
초기: prev=5, now=4
반복1: prev->next = now->next  → 5의 next를 3으로 (4 건너뜀)
       free(4)
       prev = prev->next = 3
       now = prev->next = 2
       현재 리스트: 5→3→2→1
반복2: prev(3)->next = now(2)->next → 3의 next를 1로 (2 건너뜀)
       free(2)
       prev = prev->next = 1
       now = prev->next = NULL
       현재 리스트: 5→3→1
반복3: while(prev && now) → now==NULL → 종료
```
→ 남은 리스트: `5 → 3 → 1 → NULL`

### ✅ 정답

```
5 3 1 
```
> ⚠️ 두 함정: (1) 앞쪽 삽입이라 **5→4→3→2→1 역순**, (2) del_node는 **한 칸 걸러(짝수 위치) 삭제** → 5,3,1 생존.

## Q20. Python — 재귀 분할 + 짝/홀 병합

### 문제

```python
def mystery(lst):
    if len(lst) <= 1: return lst
    mid = len(lst) // 2
    left = mystery(lst[:mid])
    right = mystery(lst[mid:])
    return merge(left, right)
def merge(left, right):
    result = []
    while left and right:
        if (left[0] + right[0]) % 2 == 0:   # 두 앞원소 합이 짝수 → left
            result.append(left.pop(0))
        else:                                # 홀수 → right
            result.append(right.pop(0))
    result.extend(left or right)             # 남은 것 붙이기
    return result
print(mystery([5, 8, 2, 7]))
```

### 🤔 핵심 — 병합정렬 골격, 비교 기준은 "두 앞원소 합의 짝/홀"

### 1단계 — 분할(재귀 하강)

```
mystery([5,8,2,7]): mid=2 → left=mystery([5,8]), right=mystery([2,7])

  mystery([5,8]): mid=1 → left=mystery([5])=[5], right=mystery([8])=[8]
     merge([5],[8]): 5+8=13 홀수 → right(8) pop → result=[8]
        left=[5] 남음 → extend → [8,5]
     → 반환 [8,5]

  mystery([2,7]): mid=1 → [2], [7]
     merge([2],[7]): 2+7=9 홀수 → right(7) pop → result=[7]
        left=[2] 남음 → extend → [7,2]
     → 반환 [7,2]
```

### 2단계 — 최종 merge([8,5], [7,2]) 한 줄씩

```
left=[8,5], right=[7,2]
 8+7=15 홀 → right pop 7 → result=[7]        left=[8,5] right=[2]
 8+2=10 짝 → left pop 8  → result=[7,8]       left=[5]   right=[2]
 5+2=7  홀 → right pop 2 → result=[7,8,2]     left=[5]   right=[]
 while 종료(right 빔) → extend(left=[5]) → [7,8,2,5]
```

### ✅ 정답

```
[7, 8, 2, 5]
```
