## 🧭 먼저 읽는 법 2가지

### C 포인터 읽는 법

```
* = "그 주소가 가리키는 값" (역참조)   & = "그 변수의 주소"
arr[i]  ==  *(arr + i)   (항상 이렇게 바꿔 읽기)
void f(int *p)  = "p라는 주소를 받아서, 그 주소의 값을 건드릴 수 있는 함수"
                  (반환값 없음 = void, 하지만 포인터로 원본을 바꿈)
```

### Java 상속 "객체 따라가기" 3규칙

```
A obj = new C();  → obj가 적힌 타입(선언타입)=A,  실제 만들어진 객체=C
① 필드 obj.x        → 선언타입(A) 것을 봄
② 정적 메서드 obj.f() → 선언타입(A) 것을 봄
③ 인스턴스 메서드 obj.m() → 실제객체(C) 것을 봄 (오버라이딩되면 자식 것)
```
> 문제를 풀 땐 **"지금 this가 어느 객체냐"**, **"이건 필드냐 메서드냐"**를 매 줄 물어보세요.

## 6회 Q04. C — 연결 리스트 홀짝 분리 (`void sort` 완전 해석)

### 코드 모든 줄 해석

```c
struct Node { int value; struct Node* next; };
// value = 이 노드가 담는 값, next = "다음 노드의 주소"(포인터). NULL이면 끝.

void sort(Node* head) {          // Node* head = 첫 노드의 주소를 받음. 반환 없음(void).
    if (!head || !head->next) return;   // 노드가 0개(!head)거나 1개(!head->next)면 정렬 불필요 → 종료
    Node *odd = head;            // odd = 1번째 노드부터 (홀수 번째들을 모을 포인터)
    Node *even = head->next;     // even = 2번째 노드부터 (짝수 번째들)
    Node *even_head = even;      // even_head = 짝수 리스트의 "머리"를 기억(나중에 이어붙일 때 씀)
    while (even && even->next) { // 짝수 노드와 그 다음이 있는 동안
        odd->next = even->next;  // 홀수의 다음 = 짝수의 다음(=그다음 홀수)로 건너뜀
        odd = odd->next;         // odd를 그 홀수로 전진
        even->next = odd->next;  // 짝수의 다음 = 홀수의 다음(=그다음 짝수)
        even = even->next;       // even 전진
    }
    odd->next = even_head;       // 홀수 리스트 끝에 짝수 리스트 머리를 붙임
}
```

### 리스트 초기 상태 (add로 1..6 만들어 1→2→3→4→5→6)

```
[1]→[2]→[3]→[4]→[5]→[6]→NULL
 odd       even_head=[2]
```

### while 한 바퀴씩 "포인터가 어디를 가리키나"

```
시작: odd=[1], even=[2]
바퀴1: odd([1]).next = even([2]).next = [3]  ⇒ [1]→[3]
       odd = [3]
       even([2]).next = odd([3]).next = [4]  ⇒ [2]→[4]
       even = [4]
바퀴2: odd([3]).next = [4].next = [5]         ⇒ [3]→[5]
       odd = [5]
       even([4]).next = [5].next = [6]        ⇒ [4]→[6]
       even = [6]
바퀴3: even([6]).next == NULL → while 끝
끝맺음: odd([5]).next = even_head([2])          ⇒ [5]→[2]
```

### 최종 연결 = 출력

```
[1]→[3]→[5]→[2]→[4]→[6]→NULL
```

### ✅ 정답 `1 3 5 2 4 6`

## 6회 Q12. Java — 생성자 속 가상 호출 + 필드 은닉 (객체 따라가기)

### 코드 모든 줄 해석

```java
class A {
    A(){ print(); }              // A의 생성자 → print() 호출 (여기가 함정)
    void print(){ System.out.print("A"); }
}
class B extends A {
    String val = "B";            // B의 필드 val
    B(){ val = "B+"; }           // B 생성자 → val을 "B+"로
    void print(){ System.out.print(val); }   // print()를 재정의(오버라이딩) → B의 val 출력
}
class C extends B {
    String val = "C";            // C의 필드 val (B.val과 "다른 변수", 은닉)
    C(){ val = "C+"; }
    void print(){ System.out.print(val); }   // C의 val 출력
}
A obj1 = new B();   System.out.print("-");
A obj2 = new C();   System.out.print("-");
obj1.print();       System.out.print("-");   obj2.print();
```

### 객체 생성을 "따라가며" 추적

**`A obj1 = new B();`** — 실제 객체는 B
```
B() 실행 → 먼저 부모 A() 실행(super 자동)
   A() → print() 호출
        print()는 인스턴스 메서드 → "실제 객체 B" 기준 → B.print() 실행
        B.print()는 val 출력 → 이 시점 B.val은 아직 초기화 전!
        (순서: 부모 생성자 → 자식 필드 초기화 → 자식 생성자 본문)
        → B.val = null → "null" 출력
   A() 끝 → 이제 B.val = "B" 초기화 → 그 후 B() 본문 val="B+"
```
출력: `null`

**`-`**

**`A obj2 = new C();`** — 실제 객체는 C
```
C() → 부모 B() → 부모 A() → print() 호출
   실제 객체 C → C.print() 실행 → C.val 출력
   이 시점 C.val 미초기화 → null → "null"
```
출력: `null`

**`-`**

**`obj1.print();`** — obj1의 실제 객체는 B
```
print()는 인스턴스 메서드 → 실제 객체 B → B.print() → 이제 B.val="B+" → "B+"
```

**`-`**

**`obj2.print();`** — 실제 객체 C
```
C.print() → C.val="C+" → "C+"
```

### ✅ 정답 `null-null-B+-C+`

> 🎯 핵심: 생성자가 부를 때 print()는 **실제 객체(B/C)의 것**이 실행되지만, 그 순간 자식 필드는 **아직 null**. 생성이 끝난 뒤 부르면 제값("B+"/"C+").

## 6회 Q15. C — 시저 암호 + 문자열 배열 스왑 (`void`·이중포인터)

### 코드 모든 줄 해석

```c
void cipher(char *str, int shift){    // str = 문자열 시작 주소, shift만큼 밀기. 반환 없음.
    char *p = str;                    // p로 한 글자씩 순회
    while(*p){                        // *p != '\0'(문자열 끝)까지
        if (대문자)  *p = ((*p-'A'+shift)%26)+'A';   // 알파벳 26칸 회전
        else if(소문자) *p = ((*p-'a'+shift)%26)+'a';
        else if(숫자)  *p = ((*p-'0'+shift)%10)+'0';  // 숫자 10칸 회전
        p++;                          // 다음 글자
    }
}
void swap(char **a, char **b){        // char** = "문자열 포인터의 주소" → 포인터끼리 교환
    char *temp = *a; *a = *b; *b = temp;
}
int compare(char *a, char *b){        // 두 문자열 사전순 비교(음수/0/양수)
    while(*a && *b){ if(*a!=*b) return *a-*b; a++; b++; }
    return *a - *b;
}
```

### main 따라가기

```c
char s1[]="A9c", s2[]="B8a";
char *arr[]={s1,s2};      // arr[0]=s1주소, arr[1]=s2주소
cipher(arr[0], 2);        // s1 암호화
cipher(arr[1], 3);        // s2 암호화
if(compare(arr[0],arr[1])>0) swap(&arr[0],&arr[1]);
printf("%s-%s", arr[0], arr[1]);
```

**cipher(s1="A9c", 2)** — 한 글자씩
```
'A' 대문자: (0+2)%26 = 2 → 'C'
'9' 숫자:   (9+2)%10 = 1 → '1'
'c' 소문자: (2+2)%26 = 4 → 'e'
s1 = "C1e"
```
**cipher(s2="B8a", 3)**
```
'B': (1+3)=4 → 'E'
'8': (8+3)%10=1 → '1'
'a': (0+3)=3 → 'd'
s2 = "E1d"
```
**compare("C1e","E1d")**: 첫 글자 'C'(67) vs 'E'(69) → 67-69 = **-2** (음수)
→ `-2 > 0` 거짓 → swap 안 함 → arr 순서 그대로

### ✅ 정답 `C1e-E1d`

## 7회 Q11. C — 이중 포인터 문자열 배열 (완전 그림 해석)

### 코드 모든 줄 해석

```c
char arr[3][10] = {"hello", "world", "good"};   // 2차원 문자 배열(각 행이 문자열)
char *pArr[3] = {arr[2], arr[1], arr[0]};       // 포인터 배열: 행 주소를 "역순"으로 담음
char **p = pArr;                                 // p = pArr의 시작(이중 포인터)
printf("%s\n", *(p+1));       // ①
printf("%c\n", *(*(p)+1));    // ②
printf("%s\n", *(p+2)+2);     // ③
```

### 포인터 매핑 그림

```
arr[0]="hello"   arr[1]="world"   arr[2]="good"
pArr[0] → arr[2]="good"
pArr[1] → arr[1]="world"
pArr[2] → arr[0]="hello"
p → pArr[0]
```

### 한 줄씩 "별을 벗기며"

```
① *(p+1)
   p+1 = pArr[1]의 위치, *(...) = pArr[1] = arr[1] = "world"
   %s(문자열) → "world"

② *(*(p)+1)
   *(p) = pArr[0] = "good"
   "good"+1 = 'o'의 주소(두 번째 글자)
   *(...) = 'o'
   %c(한 글자) → 'o'

③ *(p+2)+2
   *(p+2) = pArr[2] = "hello"
   "hello"+2 = 인덱스2부터 = "llo"
   %s → "llo"
```

### ✅ 정답 `world` / `o` / `llo`

## 7회 Q05. C — 2차원 배열을 1차원 포인터로 (`*p` 순회)

### 코드 모든 줄 해석

```c
int arr[][3] = {3,5,7, 2,4,6, 1,8,9};   // 3×3으로 채워짐(선형: 3 5 7 2 4 6 1 8 9)
int sum = 0;
int *p = &arr[0][0];                     // p = 첫 원소 주소 → 전체를 1차원처럼 훑음
for(int i=0;i<9;i++){                     // 9칸을 인덱스로 접근
    if(i % 2 == 0) sum += *(p+i);         // 짝수 인덱스는 더하고
    else           sum -= *(p+i);         // 홀수 인덱스는 뺌
}
```

### 한 칸씩 (선형 배열 `3 5 7 2 4 6 1 8 9`)

```
i=0(+3): sum=3     i=1(-5): -2     i=2(+7): 5
i=3(-2): 3         i=4(+4): 7      i=5(-6): 1
i=6(+1): 2         i=7(-8): -6     i=8(+9): 3
```

### ✅ 정답 `결과 : 3`

## 7회 Q14. C — 뒤에서부터 최댓값 갱신 카운트 (`*p`가 기준을 따라감)

### 코드 모든 줄 해석

```c
int bars[] = {7,6,9,7,6,4,6};
int size = sizeof(bars)/sizeof(int);   // 7
int *p = bars + size - 1;              // p = 마지막 원소(bars[6]=6)를 가리킴 = "현재 기준"
int cnt = 1;                            // 마지막은 항상 카운트
for(int i=size-2; i>=0; i--){           // 뒤에서 두 번째부터 앞으로
    if(*(bars+i) > *p){                 // 현재 값이 기준(*p)보다 크면
        cnt++;                          //   카운트 +1
        p = bars + i;                   //   기준을 이 값으로 갱신(p가 따라 이동)
    }
}
```

### 한 칸씩 (기준 *p가 어디를 가리키는지)

```
초기: p→bars[6]=6, cnt=1
i=5: 4 > 6? X
i=4: 6 > 6? X
i=3: 7 > 6? O → cnt=2, p→bars[3]=7
i=2: 9 > 7? O → cnt=3, p→bars[2]=9
i=1: 6 > 9? X
i=0: 7 > 9? X
```

### ✅ 정답 `결과 : 3` (오른쪽에서 볼 때 커지는 막대 6→7→9 = 3개)

## 7회 Q07. Java — 싱글턴 (객체가 "하나"임을 따라가기)

### 코드 모든 줄 해석

```java
class CounterManager {
    private static CounterManager manager = null;  // 딱 하나 저장할 정적 변수
    private int value = 0;
    private CounterManager() { }                    // 생성자 private → 외부에서 new 못함
    public static CounterManager getManager() {     // 인스턴스 얻는 유일한 통로
        if (manager == null) manager = new CounterManager();  // 없으면 1번만 생성
        return manager;                             // 항상 "같은" 객체 반환
    }
    public void inValue() { value++; }
}
```

### main 따라가기 (c1, c2가 "같은 객체"임에 주목)

```
c1 = getManager() → manager 최초 생성, c1이 그 객체를 가리킴
c2 = getManager() → 이미 있음 → c2도 "같은" 객체
c1.inValue() → value 0→1
c1.inValue() → value 1→2
c2.inValue() → value 2→3   (c2도 같은 객체라 계속 누적)
c1.getValue() → 3
c2.getValue() → 3
c1 == c2 → 같은 주소(객체) → true
```

### ✅ 정답 `3` / `3` / `true`

## 7회 Q19. Java — 참조 전달, 두 객체를 서로 바꾸기 (객체 따라가기)

### 코드 모든 줄 해석

```java
class Counter {
    int value;
    Counter(int init){ value = init; }
    int exchange(Counter other){            // 나(this)와 other를 서로 바꿔치기
        int temp = this.value;               // 내 값 백업
        this.value = other.value * 2;        // 내 값 = 상대값 ×2
        other.value = temp / 2;              // 상대값 = 내 백업 ÷2
        return this.value - other.value;
    }
    int compute(Counter other){
        int result = this.exchange(other);   // ① 나.exchange(상대)
        result += other.exchange(this);      // ② 상대.exchange(나)
        return result;
    }
}
Counter cnt1 = new Counter(8);   // 객체① value=8
Counter cnt2 = new Counter(3);   // 객체② value=3
int result = cnt1.compute(cnt2);
```

### 객체를 따라가며 (누가 this고 누가 other인지 매번 확인)

```
cnt1.compute(cnt2): this=cnt1, other=cnt2
① result = cnt1.exchange(cnt2):  this=cnt1(8), other=cnt2(3)
     temp = 8
     cnt1.value = 3×2 = 6
     cnt2.value = 8÷2 = 4
     반환 = 6 - 4 = 2 → result=2
   현재: cnt1=6, cnt2=4
② result += cnt2.exchange(cnt1): this=cnt2(4), other=cnt1(6)
     temp = 4
     cnt2.value = 6×2 = 12
     cnt1.value = 4÷2 = 2
     반환 = 12 - 2 = 10 → result = 2+10 = 12
   현재: cnt1=2, cnt2=12
반환 result=12,  cnt1.value=2,  cnt2.value=12
```

### ✅ 정답 `12, 2, 12`

> 🎯 핵심: `exchange` 안의 `this`가 ①에서는 cnt1, ②에서는 cnt2로 **바뀐다**. 매 호출마다 "지금 this=누구, other=누구"를 적어두면 안 헷갈림.

## 8회 Q19. C — strstr + strncpy (포인터가 같은 버퍼를 공유)

### 코드 모든 줄 해석

```c
char arr[] = "AB12CD34EF";     // 한 개의 문자 버퍼
char *p1, *p2, *p3;
char temp[20] = "XYZ";
p1 = &arr[2];                  // p1 → arr의 인덱스2("12CD34EF")
p2 = strstr(arr, "12");        // arr에서 "12" 찾은 주소 = &arr[2]
if(p2) strncpy(temp, p2, 4);   // temp에 p2부터 4글자 복사("12CD")
printf("%s\n", temp);          // ①
arr[4] = 'C';                  // arr[4]는 원래 'C' → 변화 없음
p3 = strstr(p1, "CD");         // p1("12CD34EF")에서 "CD" 위치 = &arr[4]
if(p3) strncpy(p3, "789", 2);  // arr[4],[5]를 "78"로(2글자만)
printf("%s\n", arr);           // ②
strncpy(temp, arr+1, 3);       // temp 앞 3글자를 arr[1..3]로 덮음
printf("%s\n", temp);          // ③
```

> ⚠️ `strncpy(dst, src, n)` = "n글자만 복사, 널('\0')은 자동으로 안 붙임" → 뒤에 기존 글자가 남을 수 있음.

### 한 단계씩

```
① temp: strncpy(temp, "12CD34EF", 4) → 앞 4칸 "12CD" (temp="XYZ"였지만 4칸 덮음 → "12CD")
   출력 "12CD"
② arr: arr[4]='C'(변화없음). p3=&arr[4]. strncpy(p3,"789",2) → arr[4]='7',arr[5]='8'
   "AB12CD34EF" → "AB127834EF"
   출력 "AB127834EF"
③ temp: strncpy(temp, arr+1, 3) → arr[1..3]="B12" → temp 앞 3칸 덮음
   temp는 "12CD"였음 → 앞 3칸 "B12" → "B12D"(4번째 'D' 남음)
   출력 "B12D"
```

### ✅ 정답 `12CD` / `AB127834EF` / `B12D`

## 8회 Q04. Java — 전위 증감 + int/String 결합 (상속 아님, 연산 함정)

### 코드 모든 줄 해석

```java
int x = 3, y = 2;
for (int i = 1; i <= 3; i++) {
    x += i;                 // x에 i 더함
    y += ++x;               // ++x(먼저 1 증가한 값)를 y에 더함
    if ((x + y) % 2 == 0)
        System.out.println(x + y + "Java");   // 짝수: (x+y)를 "먼저 계산" 후 문자열
    else
        System.out.println("Java" + x + y);   // 홀수: "Java"부터라 전부 문자열 이어붙임
}
```

### 한 바퀴씩

```
초기 x=3, y=2
i=1: x+=1 → x=4;  ++x → x=5, y=2+5=7
     x+y=12 짝 → println(12 + "Java") = "12Java"   (12는 숫자로 먼저 계산)
i=2: x+=2 → x=7;  ++x → x=8, y=7+8=15
     x+y=23 홀 → println("Java"+8+15) = "Java815"   ("Java"부터라 8,15 그대로 이어붙임)
i=3: x+=3 → x=11; ++x → x=12, y=15+12=27
     x+y=39 홀 → println("Java"+12+27) = "Java1227"
```

### ✅ 정답 `12Java` / `Java815` / `Java1227`

> 🎯 핵심: `숫자+숫자+"문자열"`은 **왼쪽부터** → 숫자끼리 먼저 더한 뒤 문자열. `"문자열"+숫자+숫자`는 처음부터 문자열이라 그냥 이어붙임.

## 8회 Q15. Java — super 체인 + 필드 은닉 (⭐객체 따라가기 종합)

### 코드 모든 줄 해석

```java
class A { int num=100; A(){ num+=50; }              // A.num: 100 → 150
    void show(){ print(num+","); } }                 // A.show → A.num 출력
class B extends A { int num=200; int val;            // B.num, B.val (A.num과 별개)
    B(){ val=num+30; }                                // val = (B의)num+30 = 230
    void show(){ print(val+","); }                    // B.show → B.val
    void superShow(){ super.show(); } }               // A.show()를 부름
class C extends B { int num=300; int val=400;         // C.num, C.val (또 별개)
    C(){ super(); num+=50; val+=100; }                // C.num→350, C.val→500
    void show(){ print(val+","); }                    // C.show → C.val
    void display(){ show(); super.show(); super.superShow();
                    print(num+","); println(super.num); } }
C c = new C();  c.display();
```

### 1단계 — "각 num/val이 어느 클래스 것이고 최종 얼마?"

```
A.num = 100 → A() num+=50 → 150
B.num = 200
B.val = (B의 num=200)+30 = 230          ← B() 안의 num은 B.num!
C.num = 300 → C() num+=50 → 350
C.val = 400 → C() val+=100 → 500
```

### 2단계 — display()를 객체 따라가며 (`c`는 C 객체)

```
show()            → 인스턴스 메서드, 실제객체 C → C.show() → C.val = "500,"
super.show()      → B.show() → B.val = "230,"
super.superShow() → B.superShow() → 그 안 super.show() = A.show() → A.num = "150,"
print(num+",")    → display 코드는 C에 있음 → num = C.num = "350,"
println(super.num)→ super = B → B.num = "200"
```

### ✅ 정답 `500,230,150,350,200`

> 🎯 함정 2개:
> 1. `B()`의 `val=num+30`에서 num은 **B.num(200)** → 230 (A.num 아님).
> 2. `super.superShow()`가 **A.show()까지 두 단계** 올라가 A.num(150)을 출력.

# ✅ 6·7·8회에서 얻어갈 핵심 요약

### C 포인터
```
1. void 함수라도 포인터(*p)로 원본을 바꾼다 (연결 리스트 sort, 시저 cipher).
2. char *arr[] = 문자열들의 주소 배열, char **p로 접근. *(p+i)=문자열, *(*(p+i)+j)=글자.
3. 2차원도 &arr[0][0]로 1차원처럼 훑을 수 있다.
4. strncpy(n)은 n글자만, 널 안 붙임 → 뒤 글자 잔존. 같은 버퍼는 여러 포인터가 공유.
5. 포인터가 "기준"을 따라 이동(7회 Q14)하는 패턴 익히기.
```

### Java 상속/객체
```
1. A obj=new C(): 필드/정적메서드=선언타입(A), 인스턴스메서드=실제객체(C).

3. 필드는 "코드가 있는 클래스" 기준: B() 안의 num=B.num.
4. super.x=부모 필드, super.superShow()처럼 2단계 위로도 감.

6. 싱글턴: getManager()는 항상 같은 객체 → 값 공유(7회 Q07).
```
