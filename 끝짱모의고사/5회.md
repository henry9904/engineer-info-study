## Q01. C언어 — 이중 루프 최소 절대차

### 문제

```c
int func(int *a, int b) {
    int x = abs(a[1] - a[0]);          // ⓐ 초기 최소값
    int y;
    for (int i = 0; i < b - 1; i++) {         // ⓑ 바깥 루프
        for (int j = i + 1; j < b; j++) {     // ⓒ 안쪽 루프
            y = abs(a[j] - a[i]);             // ⓓ 두 원소 차의 절대값
            if (y < x) x = y;                 // ⓔ 더 작으면 갱신
        }
    }
    return x;
}
int main() {
    int p[] = {35, 20, 16, 45, 33};
    int q = 5;
    printf("결과:%d\n", func(p, q));
}
```

### 🤔 이 함수가 하는 일

배열의 **모든 두 원소 쌍의 차이(절대값) 중 가장 작은 값**을 찾는다.

### 한 줄씩 따라가기

```
ⓐ x = abs(a[1]-a[0]) = abs(20-35) = 15     ← 초기 최소값 15
```

바깥 i=0 (a[0]=35와 나머지 비교):
```
 j=1: y=abs(a[1]-a[0])=abs(20-35)=15   15<15? X       x=15
 j=2: y=abs(16-35)=19                  19<15? X       x=15
 j=3: y=abs(45-35)=10                  10<15? O →     x=10
 j=4: y=abs(33-35)=2                    2<10? O →     x=2
```
바깥 i=1 (a[1]=20와 뒤 비교):
```
 j=2: y=abs(16-20)=4     4<2? X    x=2
 j=3: y=abs(45-20)=25   25<2? X    x=2
 j=4: y=abs(33-20)=13   13<2? X    x=2
```
바깥 i=2 (a[2]=16):
```
 j=3: y=abs(45-16)=29   x=2
 j=4: y=abs(33-16)=17   x=2
```
바깥 i=3 (a[3]=45):
```
 j=4: y=abs(33-45)=12   x=2
```
→ 최소값 x = **2** (35와 33의 차)

### ✅ 정답

```
결과:2
```

## Q03. Java — static/instance 필드·메서드 바인딩

### 문제 (요지)

```java
class A {
    public static String s = "A";   // 정적 필드
    public String v = "B";           // 인스턴스 필드
    static void m() { print("C"); }  // 정적 메서드
    void n() { print("D"); }          // 인스턴스 메서드
    void x() { print(s + "/" + v); }
}
class B extends A {
    public static String s = "Z";
    public String v = "Y";
    static void m() { print("X"); }
    void n() { print("W"); }
    void x() { print(s + "/" + v); }   // 여기서 s,v는 B의 것
}
A p = new B();  B q = new B();
p.s; p.v; p.m(); p.n(); p.x();   // 첫 줄
q.s; q.v; q.m(); q.n(); q.x();   // 둘째 줄
```

### 🤔 핵심 규칙

| 구분           | 바인딩 기준       | 예           |
| ------------ | ----------- | ----------- |
| 정적 필드 `s`    | **선언 타입**   | p는 A타입 → A.s |
| 인스턴스 필드 `v`  | **선언 타입**   | p는 A타입 → A.v |
| 정적 메서드 `m()` | **선언 타입**   | p.m() → A.m() |
| 인스턴스 메서드 `n()`,`x()` | **실제 객체 타입** | p는 B객체 → B.n() |

### 한 줄씩 따라가기 — 첫 줄 `A p = new B();`

```
p.s   → 정적 필드, 선언타입 A → "A"
p.v   → 인스턴스 필드, 선언타입 A → "B"
p.m() → 정적 메서드, 선언타입 A → A.m() → "C"
p.n() → 인스턴스 메서드, 실제타입 B → B.n() → "W"
p.x() → 인스턴스 메서드, 실제타입 B → B.x() 실행
        B.x() 안의 s,v는 B의 필드 → "Z" + "/" + "Y" → "Z/Y"
```
첫 줄 = "A" + "B" + "C" + "W" + "Z/Y" = **ABCWZ/Y**

### 한 줄씩 따라가기 — 둘째 줄 `B q = new B();`

```
q.s   → 정적 필드, 선언타입 B → "Z"
q.v   → 인스턴스 필드, 선언타입 B → "Y"
q.m() → 정적 메서드, 선언타입 B → B.m() → "X"
q.n() → 실제타입 B → B.n() → "W"
q.x() → B.x() → "Z/Y"
```
둘째 줄 = "Z" + "Y" + "X" + "W" + "Z/Y" = **ZYXWZ/Y**

### ✅ 정답

```
ABCWZ/Y
ZYXWZ/Y
```
> ⚠️ 필드와 정적 메서드는 **선언 타입**(p→A), 인스턴스 메서드만 **실제 타입**(B)으로 결정.

## Q04. 캐싱 + CDN

## Q05. Python — map/filter/lambda 중첩

### 문제

```python
nums = [1, 2, 3, 4, 5]
result = list(map(lambda x: x * 2,
                  filter(lambda x: x % 2 != 0,
                         map(lambda x: x + 1, nums))))
print(result)
```

### 🤔 핵심 — 안쪽부터 바깥으로 3단계

`map(+1)` → `filter(홀수)` → `map(×2)` 순서로 데이터가 흐른다.

### 한 단계씩 따라가기

**1단계: `map(lambda x: x+1, nums)`** — 각 원소 +1
```
[1,2,3,4,5] → [2, 3, 4, 5, 6]
```

**2단계: `filter(lambda x: x%2!=0, ...)`** — 홀수만 통과
```
[2,3,4,5,6] → 홀수는 3, 5 → [3, 5]
```

**3단계: `map(lambda x: x*2, ...)`** — 각 원소 ×2
```
[3,5] → [6, 10]
```

| 단계        | 결과          |
| --------- | ----------- |
| +1        | [2,3,4,5,6] |
| 홀수 필터     | [3,5]       |
| ×2        | [6,10]      |

### ✅ 정답

```
[6, 10]
```
> ⚠️ 짝수→+1→홀수 되는 것에 주의. 원래 짝수(2,4)가 +1로 홀수(3,5)가 되어 통과.

## Q06. SQL — FULL OUTER JOIN + COUNT/SUM

### 문제

```sql
SELECT COUNT(A.VAL) AS A, COUNT(B.VAL) AS B, COUNT(*) AS C,
       SUM(A.VAL) AS D, SUM(B.VAL) AS E
FROM A FULL OUTER JOIN B ON A.ID = B.ID
WHERE A.VAL IS NULL OR B.VAL IS NULL;
```
A: (1,10),(2,NULL),(3,30),(4,40)  /  B: (3,NULL),(4,40),(5,50),(6,60)

### 🤔 핵심 — FULL OUTER JOIN은 양쪽 다 살림

매칭(ID 일치)은 3,4. 나머지는 짝 없이 NULL로 채워짐.

### 1단계: FULL OUTER JOIN 결과 (ID로 매칭)

| A.ID | A.VAL | B.ID | B.VAL | 비고        |
| ---- | ----- | ---- | ----- | --------- |
| 1    | 10    | NULL | NULL  | A만        |
| 2    | NULL  | NULL | NULL  | A만        |
| 3    | 30    | 3    | NULL  | 매칭        |
| 4    | 40    | 4    | 40    | 매칭        |
| NULL | NULL  | 5    | 50    | B만        |
| NULL | NULL  | 6    | 60    | B만        |

### 2단계: `WHERE A.VAL IS NULL OR B.VAL IS NULL`

각 행을 검사 — A.VAL이 NULL이거나 B.VAL이 NULL이면 통과:

| 행       | A.VAL | B.VAL | 통과?              |
| ------- | ----- | ----- | ---------------- |
| (1,10)  | 10    | NULL  | ✅ (B.VAL NULL)   |
| (2,-)   | NULL  | NULL  | ✅                |
| (3,30)  | 30    | NULL  | ✅ (B.VAL NULL)   |
| (4,40)  | 40    | 40    | ❌ (둘 다 값 있음)     |
| (5,50)  | NULL  | 50    | ✅ (A.VAL NULL)   |
| (6,60)  | NULL  | 60    | ✅                |

→ 통과 행 5개 (4-4 매칭행만 제외)

### 3단계: 집계

```
남은 5행의 A.VAL: 10, NULL, 30, NULL, NULL
남은 5행의 B.VAL: NULL, NULL, NULL, 50, 60

A = COUNT(A.VAL) = NULL 제외 → 10,30 → 2
B = COUNT(B.VAL) = NULL 제외 → 50,60 → 2
C = COUNT(*)     = 전체 행 → 5
D = SUM(A.VAL)   = 10+30 = 40
E = SUM(B.VAL)   = 50+60 = 110
```

### ✅ 정답

| A | B | C | D  | E   |
| - | - | - | -- | --- |
| 2 | 2 | 5 | 40 | 110 |

> ⚠️ COUNT(컬럼)·SUM은 NULL 무시. COUNT(*)만 NULL 포함 전체 행 수.

## Q07. 폴링 / 인터럽트 / 스풀링

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                              | 정답          |
| --- | ---------------------------------- | ----------- |
| ①  | OS가 장치 준비 상태를 **직접 주기적으로 확인**       | **폴링(Polling)** |
| ②  | 장치가 준비되면 **CPU에 신호(인터럽트)** 보냄       | **인터럽트(Interrupt)** |
| ③  | 빠른 데이터를 **보조기억장치에 임시 저장** 후 순차 처리   | **스풀링(Spooling)** |

### 메커니즘

```
폴링   : CPU가 계속 "준비됐니?" 물어봄 (CPU 낭비)
인터럽트: 장치가 "준비됐어!" 신호 → CPU가 하던 일 중단하고 처리 (효율↑)
스풀링 : 느린 장치(프린터)용 버퍼를 디스크에 두고 순서대로 처리
```

### ✅ 정답

① 폴링(Polling)  ② 인터럽트(Interrupt)  ③ 스풀링(Spooling)

## Q08. C언어 — 지그재그 2차원 배열 채우기

### 문제

```c
int matrix[3][3];
int num = 2;
for (int row = 0; row < 3; row++) {
    if (row % 2 == 0) {                      // 짝수 행: 왼→오
        for (int col = 0; col < 3; col++) { matrix[row][col] = num; num += 2; }
    } else {                                  // 홀수 행: 오→왼
        for (int col = 2; col >= 0; col--) { matrix[row][col] = num; num += 2; }
    }
}
// %3d로 출력
```

### 🤔 핵심 — 짝수 행은 정방향, 홀수 행은 역방향(부메랑/지그재그)

num은 2부터 시작해 **채울 때마다 +2** (짝수만 생성).

### 한 줄씩 따라가기

**row=0 (짝수 → col 0→2)**
```
col0: matrix[0][0]=2, num→4
col1: matrix[0][1]=4, num→6
col2: matrix[0][2]=6, num→8
```
1행: `2 4 6`

**row=1 (홀수 → col 2→0)**
```
col2: matrix[1][2]=8,  num→10
col1: matrix[1][1]=10, num→12
col0: matrix[1][0]=12, num→14
```
2행(인덱스 순 0,1,2): `12 10 8`

**row=2 (짝수 → col 0→2)**
```
col0: matrix[2][0]=14, num→16
col1: matrix[2][1]=16, num→18
col2: matrix[2][2]=18, num→20
```
3행: `14 16 18`

### 최종 배열 (`%3d` = 3칸 오른쪽 정렬)

```
  2  4  6
 12 10  8
 14 16 18
```

### ✅ 정답

```
  2  4  6
 12 10  8
 14 16 18
```
> ⚠️ 홀수 행(row=1)은 col을 2→0으로 채워 저장 위치가 뒤집힘 → 출력 시 12 10 8.

## Q10. Python — 중첩 자료구조 인덱싱

### 문제

```python
x = [
    (1, 2, {'A': [10, 20]}),          # x[0]
    [3, 4, ('B', {'C': 30})]          # x[1]
]
print(x[0][2]['A'][1])
print(x[1][2][1]['C'])
print(x[1][0])
print(x[0][1])
print(x[1][2][1])
```

### 한 줄씩 따라가기 — 안쪽으로 파고들기

```
x[0][2]['A'][1]:
  x[0] = (1, 2, {'A':[10,20]})
  x[0][2] = {'A':[10,20]}
  x[0][2]['A'] = [10,20]
  [1] = 20                      → 20

x[1][2][1]['C']:
  x[1] = [3, 4, ('B',{'C':30})]
  x[1][2] = ('B', {'C':30})
  x[1][2][1] = {'C':30}
  ['C'] = 30                    → 30

x[1][0] = 3                     → 3
x[0][1] = 2                     → 2
x[1][2][1] = {'C':30}           → {'C': 30}
```

### ✅ 정답

```
20
30
3
2
{'C': 30}
```

## Q11. 정규형 판별 (1NF)

### 문제 (직원 테이블)

| 101  | 영업부 | A 프로젝트  | 3개월     |
| 102  | 인사부 | B 프로젝트  | 2개월     |
| 101  | 영업부 | C 프로젝트  | 5개월     |
| ...  | ... | ...     | ...     |

### 🤔 핵심 — 정규형은 "만족하는 가장 높은 단계"

이 테이블은 **직원번호 101이 여러 프로젝트**를 가짐 → 기본키가 (직원번호+프로젝트명) 복합키여야 유일 식별 가능.

### 판별 과정

```
1NF: 모든 값이 원자값(하나) → ✅ 만족 (반복 그룹 없음)

2NF 확인: 기본키 = (직원번호, 프로젝트명)
  - 부서명은 직원번호에만 종속 (프로젝트명과 무관)
  → 부분 함수 종속 존재 → 2NF 위반 ❌
```
따라서 만족하는 **가장 높은 정규형 = 1NF**.

### 정규형 요약

| 정규형 | 조건                    |
| --- | --------------------- |
| 1NF | 원자값 (반복 그룹 제거)        |
| 2NF | 부분 함수 종속 제거           |
| 3NF | 이행 함수 종속 제거           |
| BCNF | 모든 결정자가 후보키           |

### ✅ 정답

1NF (제1정규형)
> ⚠️ 부서명이 기본키 일부(직원번호)에만 종속 → 부분종속 → 2NF 못 감. 그래서 1NF가 최고.

## Q12. Java — 전략 패턴(Strategy)

### 문제 (요지)

```java
interface Strategy { int run(int a, int b); }
class Add implements Strategy { public int run(int a,int b){return a+b;} }
class Multiply implements Strategy { public int run(int a,int b){return a*b;} }
class Context {
    private Strategy strategy;
    public void set(Strategy s){ this.strategy = s; }
    public int execute(int a,int b){ return strategy.run(a,b); }
}
Context ctx = new Context();
ctx.set(new Add());       print(ctx.execute(10,5) + " ");
ctx.set(new Multiply());  print(ctx.execute(10,5));
```

### 🤔 핵심 — 전략을 갈아 끼우는 디자인 패턴

`Context`는 어떤 `Strategy`가 꽂혔는지에 따라 다르게 동작.

### 한 줄씩 따라가기

```
ctx.set(new Add())      → ctx.strategy = Add 객체
ctx.execute(10,5)       → Add.run(10,5) = 10+5 = 15 → print "15 "
ctx.set(new Multiply()) → ctx.strategy = Multiply 객체 (교체)
ctx.execute(10,5)       → Multiply.run(10,5) = 10*5 = 50 → print "50"
```

### ✅ 정답

```
15 50
```
> 💡 전략 패턴: 알고리즘(전략)을 캡슐화해 런타임에 교체. execute는 같지만 결과가 달라짐.

## Q13. XSS + CSRF

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                              | 정답       |

| ①  | 악성 스크립트 주입 → 방문자 브라우저에서 실행         | **XSS**  |


## Q16. C언어 — char 포인터 + 이중 포인터 문자열

### 문제

```c
char arr[] = "NETWORK";
char *ptr = arr;
*(ptr + 2) = 'X';      // arr[2] 'T' → 'X'
*(ptr + 5) = '\0';     // arr[5] 'R' → 널 종료
char *pArr[] = {"SYSTEM", "MEMORY", "MODULE"};
char **dp = pArr;
printf("%s\n", ptr);              // ①
printf("%c\n", *(*(dp + 1) + 2)); // ②
printf("%s\n", *(dp + 2) + 3);    // ③
printf("%c\n", **dp + 1);         // ④
```

### 한 줄씩 따라가기 — 문자열 변형

```
arr = "NETWORK"  (N0 E1 T2 W3 O4 R5 K6 \0)
*(ptr+2)='X' → arr = "NEXWORK"
*(ptr+5)='\0' → arr[5]자리를 널로 → 문자열은 "NEXWO"로 잘림
```

### 출력 한 줄씩

```
① printf("%s", ptr)
   ptr은 arr 시작 → "NEXWO" (널로 잘려서)

② printf("%c", *(*(dp+1)+2))
   dp = pArr,  *(dp+1) = pArr[1] = "MEMORY"
   *(pArr[1] + 2) = "MEMORY"[2] = 'M'   ← M0 E1 M2 → 'M'

③ printf("%s", *(dp+2)+3)
   *(dp+2) = pArr[2] = "MODULE"
   +3 → "MODULE" 인덱스 3부터 = "ULE"

④ printf("%c", **dp + 1)
   **dp = *(pArr[0]) = "SYSTEM"[0] = 'S'
   'S' + 1 = 'T'   ← 문자 코드 +1 (아스키 83→84)
```

| 출력 | 값       | 이유                    |
| -- | ------- | --------------------- |
| ①  | NEXWO   | 'T'→'X', 인덱스5 널 종료    |
| ②  | M       | "MEMORY"[2]           |
| ③  | ULE     | "MODULE"+3            |
| ④  | T       | 'S'+1 (아스키 증가)        |

### ✅ 정답

```
NEXWO
M
ULE
T
```
> ⚠️ ④는 `**dp`가 문자 'S'(정수 83)라 +1하면 문자 'T'(%c 출력). 문자열이 아닌 문자 연산.

## Q18. C언어 — 2차원 동적할당 + 인덱스 회전

### 문제

```c
void func(int **res, int *data, int rows, int cols){
    for(int i = 0; i < rows*cols; i++){
        int idx = (i + 2) % (rows*cols);     // 2칸 밀기(회전)
        res[idx/cols][idx%cols] = data[i];
    }
}
int main(){
    int data[6] = {5, 2, 7, 8, 1, 4};
    int rows = 2, cols = 3;   // 2×3 동적 배열
    func(res, data, rows, cols);
    int sum = 0;
    for(int i = 0; i < rows*cols; i++)
        sum += res[i/cols][i%cols] * ((i%2==0) ? 1 : -1);  // 짝수 +, 홀수 -
    printf("Result: %d\n", sum);
}
```

### 🤔 핵심 — data를 2칸 회전해 배치한 뒤, 교대 부호 합

### 1단계: func — data[i]를 idx=(i+2)%6 위치에 저장

전체 6칸(선형 인덱스 0~5). res를 선형으로 보면:

| i | data[i] | idx=(i+2)%6 | res 선형[idx] |
| - | ------- | ----------- | ----------- |
| 0 | 5       | 2           | res[2]=5    |
| 1 | 2       | 3           | res[3]=2    |
| 2 | 7       | 4           | res[4]=7    |
| 3 | 8       | 5           | res[5]=8    |
| 4 | 1       | 0           | res[0]=1    |
| 5 | 4       | 1           | res[1]=4    |

→ res 선형 = [1, 4, 5, 2, 7, 8]  (data를 오른쪽으로 2칸 회전)

### 2단계: 교대 부호 합 (i 짝수 +, 홀수 −)

```
i=0: res[0]=1 × (+1) = +1     sum=1
i=1: res[1]=4 × (-1) = -4     sum=-3
i=2: res[2]=5 × (+1) = +5     sum=2
i=3: res[3]=2 × (-1) = -2     sum=0
i=4: res[4]=7 × (+1) = +7     sum=7
i=5: res[5]=8 × (-1) = -8     sum=-1
```
→ sum = **-1**

### ✅ 정답

```
Result: -1
```
> ⚠️ `(i+2)%6`은 배치를 2칸 밀어 회전. res 선형=[1,4,5,2,7,8]에 +,−,+,−,+,− 적용.

## Q20. 스케줄링 — FCFS 최대-최소 평균 반환시간 차

### 문제

| 프로세스 | 실행 시간 |
| ---- | ----- |
| P1   | 8     |
| P2   | 4     |
| P3   | 9     |
| P4   | 3     |

> 모두 동시 도착. 순서를 바꿔 얻는 **최대 평균 반환시간 T**, **최소 t**, T−t는?

### 🤔 핵심 — FCFS 평균 반환시간은 순서에 좌우됨

동시 도착 FCFS에서 반환시간 = 완료시간. 짧은 작업을 앞에 두면(SJF) 평균 최소, 긴 작업을 앞에 두면 평균 최대.

### 최소 평균 (짧은 것부터: P4=3, P2=4, P1=8, P3=9)

```
완료시간 누적:
P4: 3
P2: 3+4 = 7
P1: 7+8 = 15
P3: 15+9 = 24
합 = 3+7+15+24 = 49
평균 t = 49 / 4 = 12.25
```

### 최대 평균 (긴 것부터: P3=9, P1=8, P2=4, P4=3)

```
완료시간 누적:
P3: 9
P1: 9+8 = 17
P2: 17+4 = 21
P4: 21+3 = 24
합 = 9+17+21+24 = 71
평균 T = 71 / 4 = 17.75
```

### T − t

```
T − t = 17.75 − 12.25 = 5.5
```

### ✅ 정답

```
5.5
```
> 💡 앞선 작업의 실행시간이 뒤 작업 대기에 누적되므로, 긴 작업을 앞세울수록 평균이 커짐.
