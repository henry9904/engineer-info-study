## Q01. 후보키 판별

### 문제

> [직원] 튜플 기준 후보키 가능한 것: ①{직원번호} ②{직원이름} ③{전화번호} ④{직급,부서명} ⑤{직원이름,부서명} ⑥{직원번호,직급}

### 🤔 핵심 — 후보키 = 유일성 O + 최소성 O

```
직원번호: E001~E004 모두 다름 → 유일 O, 단일속성 최소 O → ① 후보키 ✓
직원이름: 김철수 중복(E001,E003) → 유일성 X → ✗
전화번호: 모두 다름 → 유일 O → ③ 후보키 ✓
직급/부서명 조합: 부서명 인사부에 대리/과장 중복 가능 → 유일성 X (E001 대리 인사부, E004 대리 인사부) → ✗
직원이름+부서명: 김철수+인사부 중복(E001,E003) → ✗
직원번호+직급: 직원번호만으로 유일 → 직급은 불필요 → 최소성 위반 → ✗ (슈퍼키)
```

### ✅ 정답

①, ③

## Q04. C언어 — 연결 리스트 스택(홀짝)

### 문제

```c
// data = {3,5,7,9,2,8}
// 홀수면 push, 짝수면 pop해서 result에 더함, 끝나면 남은 것 전부 pop
```

### 🤔 이 함수가 하는 일

홀수는 스택에 쌓고, 짝수를 만나면 스택 top을 꺼내 더한다. 마지막에 남은 것도 전부 더함.

### 한 원소씩 따라가기

```
스택=[], result=0
3(홀): push → [3]
5(홀): push → [3,5]
7(홀): push → [3,5,7]
9(홀): push → [3,5,7,9]
2(짝): pop → 9, result=9 → [3,5,7]
8(짝): pop → 7, result=9+7=16 → [3,5]
루프 종료. 남은 스택 [3,5] 전부 pop:
   pop → 5, result=16+5=21
   pop → 3, result=21+3=24
```

### ✅ 정답

```
24
```

// list = [4,2,7,5,1]

## Q06. Python — 람다 클로저 함정

### 문제

```python
def confuse(nums):
    result = []
    for i in nums:
        if i % 2:
            result.append(lambda: i * 2)         # i를 나중에 참조(클로저)
        else:
            result.append(lambda x=i: x * 3)     # 기본인자로 i 값 캡처
    return [f() for f in result]
numbers = [1, 2, 3, 4]
```

### 🤔 핵심 함정 — `lambda: i*2`는 **호출 시점의 i**(마지막 값)를 참조

홀수 분기 lambda는 변수 i를 캡처(늦은 바인딩) → 반복 종료 후 i=4. 짝수 분기는 기본인자 `x=i`로 그 시점 값 고정.

### 한 원소씩 따라가기

```
i=1(홀): lambda: i*2  (i 참조, 나중에 평가)
i=2(짝): lambda x=2: x*3  (2 고정)
i=3(홀): lambda: i*2  (i 참조)
i=4(짝): lambda x=4: x*3  (4 고정)

호출 시점(반복 끝, i=4):
  [0] lambda: i*2 = 4*2 = 8
  [1] lambda x=2: 2*3 = 6
  [2] lambda: i*2 = 4*2 = 8
  [3] lambda x=4: 4*3 = 12
```

### ✅ 정답

```
[8, 6, 8, 12]
```
> ⚠️ 홀수 lambda는 모두 마지막 i(4)를 참조 → 8, 8. 짝수 lambda는 기본인자로 각각 2,4 고정 → 6, 12.

## Q07. SQL — COUNT/DISTINCT/MAX + NULL

### 문제

```sql
SELECT COUNT(A) AS A, COUNT(DISTINCT A) AS B, MAX(A + B) AS C FROM R;
```
R: (1000,500),(1000,1000),(2000,1000),(4000,NULL),(NULL,NULL)

### 단계별 트레이싱

```
A = COUNT(A): NULL 제외 → 1000,1000,2000,4000 = 4
B = COUNT(DISTINCT A): {1000,2000,4000} = 3
C = MAX(A+B): A+B 계산 (NULL이 있으면 그 행은 NULL)
   1000+500=1500, 1000+1000=2000, 2000+1000=3000, 4000+NULL=NULL, NULL+NULL=NULL
   MAX = 3000
```

### ✅ 정답

| A | B | C |
| - | - | - |
| 4 | 3 | 3000 |

## Q11. Java — 인스턴스/정적 필드 갱신

### 문제 (요지)

```java
class Counter { int num=0; static int total=0; Counter(int n){num=n; total+=n;}
   void update(Counter o){this.num += o.num; total += this.num;} }
c1 = new Counter(5); c2 = new Counter(10);
c1.update(c2); c2.update(c1);
```

### 🤔 핵심 — num은 **객체마다 따로**(인스턴스), total은 **모두 공유**(정적)

```
num(인스턴스): c1.num, c2.num 각자 존재
total(정적): 객체 상관없이 "하나"만 존재 → 누구든 total+=하면 같은 값이 쌓임
```

### num(인스턴스, 2개) | total(정적, 1개) 동시 추적

| 동작 | c1.num | c2.num | total |
| -- | :-: | :-: | :-: |
| c1=new Counter(5) | 5 | — | 5 |
| c2=new Counter(10) | 5 | 10 | 5+10=**15** |
| c1.update(c2): c1.num+=c2.num(10)=15, total+=15 | **15** | 10 | 15+15=**30** |
| c2.update(c1): c2.num+=c1.num(15)=25, total+=25 | 15 | **25** | 30+25=**55** |

**출력**: c1.num=**15**, c2.num=**25**, total=**55**

### ✅ 정답

```
15
25
55
```
> 🎯 함정: update의 `total += this.num`은 항상 **같은 정적 total 하나**에 누적. c1.update가 15 더하고, c2.update가 (그때의 c2.num=25) 더해 55. num은 각 객체 것이라 15, 25로 나뉨.

## Q12. SQL — 관계 나눗셈(NOT EXISTS + EXCEPT)

### 문제

```sql
SELECT DISTINCT sid FROM student X
WHERE NOT EXISTS (
    (SELECT course FROM required)
    EXCEPT
    (SELECT course FROM student Y WHERE X.sid = Y.sid)
);
```
required = {국어, 영어, 수학}

### 🤔 핵심 — 관계 나눗셈: **required의 모든 과목을 수강한 학생**

`(required) EXCEPT (해당 학생 수강)` = 학생이 안 들은 필수 과목. 이게 비어있으면(NOT EXISTS) 모든 필수를 들은 것.

### 단계별 트레이싱

```
2001: 수학,영어 → 국어 안 들음 → EXCEPT={국어} 비어있지 않음 → 제외
2002: 영어,수학,국어 → 전부 들음 → EXCEPT={} → NOT EXISTS 참 → 포함
2003: 영어만 → 제외
2004: 국어,영어,수학 → 전부 → 포함
2005: 국어,과학 → 영어,수학 안 들음 → 제외
```

### ✅ 정답

| sid |
| --- |
| 2002 |
| 2004 |

## Q13. Python — 얕은 복사 vs 슬라이스 복사

### 문제

```python
a = [1, 2, 3, 4, 5]
b = a           # 같은 객체(별칭)
c = a[:]        # 얕은 복사(다른 객체)
b[1:4] = [10, 20]   # a와 b는 같음 → a도 변함
c[2:] = [30, 40, 50]
a.append(6)
b[0] = 100
c += [60]
```

### 한 줄씩 따라가기

```
a = [1,2,3,4,5], b=a(별칭), c=a[:]=[1,2,3,4,5](복사본)
b[1:4]=[10,20]: b(=a) → [1,10,20,5]  (인덱스1~3의 3개를 2개로 교체)
c[2:]=[30,40,50]: c → [1,2,30,40,50]
a.append(6): a(=b) → [1,10,20,5,6]
b[0]=100: b(=a) → [100,10,20,5,6]
c += [60]: c → [1,2,30,40,50,60]

print a: [100,10,20,5,6]
print b: [100,10,20,5,6]  (a와 동일)
print c: [1,2,30,40,50,60]
```

### ✅ 정답

```
[100, 10, 20, 5, 6]
[100, 10, 20, 5, 6]
[1, 2, 30, 40, 50, 60]
```
> ⚠️ b=a는 별칭(같은 객체)이라 a,b가 항상 동일. c=a[:]는 독립 복사본.

## Q15. C언어 — while 복합 조건 + 증감

### 문제

```c
int a=1, b=2, c=3, sum=0;
while (++a < 5 || --b > 0) {
    sum += a++ + --c;
    if (a % 2 && b-- > 0) {
        sum -= (++c) * 2;
    }
}
printf("%d %d %d %d", a, b, c, sum);
```

### 한 반복씩 따라가기

```
초기: a=1, b=2, c=3, sum=0

[반복1] ++a → a=2, 2<5 참 → || 단락(--b 안 함), 진입
   sum += a++ + --c:  a++ 값2 사용 a→3, --c → c=2 → sum += 2+2=4 → sum=4
   if (a%2 && b-->0): a=3, 3%2=1 참 → b-- 값2>0 참 b→1 → 참
      sum -= (++c)*2: ++c → c=3, sum -= 3*2=6 → sum=4-6=-2
   (a=3, b=1, c=3, sum=-2)
[반복2] ++a → a=4, 4<5 참 → 진입
   sum += a++ + --c: a++ 값4 a→5, --c → c=2 → sum += 4+2=6 → sum=4
   if (a%2 && ...): a=5, 5%2=1 참 → b-- 값1>0 참 b→0 → 참
      sum -= (++c)*2: ++c → c=3, sum -= 6 → sum=4-6=-2
   (a=5, b=0, c=3, sum=-2)
[반복3] ++a → a=6, 6<5 거짓 → || --b → b=-1, -1>0 거짓 → 전체 거짓 → 루프 종료
   (a=6, b=-1, c=3)
```
최종: a=6, b=-1, c=3, sum=-2

### ✅ 정답

```
6 -1 3 -2
```
> ⚠️ 마지막 반복에서 ++a(6)로 첫 조건 거짓, --b(-1)로 둘째도 거짓 → 종료. a=6, b=-1.

## Q18. C언어 — 2차원 배열 전치 누적

### 문제

```c
void func(int (*ptr)[3], int n) {
    for (i=0; i<n; i++)
      for (j=0; j<n; j++)
        *(*(ptr+i)+j) += *(*(ptr+j)+i);   // arr[i][j] += arr[j][i]
}
int arr[3][3] = {{1,2,3},{4,5,6},{7,8,9}};
```

### 포인터 해석

```c
void func(int (*ptr)[3], int n)   // int(*)[3] = "int[3] 한 행을 가리키는 포인터"(배열 포인터)
   *(*(ptr+i)+j)                  // = ptr[i][j]  (i행 j열)
   arr[i][j] += arr[j][i]         // 자기 자신에 "전치 위치(j,i)"의 값을 더함
```

### 🤔 핵심 함정 — 전치를 더하지만 **순차 진행 중 배열이 바뀌어**, 대칭 위치가 "이미 갱신된 값"을 씀

### 한 칸씩 — 행렬 상태 변화 (더하는 대상을 굵게)

초기:
```
1 2 3
4 5 6
7 8 9
```

| 칸 (i,j) | 식 | 더하는 값 | 결과 |
| ------ | -- | ----- | -- |
| (0,0) | a[0][0]+=a[0][0] | 1 | 2 |
| (0,1) | a[0][1]+=a[1][0] | 4 | 6 |
| (0,2) | a[0][2]+=a[2][0] | 7 | 10 |
| (1,0) | a[1][0]+=**a[0][1]** | **6**(이미 바뀜!) | 4+6=10 |
| (1,1) | a[1][1]+=a[1][1] | 5 | 10 |
| (1,2) | a[1][2]+=a[2][1] | 8 | 14 |
| (2,0) | a[2][0]+=**a[0][2]** | **10**(이미 바뀜!) | 7+10=17 |
| (2,1) | a[2][1]+=**a[1][2]** | **14**(이미 바뀜!) | 8+14=22 |
| (2,2) | a[2][2]+=a[2][2] | 9 | 18 |

### ✅ 정답

```
2 6 10 
10 10 14 
17 22 18 
```
> 🎯 함정: 정상 전치 합이면 대칭이 나와야 하지만, a[0][1]을 먼저 6으로 바꾼 뒤 a[1][0]이 그 6을 더해 **10**(원래 4+2=6이 아님!). 순차 갱신 때문에 대칭이 깨짐.

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트

1. **Q01 후보키**: 유일성+최소성. 직원번호·전화번호만 유일(이름 중복). 직원번호+직급은 최소성 위반.
2. **Q06 람다 클로저**: `lambda: i*2`는 늦은 바인딩(마지막 i=4). `lambda x=i`는 기본인자로 값 고정.
3. **Q12 관계 나눗셈**: required 전부 수강한 학생. NOT EXISTS(required EXCEPT 수강) = 안 들은 필수 없음.
4. **Q13 별칭 vs 복사**: b=a는 같은 객체, c=a[:]는 복사본.
5. **Q15 while 복합조건**: 마지막 ++a(6) 첫 조건 거짓 → --b(-1) 둘째 거짓 → 종료.
6. **Q18 전치 누적 함정**: 순차 갱신이라 대칭 위치가 이미 바뀐 값을 더함.
