## Q01. Python — 차분(difference) 반복

### 문제

```python
def func(items, n=1):
    result = items.copy()
    for _ in range(n):
        temp = result[:]
        result.clear()
        for i in range(1, len(temp)):
            result.append(temp[i] - temp[i-1])   # 인접 차분
    return result
data = [4, 6, 10, 16, 24]
print(func(data))       # n=1
print(func(data, 2))    # n=2
print(func(data, 3))    # n=3
```

### 🤔 이 함수가 하는 일

리스트의 **인접 원소 차이**를 n번 반복 계산 (계차수열).

### 한 단계씩 따라가기

```
data = [4, 6, 10, 16, 24]

n=1: 차분 1회
   [6-4, 10-6, 16-10, 24-16] = [2, 4, 6, 8]

n=2: 차분 2회
   1회 → [2, 4, 6, 8]
   2회 → [4-2, 6-4, 8-6] = [2, 2, 2]

n=3: 차분 3회
   1회 → [2, 4, 6, 8]
   2회 → [2, 2, 2]
   3회 → [2-2, 2-2] = [0, 0]
```

### ✅ 정답

```
[2, 4, 6, 8]
[2, 2, 2]
[0, 0]
```

## Q04. C언어 — static 지역변수 + 포인터

### 문제

```c
int num = 10;              // 전역
int* increment() {
    static int num = 1;    // 정적 지역(호출 간 유지), 전역 num과 별개
    num += 2;
    return &num;           // 같은 static 변수 주소 반환
}
void modify(int* n) { *n += 5; }
int main() {
    int* p1 = increment();  // static num: 1→3
    int* p2 = increment();  // static num: 3→5
    modify(p1);             // static num += 5
    modify(p2);             // static num += 5
    printf("%d, ", num);    // 전역 num
    printf("%d, ", *p1);
    printf("%d\n", *p2);
}
```

### 🤔 핵심 3가지

```
① 전역 num(10)과 increment 안의 static num은 "이름만 같은 다른 변수".
② static 지역변수 num은 함수가 끝나도 안 사라지고 "딱 하나"만 유지됨.
③ 그래서 increment()를 두 번 불러도 반환하는 &num은 "같은 주소"
   → p1, p2가 결국 같은 변수를 가리킴!
```

```
그림:  [전역 num=10]  ← 완전 별개, 아무도 안 건드림
       [static num] ← increment의 &num, p1도 p2도 여기를 가리킴(하나뿐)
```

### static num 값 추적 (p1, p2는 이 하나를 공유)

| 호출/동작 | static num | 비고 |
| ------ | --------- | -- |
| increment() 1회: num+=2 | 1 → **3** | p1 = &num |
| increment() 2회: num+=2 | 3 → **5** | p2 = &num (같은 주소!) |
| modify(p1): *p1+=5 | 5 → **10** | p1이 가리키는 static num |
| modify(p2): *p2+=5 | 10 → **15** | p2도 같은 곳 → 계속 누적 |

**출력**: `num`(전역)=**10**(안 바뀜), `*p1`=**15**, `*p2`=**15**(p1과 같은 변수)

### ✅ 정답

```
10, 15, 15
```
> 🎯 함정: p1과 p2는 다른 변수처럼 보이지만 **같은 static num을 가리킴**. modify 두 번이 같은 변수에 누적 → 둘 다 15. 전역 num(10)은 이름만 같고 별개라 그대로.

## Q06. C언어 — 8/16진수 + 비트 복합

### 문제

```c
int x = 016;    // 8진수 → 14
int y = 0x0F;   // 16진수 → 15
int z;
z = (x++ & y) | (--y << 1);       printf("%d, ", z);
z = (++x ^ y--) && (y | 0x01);    printf("%d, ", z);
z = (x & y) + (~x | y);           printf("%d, ", z);
printf("%d, %d\n", x, y);
```

### 한 문장씩 따라가기

```
초기: x=14(016), y=15(0x0F)

문장1: z = (x++ & y) | (--y << 1)
   x++ → 값 14 사용, x→15
   14 & 15 = 1110 & 1111 = 1110 = 14
   --y → y→14, 14<<1 = 28
   14 | 28 = 01110 | 11100 = 11110 = 30
   z=30, (x=15, y=14)

문장2: z = (++x ^ y--) && (y | 0x01)
   ++x → x→16, y-- → 값 14 사용, y→13
   16 ^ 14 = 10000 ^ 01110 = 11110 = 30 (참)
   (y|1) = 13|1 = 13 (참)
   30 && 13 → 참 → z=1
   z=1, (x=16, y=13)

문장3: z = (x & y) + (~x | y)
   x & y = 16 & 13 = 10000 & 01101 = 0
   ~x = ~16 = -17
   ~x | y = -17 | 13 = -17  (…11101111 | 00001101 = …11101111 = -17)
   z = 0 + (-17) = -17
   z=-17

최종 x, y: x=16, y=13
```

### ✅ 정답

```
30, 1, -17, 16, 13
```
> ⚠️ 016=8진수 14, 0x0F=15. 비트 계산은 이진수로. ~x|y가 음수(-17)임에 주의.

## Q08. Python — 타입별 자료구조 변환

### 문제

```python
def func(data):
    if isinstance(data, dict):
        return list(data.keys())[::2], list(data.values())[1::2]
    elif isinstance(data, list):
        if len(data) > 2:
            return {data[i]: data[i+1] for i in range(0, len(data)-1, 2)}
        return data
    elif isinstance(data, tuple):
        return dict(zip(data, data[::-1]))
a1 = [1,"two",3,"four",5]; a2 = {"x":10,"y":20,"z":30,"w":40}; a3 = (1,2,3,4)
```

### 한 입력씩 따라가기

```

   → {1:"two", 3:"four"}

func(a2) — dict:
   keys[::2] = ['x','y','z','w'][::2] = ['x','z']
   values[1::2] = [10,20,30,40][1::2] = [20,40]
   → (['x','z'], [20,40])

func(a3) — tuple:
   zip((1,2,3,4), (4,3,2,1)) → {1:4, 2:3, 3:2, 4:1}
```

### ✅ 정답

## Q10. Java — 참조 연결 + 연쇄 처리(counter)

### 문제 (요지)

```java
class Ref { int value; Ref p; static int counter=0;
  Ref(int v){value=v; counter++;}
  void link(Ref o){this.p=o;}
  int process(Ref o){ int r=this.value+o.value; this.value-=2; o.value*=2; if(p!=null)p.value+=5; counter+=r; return r; }
  int chainProcess(Ref mid, Ref end){ int s1=process(mid); int s2=mid.process(end); int s3=end.process(this); this.value=this.value+mid.value-end.value; return s1+s2+s3; }
}
obj1=Ref(10), obj2=Ref(20), obj3=Ref(30);
obj1.link(obj2); obj2.link(obj3); obj3.link(obj1);
```

### 🤔 process가 하는 일 (한 번 호출에 3개 객체가 바뀜)

```
process(o):  r = this.value + o.value      (반환할 값)
             this.value -= 2               (나를 -2)
             o.value *= 2                   (상대를 ×2)
             if(p!=null) p.value += 5        (내가 link한 대상을 +5)
             counter += r
```
> ⚠️ 매 호출마다 **this·o·p 세 객체**가 동시에 바뀐다. "지금 this=누구, o=누구, this.p=누구"를 적어가며 따라가야 함. link 관계: obj1.p=obj2, obj2.p=obj3, obj3.p=obj1.

### 한 단계씩 따라가기 (매 줄 "누가 this/o/p인지" 표기)

```
생성: counter=3 (3개 객체). print → 3

result1 = obj1.process(obj3):   (this=obj1, o=obj3, obj1.p=obj2)
   r = 10+30 = 40
   obj1.value -= 2 → 8
   obj3.value *= 2 → 60
   obj1.p(obj2).value += 5 → 25
   counter += 40 → 43
   return 40 → print 40
   상태: obj1=8, obj2=25, obj3=60 → print "8, 25, 60"

result2 = obj2.chainProcess(obj3, obj1):
   s1 = obj2.process(obj3): (this=obj2, o=obj3, obj2.p=obj3)
      r=25+60=85; obj2=23; obj3=120; obj2.p(obj3).value+=5→125; counter+=85→128
   s2 = obj3.process(obj1): (this=obj3, o=obj1, obj3.p=obj1)
      r=125+8=133; obj3=123; obj1=16; obj3.p(obj1).value+=5→21; counter+=133→261
   s3 = obj1.process(obj2): (this=obj1, o=obj2, obj1.p=obj2)
      r=21+23=44; obj1=19; obj2=46; obj1.p(obj2).value+=5→51; counter+=44→305
   obj2.value = obj2(51) + obj3(123) - obj1(19) = 155
   return s1+s2+s3 = 85+133+44 = 262 → print 262
   counter → print 305
   상태: obj1=19, obj2=155, obj3=123 → print "19, 155, 123"
```

### ✅ 정답

```
3
40
8, 25, 60
262
305
19, 155, 123
```
> ⚠️ process가 this·other·p 세 객체를 동시에 바꾸고 counter에 결과 누적. 단계마다 상태를 정확히 추적해야 함.

## Q14. C언어 — 분수 배낭(그리디)

### 문제

```c
// 가치/무게 비율 내림차순 정렬 후, 용량 c까지 채우는 분수 배낭
Item a[] = {{10,60},{20,100},{30,180},{25,75}};  // {무게, 가치}
int c = 50;
```

### 🤔 이 함수가 하는 일

**가치/무게 비율이 높은 순**으로 정렬 후, 용량이 남는 한 통째로 담고, 마지막은 분할해서 담는 분수 배낭.

### 1단계 — 비율 계산 후 정렬(내림차순)

```
{10,60}: 60/10 = 6.0
{20,100}: 100/20 = 5.0
{30,180}: 180/30 = 6.0
{25,75}: 75/25 = 3.0
정렬(비율 내림차순, 동률은 정렬 로직대로): [6.0, 6.0, 5.0, 3.0]
→ {10,60}, {30,180}, {20,100}, {25,75}  (비율 6,6,5,3)
```

### 2단계 — greedy로 용량 50 채우기

```
c=50
{10,60}: 무게10 ≤ 50 → 통째로 담기. val += 60. c=40
{30,180}: 무게30 ≤ 40 → 통째로. val += 180. c=10
{20,100}: 무게20 > 10 → 10만큼만: (10*100)/20 = 50. val += 50. c=0
종료
val = 60 + 180 + 50 = 290
```

### ✅ 정답

```

```

## Q17. Java — 생성자 가상 호출 + 필드 초기화

### 문제 (요지)

```java
class Animal { Animal(){ print(); } void print(){ println("Animal "); } }
class Dog extends Animal { String type="Dog "; Dog(){ print(); } void print(){ println(type); } }
class Cat extends Animal { String type="Cat "; Cat(){ print(); } void print(){ println(type); } }
Animal a = new Dog();  Animal b = new Cat();
```

### 한 줄씩 따라가기

```
new Dog():
   Dog() → super Animal() → Animal() print() → 실제타입 Dog → Dog.print() → type
      이때 Dog.type 미초기화 → null → "null"
   (Animal 종료) type="Dog " 초기화
   Dog() 본문 print() → Dog.print() → type="Dog " → "Dog "
new Cat(): 동일 → "null", "Cat "
```

### ✅ 정답

```
null
Dog 
null
Cat 
```
> ⚠️ 생성자 내 print()는 자식 메서드지만 그 시점 자식 필드 미초기화 → null. 이후 본문에서 "Dog "/"Cat ".

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트

1. **Q01 계차수열**: 인접 차분을 n번 반복. [4,6,10,16,24] → [2,4,6,8] → [2,2,2] → [0,0].

3. **Q06 진수+비트**: 016=14, 0x0F=15. ~x|y가 음수(-17). 이진수 계산 필수.
4. **Q07 COUNT DISTINCT**: 고객 수(3) vs 전체 건수(4). Alice 중복 주의.
5. **Q10 연쇄 참조**: process가 this·o·p 3객체 변경 + counter 누적. 단계별 상태 정확히.
6. **Q14 분수 배낭**: 비율 높은 순, 마지막은 분할. 60+180+50=290.
7. **Q17 생성자 가상 호출**: 미초기화 필드 null.
8. **Q19 스탬프 결합도**: 구조체 통째로 전달, 일부만 사용.

### 자주 나오는 용어 (이 회차 등장)

- **구조적 분석**: DFD/DD/Minispec/STD.
- **IPv4 vs IPv6**: 헤더/NAT/애니캐스트/체크섬/IPSec.
- **웹**: 시맨틱 웹/온톨로지.
- **결합도**: 스탬프(구조체 전달).
