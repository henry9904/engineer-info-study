## Q01. 후보키 — 유일성 / 최소성

### 지문 키워드 → 정답

| 빈칸 | 지문 키워드                          | 정답        |
| --- | ------------------------------ | --------- |
| ①  | 튜플을 명확히 구별 (모든 슈퍼키가 만족)        | **유일성(Uniqueness)** |
| ②  | 불필요한 속성을 포함하지 않음               | **최소성(Minimality)** |

### 메커니즘 — 키의 종류

```
슈퍼키   : 유일성 O, 최소성 X (불필요한 속성 포함 가능)
후보키   : 유일성 O + 최소성 O  ← 이 문제
기본키   : 후보키 중 대표로 선택한 것
대체키   : 기본키가 아닌 후보키
```
> 💡 후보키 = 슈퍼키(유일성) + 최소성. 최소성은 "속성 하나라도 빼면 유일성 깨짐".

### ✅ 정답

① 유일성  ② 최소성

## Q04. C언어 — 연결 리스트 홀짝 분리 재배열

### 문제

```c
void sort(Node* head) {
    if (!head || !head->next) return;
    Node *odd = head;              // 홀수 위치(1,3,5...) 시작
    Node *even = head->next;       // 짝수 위치(2,4,6...) 시작
    Node *even_head = even;        // 짝수 리스트의 머리 보관
    while (even && even->next) {
        odd->next = even->next;    // 홀수 다음 = 그다음 홀수
        odd = odd->next;
        even->next = odd->next;    // 짝수 다음 = 그다음 짝수
        even = even->next;
    }
    odd->next = even_head;         // 홀수 리스트 끝에 짝수 리스트 연결
}
// 리스트: 1→2→3→4→5→6 (insert가 역순 삽입으로 1..6 순서로 만듦)
```

### 🤔 이 함수가 하는 일

**홀수 번째 노드와 짝수 번째 노드를 분리**한 뒤, 홀수 리스트 뒤에 짝수 리스트를 이어붙인다. (odd-even linked list)

### 1단계 — 초기 리스트

`insert`를 values[5]부터 values[0]까지 역순 호출하므로 결국 `1→2→3→4→5→6`.

### 2단계 — while 한 바퀴씩 따라가기

```
초기: odd=1, even=2, even_head=2
[반복1] odd(1)->next = even(2)->next = 3   → 1→3
        odd = 3
        even(2)->next = odd(3)->next = 4   → 2→4
        even = 4
[반복2] odd(3)->next = even(4)->next = 5   → 3→5
        odd = 5
        even(4)->next = odd(5)->next = 6   → 4→6
        even = 6
[반복3] even(6)->next == NULL → while 종료
마지막: odd(5)->next = even_head(2)         → 5→2
```
홀수 체인: 1→3→5, 짝수 체인: 2→4→6, 연결: 5→2

### 3단계 — 최종 연결

```
1 → 3 → 5 → 2 → 4 → 6 → NULL
```

### ✅ 정답

```
1 3 5 2 4 6 
```
> 💡 홀수 위치(1,3,5)를 앞으로, 짝수 위치(2,4,6)를 뒤로 모으는 재배열.

## Q06. Python — 슬라이스 대입(확장/스텝)

### 문제

```python
lst = [1, 2, 3, 4, 5, 6, 7, 8, 9]
lst[2:7:2] = ['A', 'B', 'C']    # 인덱스 2,4,6 자리 교체
lst[::-3] = [10, 20, 30]        # 뒤에서부터 3칸 간격 교체
print(lst)
```

### 🤔 핵심 — 스텝 있는 슬라이스 대입은 개수가 정확히 맞아야

### 1단계 — `lst[2:7:2] = ['A','B','C']`

인덱스 2, 4, 6을 각각 'A', 'B', 'C'로 교체:
```
원본:   [1, 2, 3, 4, 5, 6, 7, 8, 9]
인덱스:   0  1  2  3  4  5  6  7  8
교체:          2→'A'   4→'B'   6→'C'
결과:   [1, 2, 'A', 4, 'B', 6, 'C', 8, 9]
```

### 2단계 — `lst[::-3] = [10,20,30]`

`[::-3]`은 **마지막 인덱스부터 -3씩**: 인덱스 8, 5, 2 순서로 대상.
```
lst[8] = 10   → 9 자리 → 10
lst[5] = 20   → 6 자리 → 20
lst[2] = 30   → 'A' 자리 → 30
결과: [1, 2, 30, 4, 'B', 20, 'C', 8, 10]
```

### ✅ 정답

```
[1, 2, 30, 4, 'B', 20, 'C', 8, 10]
```
> ⚠️ `[::-3]`은 인덱스 8→5→2 순으로 대입. 대입 리스트 [10,20,30]도 그 순서로 매칭.

## Q08. SQL — ANY / ALL 서브쿼리

### 문제

```sql
SELECT NAME, PRICE FROM PRODUCT
WHERE PRICE > ANY (SELECT PRICE FROM PRODUCT WHERE PRICE <= 3000)
AND   PRICE < ALL (SELECT PRICE FROM PRODUCT WHERE PRICE >= 3000);
```
가격: 사과3000, 바나나1500, 오렌지2000, 복숭아3500, 포도4000

### 🤔 핵심 — ANY는 "하나라도", ALL은 "전부"

```
> ANY (집합)  →  집합의 최솟값보다 크면 됨
< ALL (집합)  →  집합의 최솟값보다 작아야 함
```

### 단계별 트레이싱

```
서브쿼리1: PRICE<=3000 → {3000, 1500, 2000}, 최솟값 1500
  → PRICE > ANY = PRICE > 1500
서브쿼리2: PRICE>=3000 → {3000, 3500, 4000}, 최솟값 3000
  → PRICE < ALL = PRICE < 3000
조건: 1500 < PRICE < 3000
```
각 행 검사:
```
사과3000 : 3000<3000? X
바나나1500: >1500? X
오렌지2000: 1500<2000<3000 O  ← 통과
복숭아3500: <3000? X
포도4000  : <3000? X
```

### ✅ 정답

| NAME | PRICE |
| ---- | ----- |
| 오렌지  | 2000  |

## Q09. C언어 — 전위/후위 증감 복합식

### 문제

```c
int a = 5, b = 3, c = 2;
int result;
result = a++ + --b * c++ - ++a + b--;
printf("%d, %d, %d, %d\n", a, b, c, result);
```

### 🤔 핵심 — 후위(a++)는 사용 후 증가, 전위(++a)는 먼저 증가

식을 왼쪽부터 평가하되, 각 증감 시점을 추적한다.

### 한 항씩 따라가기

```
초기: a=5, b=3, c=2
식: a++ + --b * c++ - ++a + b--

① a++      → 값 5 사용, a→6
② --b      → b→2, 값 2
③ c++      → 값 2 사용, c→3
④ --b * c++ → 2 * 2 = 4   (②③ 결합, * 우선)
⑤ ++a      → a→7, 값 7
⑥ b--      → 값 2 사용, b→1

식 계산: 5 + 4 - 7 + 2
       = 5 + (2*2) - 7 + 2
       = 5 + 4 - 7 + 2 = 4
```
최종: a=7, b=1, c=3, result=4

### ✅ 정답

```
7, 1, 3, 4
```
> ⚠️ `--b`가 두 번 등장: 처음 b→2(값2), 나중 `b--`는 값2 사용 후 b→1. a는 a++(6)→++a(7).

## Q12. Java — 생성자 내 가상 호출 + 필드 은닉

### 문제 (요지)

```java
class A { A(){ print(); }  void print(){ System.out.print("A"); } }
class B extends A { String val="B"; B(){ val="B+"; } void print(){ System.out.print(val); } }
class C extends B { String val="C"; C(){ val="C+"; } void print(){ System.out.print(val); } }

A obj1 = new B();   System.out.print("-");
A obj2 = new C();   System.out.print("-");
obj1.print();       System.out.print("-");   obj2.print();
```

### 🤔 핵심 — 생성자에서 부른 print()는 실제 타입 메서드, 그때 필드는 미초기화

`A()` 생성자가 `print()`를 호출하는데, 실제 객체가 B/C면 오버라이딩된 print()가 실행된다. 하지만 **그 시점엔 자식 필드가 아직 초기화 전 → null**.

### 한 줄씩 따라가기

```
A obj1 = new B():
   B() → super A() → A() 안에서 print() 호출
   실제타입 B → B.print() → val 출력
   이때 B.val 아직 초기화 전(field init은 super 후) → null → "null"
   (이후 B.val="B" 초기화, B() 본문 val="B+")
"-" 출력

A obj2 = new C():
   C() → super B() → super A() → A() print() 호출
   실제타입 C → C.print() → C.val 미초기화 → null → "null"
   (이후 각 필드 초기화, 생성자 본문)
"-" 출력

obj1.print(): obj1 실제타입 B → B.print() → B.val="B+" → "B+"
"-" 출력
obj2.print(): obj2 실제타입 C → C.print() → C.val="C+" → "C+"
```

### ✅ 정답

```
null-null-B+-C+
```
> ⚠️ 생성 중 출력은 필드 미초기화라 null 두 번. 생성 완료 후엔 B+/C+.

## Q14. SQL — GROUP BY + AVG + 정렬

### 정답 SQL

```sql
SELECT CATEGORY, AVG(PRICE) AS AVG_PRICE
FROM PRODUCT
GROUP BY CATEGORY
ORDER BY AVG_PRICE DESC;
```
검산: 과일(1200,800,2200)/3=1400, 채소(500,700,600)/3=600 → 내림차순 과일, 채소

## Q15. C언어 — 시저 암호 + 문자열 비교/스왑

### 문제 (요지)

```c
void cipher(char *str, int shift){ // 알파벳/숫자를 shift만큼 밀기
    ... 대문자 %26, 소문자 %26, 숫자 %10 ...
}
char s1[]="A9c", s2[]="B8a";
char *arr[]={s1,s2};
cipher(arr[0], 2);   // s1 암호화
cipher(arr[1], 3);   // s2 암호화
if(compare(arr[0],arr[1])>0) swap(&arr[0],&arr[1]);  // 사전순 정렬
printf("%s-%s", arr[0], arr[1]);
```

### 1단계 — cipher(s1="A9c", 2)

```
'A' → (A-A+2)%26 + A = 2 → 'C'
'9' → (9-0+2)%10 + 0 = (11%10=1) → '1'
'c' → (c-a+2)%26 + a = (2+2=4) → 'e'
s1 = "C1e"
```

### 2단계 — cipher(s2="B8a", 3)

```
'B' → (1+3)%26 = 4 → 'E'
'8' → (8+3)%10 = 11%10 = 1 → '1'
'a' → (0+3)%26 = 3 → 'd'
s2 = "E1d"
```

### 3단계 — compare("C1e", "E1d")

```
첫 글자 'C'(67) vs 'E'(69) → 다름 → 67-69 = -2 (음수)
compare > 0? -2>0? 거짓 → swap 안 함
```
arr[0]="C1e", arr[1]="E1d"

### ✅ 정답

```
C1e-E1d
```
> ⚠️ compare가 음수(C<E)라 swap 미실행 → 원래 순서 유지.

## Q17. LOC 기반 비용 산정

### 문제

> LOC 10,000라인, 생산성 400라인/인월, 인건비 500만원/인월. 관리비 10%, QA 5%.

### 단계별 계산

```
① 개발 비용:
   필요 인월(MM) = 총LOC / 생산성 = 10,000 / 400 = 25 인월
   개발비 = 25 인월 × 500만원 = 12,500만원

② 관리 비용 = 개발비 × 10% = 12,500 × 0.10 = 1,250만원

③ QA 비용   = 개발비 × 5%  = 12,500 × 0.05 = 625만원
```
> ⚠️ "프로그래머 최대 5명"은 함정(비용 계산엔 인월만 필요, 인원수는 기간에만 영향).

### ✅ 정답

① 12,500만 원  ② 1,250만 원  ③ 625만 원

## Q20. C언어 — 복합 조건 + 증감 루프

### 문제

```c
int a = 0, b = 5, c = 10;
int result = 0;
for(int i = 0; i < 5; i++) {
    if((++a > b--) && (c-- > ++b)) result += a + b + c;
    else                          result += a - b + c;
}
printf("a=%d, b=%d, c=%d, result=%d", a, b, c, result);
```

### 🤔 핵심 — `&&` 단락 평가 + 증감이 조건 안에서 일어남

### 한 반복씩 따라가기

```
초기: a=0, b=5, c=10, result=0

[i=0] ++a→a=1;  b-- → 비교엔 5 사용, b→4
      1 > 5? 거짓 → && 단락(오른쪽 c--,++b 실행 안 됨)
      else: result += a - b + c = 1 - 4 + 10 = 7 → result=7
      (a=1, b=4, c=10)
[i=1] ++a→a=2;  b-- → 비교 4, b→3
      2 > 4? 거짓 → else
      result += 2 - 3 + 10 = 9 → result=16
      (a=2, b=3, c=10)
[i=2] ++a→a=3;  b-- → 비교 3, b→2
      3 > 3? 거짓 → else
      result += 3 - 2 + 10 = 11 → result=27
      (a=3, b=2, c=10)
[i=3] ++a→a=4;  b-- → 비교 2, b→1
      4 > 2? 참 → 오른쪽 평가: c-- → 비교 10, c→9;  ++b → b→2
              10 > 2? 참 → 전체 참
      result += a + b + c = 4 + 2 + 9 = 15 → result=42
      (a=4, b=2, c=9)
[i=4] ++a→a=5;  b-- → 비교 2, b→1
      5 > 2? 참 → c-- → 비교 9, c→8;  ++b → b→2
              9 > 2? 참 → 전체 참
      result += 5 + 2 + 8 = 15 → result=57
      (a=5, b=2, c=8)
```
최종: a=5, b=2, c=8, result=57

### ✅ 정답

```
a=5, b=2, c=8, result=57
```
> ⚠️ 앞 3회는 `++a > b--`가 거짓이라 오른쪽 c--/++b 미실행(c=10 유지). i=3부터 참이 되어 c 감소 시작.

## 📌 이 회차 핵심 암기 / 자주 틀리는 포인트

1. **Q04 홀짝 분리**: odd/even 포인터로 홀수 위치·짝수 위치를 분리 후 이어붙임 → 1 3 5 2 4 6.

3. **Q08 ANY/ALL**: `>ANY`=최솟값 초과, `<ALL`=최솟값 미만. 결국 1500<PRICE<3000 → 오렌지.
4. **Q09 증감**: `--b` 두 번(2, 그 후 b--로 2 사용→1), a는 a++(6)→++a(7). result=4.
5. **Q12 생성자 가상 호출**: 생성 중 print()는 자식 메서드지만 필드 미초기화 → null.

7. **Q20 단락 평가**: `++a>b--` 거짓이면 c--/++b 미실행 → c가 한동안 10 유지.
